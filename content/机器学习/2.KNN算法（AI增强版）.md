## 1️⃣ 完整知识库

## KNN算法原理 🔸 核心

### 定义与本质
KNN（K-Nearest Neighbors，K近邻算法）：如果一个样本在特征空间中的 K 个最相似的样本中的大多数属于某一个类别，则该样本也属于这个类别。

**核心特点**：
- ==惰性学习==（Lazy Learning）：没有显式的训练过程，训练阶段只是存储数据
- ==非参数模型==：不对数据分布做假设
- 可同时处理**分类**和**回归**问题

### 基础用法

**分类问题的处理流程（投票）**：
1. 计算未知样本到每一个训练样本的距离
2. 将训练样本根据距离升序排列
3. 取出距离最近的 K 个训练样本
4. 进行多数表决，统计 K 个样本中哪个类别最多
5. 将未知样本归属到出现次数最多的类别

**回归问题的处理流程（平均值）**：
1. 计算未知样本到每一个训练样本的距离
2. 将训练样本根据距离升序排列
3. 取出距离最近的 K 个训练样本
4. 将 K 个样本的目标值计算平均值
5. 作为未知样本的预测值

```python
"""
KNN算法介绍(K Nearest Neighbors), K近邻算法
原理:
    基于欧式距离(或者其它距离计算方式)计算测试集和每个训练集之间的距离,
    然后根据距离升序排列, 找到最近的K个样本.
    基于K个样本投票, 票数多的就作为最终预测结果 -> 分类问题.
    基于K个样本计算平均值, 作为最终预测结果 -> 回归问题.
实现思路:
    1. 分类问题: 有特征, 有标签, 且标签是不连续的(离散的)
    2. 回归问题: 有特征, 有标签, 且标签是连续的.
KNN算法, 分类问题思路如下:
    1. 计算测试集和每个训练的样本之间的距离.
    2. 基于距离进行升序排列.
    3. 找到最近的K个样本.
    4. K个样本进行投票.
    5. 票数多的结果, 作为最终的预测结果.
"""
from sklearn.neighbors import KNeighborsClassifier

# 准备数据
x_train = [[0, 0, 1], [1, 1, 0], [3, 10, 10], [4, 11, 12]]
y_train = [0, 0, 1, 1]
x_test = [[3, 11, 10]]

# 创建并训练模型 (K=3)
model = KNeighborsClassifier(n_neighbors=3)
model.fit(x_train, y_train)
print(f'K=3预测结果: {model.predict(x_test)}')  # 输出: [1]

# 对比: K=2时的预测结果
model2 = KNeighborsClassifier(n_neighbors=2)
model2.fit(x_train, y_train)
print(f'K=2预测结果: {model2.predict(x_test)}')  # 输出可能不同
```

### 进阶用法与原理

**K值选择的影响**：

| K值 | 效果 | 类比 |
|-----|------|------|
| 过小 | 过拟合，易受异常点影响 | 只听最近的1个人的意见，容易被忽悠 |
| 过大 | 欠拟合，预测趋向多数类 | 问所有人意见，结果没有区分度 |
| 合适 | 泛化能力好 | 综合考虑周围几个邻居的意见 |

**K值调优方法**：交叉验证、网格搜索（详见"超参数选择"模块）

### 避坑与局限
- KNN对==特征尺度非常敏感==，必须先做归一化/标准化
- K=N（全部样本）时，无论输入什么都预测为训练集中最多的类
- 数据量大时计算开销极高（需计算与所有训练样本的距离）

## 距离度量 🔸 核心

### 定义与本质
距离度量是KNN的核心计算，衡量两个样本在特征空间中的相似程度。样本距离越近则越相似。

### 基础用法

| 距离类型 | 公式 | 特点 |
|---------|------|------|
| 欧氏距离 | √(Σ(xi-yi)²) | 最常用，直线距离 |
![[image_008.png]]
| 曼哈顿距离 | Σ\|xi-yi\| | 城市街区距离，对异常值鲁棒 |
![[image_009.png]]
| 切比雪夫距离 | max(\|xi-yi\|) | 各维度最大差值 |
![[image_010.png]]
| 闵可夫斯基距离 | (Σ\|xi-yi\|ᵖ)^(1/p) | 上述三种的统一形式（p=2欧氏, p=1曼哈顿, p→∞切比雪夫） |
![[image_011.png]]

```python
import numpy as np
a = np.array([1, 2])
b = np.array([4, 6])
# 欧氏距离
dist = np.sqrt(np.sum((a - b) ** 2))
print(f'欧氏距离: {dist}')  # 输出：欧氏距离: 5.0
```

### 避坑与局限
- 欧氏距离对量纲敏感，不同特征尺度差异大时会被大数值特征主导
- 高维空间中欧氏距离趋于均匀分布（"维度灾难"），KNN在高维数据上效果往往不好

## 特征预处理 🔸 核心

### 定义与本质
当特征的单位或大小相差较大时，某些特征会==主导==模型的预测结果，导致其他特征的信息被淹没。特征预处理通过归一化或标准化将不同特征缩放到统一范围。

### 基础用法

**归一化（MinMaxScaler）**：将数据映射到 [min, max] 区间（默认 [0,1]）

$$x'' = \frac{x - x_{min}}{x_{max} - x_{min}} \times (mx - mi) + mi$$

```python
from sklearn.preprocessing import MinMaxScaler
scaler = MinMaxScaler(feature_range=(0, 1))
x_scaled = scaler.fit_transform(x)
# 适用于：传统精确小数据场景
```

![[image_012.png]]

**标准化（StandardScaler）**：将数据转换为均值为0、标准差为1的标准正态分布的数据

$$x' = \frac{x - \mu}{\sigma}$$

```python
from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()
x_scaled = scaler.fit_transform(x)
print(f'均值: {scaler.mean_}')   # 输出：均值: [各列均值]
print(f'标准差: {scaler.scale_}') # 输出：标准差: [各列标准差]
# 适用于：大数据集场景
```

![[image_013.png]]

> [!info] 🧠 速查卡片：StandardScaler
> **签名**：`StandardScaler()`
> **核心方法**：`fit_transform(X)` — 计算均值标准差并转换；`transform(X)` — 用已有参数转换
> **属性**：`mean_`（均值）、`scale_`（标准差）、`var_`（方差）
> 🎯 **最佳场景**：大数据集、含异常值的数据（标准化对异常值比归一化更鲁棒）

### 进阶用法与原理

> [!note] 💡 AI 扩展（基础）
> 归一化与标准化的核心区别：归一化受最大值和最小值影响，如果存在异常值，映射区间会被压缩，导致大部分正常值集中在很小的范围内。标准化基于均值和标准差，少量异常值对均值影响有限，因此更鲁棒。实践建议：大多数情况下优先使用标准化；只有当数据分布有明显边界且需要将值映射到特定范围（如图片像素[0,255]→[0,1]）时才用归一化。

### 避坑与局限
- **训练集用 `fit_transform`，测试集只用 `transform`**——测试集不能重新计算均值和标准差，否则导致数据泄露
- 归一化不适合含异常值的数据集（异常值会拉大极值范围）

> [!info] 课堂补充：方差与标准差
> **方差计算公式**：该列每个值和该列均值的差的平方和的平均值
> $$Var(X) = \frac{1}{n}\sum_{i=1}^{n}(x_i - \bar{x})^2$$
> **标准差计算公式**：方差开平方根
> $$Std(X) = \sqrt{Var(X)}$$
> 方差代表数据的离散程度，方差越大 → 数据越分散、不稳定；方差越小 → 数据越集中、稳定

## 超参数选择方法 🔸 核心

### 定义与本质
超参数是模型训练前需要手动设定的参数（如KNN中的K值），与模型通过训练自动学习的参数（权重）不同。超参数选择的目标是找到让模型泛化能力最好的参数组合。

### 基础用法

**交叉验证（Cross Validation）**：将训练集划分为 n 份（cv=10），轮流用其中1份做验证集、其余 n-1 份做训练，训练 n 次，取平均得分作为模型评估。

```text
交叉验证流程（cv=10）：
1. 第1次：第1份做验证集，其余做训练 → 评分s1
2. 第2次：第2份做验证集，其余做训练 → 评分s2
...
10. 第10次：第10份做验证集 → 评分s10
11. 最终得分 = mean(s1, s2, ..., s10)
12. 选定最优K后，用全部训练集重新训练，再用测试集评估
```

**网格搜索（Grid Search）**：自动遍历超参数的所有组合，每组都用交叉验证评估，选出最优参数组合。

```python
from sklearn.model_selection import GridSearchCV
from sklearn.neighbors import KNeighborsClassifier

estimator = KNeighborsClassifier()
param_dict = {'n_neighbors': [i for i in range(1, 11)]}
# 交叉验证4折 × 10个K值 = 40次训练
gc = GridSearchCV(estimator, param_dict, cv=4)
gc.fit(x_train, y_train)
print(f'最优K值: {gc.best_params_}')     # 输出：最优K值: {'n_neighbors': 3}
print(f'最优评分: {gc.best_score_}')       # 输出：最优评分: 0.9667
```

### 避坑与局限
- 网格搜索计算量随参数组合数线性增长，参数维度高时改用随机搜索（RandomizedSearchCV）
- 交叉验证的验证集仅用于选择超参数，最终评估仍需使用独立的测试集

## 分类问题评估方法 🔸 核心

### 定义与本质
混淆矩阵是分类问题评估的基础工具，通过统计预测结果与真实结果的组合来全面评估模型性能。

### 基础用法

**混淆矩阵四指标**：

|  | 预测正例 | 预测反例 |
|--|---------|---------|
| **真实正例** | TP（真正例） | FN（假反例） |
| **真实反例** | FP（假正例） | TN（真反例） |

![[image_054.png]]

**四大评估指标**：

| 指标              | 公式                              | 含义                 |
| --------------- | ------------------------------- | ------------------ |
| 准确率 (Accuracy)  | (TP + TN) / (TP + FP + TN + FN) | 预测正确的样本占总样本的比例     |
| 精确率 (Precision) | TP / (TP + FP)                  | 预测为正例中有多少是真正例（查准率） |
| 召回率 (Recall)    | TP / (TP + FN)                  | 真实正例中有多少被预测出来（查全率） |
| F1-Score        | 2×P×R / (P+R)                   | 精确率与召回率的调和平均       |

```python
from sklearn.metrics import classification_report
# 输出包含精确率、召回率、F1-score的完整报告
print(classification_report(y_test, y_pred))
```

### 进阶用法与原理

> [!note] 💡 AI 扩展（进阶）
> 精确率与召回率通常是此消彼长的关系。在医学诊断中，召回率更重要（漏诊的代价远大于误诊），因此可通过调低分类阈值来提高召回率；在垃圾邮件过滤中，精确率更重要（将正常邮件误判为垃圾邮件的代价更高）。AUC-ROC曲线是另一个重要的评估工具，它通过绘制不同阈值下的TPR（召回率）与FPR的关系曲线，用曲线下面积（AUC值）来综合评估分类器的性能，AUC越接近1越好。

### 避坑与局限
- 仅看准确率（Accuracy）不够，尤其在数据类别不均衡时（如99%正例、1%反例），全猜正例也有99%准确率
- F1-Score对少数类的识别能力更敏感，类别不均衡时优先参考

### 鸢尾花分类完整案例

```python
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler
from sklearn.neighbors import KNeighborsClassifier
from sklearn.metrics import accuracy_score

# 1. 加载数据集
iris = load_iris()
# 2. 划分训练集(80%)和测试集(20%)
x_train, x_test, y_train, y_test = train_test_split(
    iris['data'], iris['target'], test_size=0.2, random_state=42)

# 3. 特征工程：标准化
transfer = StandardScaler()
x_train = transfer.fit_transform(x_train)  # 训练集：fit + transform
x_test = transfer.transform(x_test)        # 测试集：只用 transform

# 4. 模型训练
estimator = KNeighborsClassifier(n_neighbors=3)
estimator.fit(x_train, y_train)

# 5. 模型预测
y_pre = estimator.predict(x_test)
print(f'预测值: {y_pre}')

# 自定义数据预测
my_data = [[4.8, 3.0, 5.2, 2]]
my_test = transfer.transform(my_data)
print(f'预测类别: {estimator.predict(my_test)}')  # 输出：预测类别: [2]
print(f'各分类概率: {estimator.predict_proba(my_test)}')

# 6. 模型评估
print(f'准确率: {estimator.score(x_test, y_test)}')  # 输出：准确率: 0.9667
print(f'准确率: {accuracy_score(y_test, y_pre)}')     # 输出：准确率: 0.9667
```

### 手写数字识别案例

> [!warning] 注意

```python
"""
数据文件 train.csv 和 test.csv 包含从 0 到 9 的手绘数字的灰度图像。
- 每个图像高 28 像素，宽 28 像素，共 784 个像素。
- 每个像素取值范围[0,255]，取值越大意味着该像素颜色越深
- 训练数据集（train.csv）共 785 列。第一列为 "标签"，为该图片对应的手写数字。
- 其余 784 列为该图像的像素值
"""
import matplotlib.pyplot as plt
import pandas as pd
from sklearn.model_selection import train_test_split
from sklearn.neighbors import KNeighborsClassifier
import joblib
from collections import Counter


# 1. 显示图片.
def show_digit(idx):
    # 1.1 加载数据.
    data = pd.read_csv('手写数字识别.csv')
    # 1.2 非法值校验.
    if idx < 0 or idx > len(data) - 1:
        return
    # 1.3 打印数据基本信息
    x = data.iloc[:, 1:]
    y = data.iloc[:, 0]
    print(f'数据基本信息: {x.shape}')
    print(f'类别数据比例: {Counter(y)}')

    # 显示图片
    # 1.4 将数据形状修改为: 28*28
    digit = x.iloc[idx].values.reshape(28, 28)
    # 1.5 关闭坐标轴标签
    plt.axis('off')
    # 1.6 显示图像
    plt.imshow(digit, cmap='gray')  # 灰色显示
    plt.show()


# 2. 训练模型.
def train_model():
    data = pd.read_csv('手写数字识别.csv')
    x = data.iloc[:, 1:]  # 784个像素特征
    y = data.iloc[:, 0]   # 数字标签(0-9)
    # 归一化到[0,1]
    x = x / 255
    # 分割训练集和测试集，保持类别比例
    x_train, x_test, y_train, y_test = train_test_split(
        x, y, test_size=0.2, stratify=y, random_state=21
    )
    # 训练KNN模型
    estimator = KNeighborsClassifier(n_neighbors=3)
    estimator.fit(x_train, y_train)
    # 评估
    print(f'测试集准确率: {estimator.score(x_test, y_test):.2f}')
    # 保存模型
    joblib.dump(estimator, 'model/knn.pth')

# 3. 测试模型.
def use_model():
    # 1. 读取图片
    img = plt.imread('data/demo.png')   # 灰度图, 28*28像素
    plt.imshow(img, cmap='gray')
    plt.show()

    # 2. 加载模型.
    estimator = joblib.load('model/knn.pth')

    # 3. 预测图片.
    img = img.reshape(1, -1)  # 形状从: (28, 28) => (1, 784)
    # print(img.shape)
    y_pred = estimator.predict(img)
    print(f'您绘制的数字是: {y_pred}')

# 在main函数中测试
if __name__ == '__main__':
    # show_digit(0)
    # train_model()
    use_model()
```

## 4️⃣ 避坑指南 & 易错对比

> [!warning] 易混概念对比
> | 概念A | 概念B | 区分要点 |
> |-------|-------|----------|
> | 归一化 | 标准化 | 归一化映射到[min,max]，受极值影响；标准化映射到均值0标准差1，更鲁棒 |
> | 精确率 | 召回率 | 精确率关注"预测为正的有多少是对的"，召回率关注"真实为正的有多少被找到" |
> | fit_transform | transform | fit_transform用于训练集（计算参数+转换），transform用于测试集（仅用已有参数转换） |
> | 参数 | 超参数 | 参数通过训练自动学习（如权重w），超参数需手动设定（如K值） |

> [!danger] 常见错误与规避
> - **错误**：测试集使用 `fit_transform` → **正确**：测试集只用 `transform`，避免数据泄露
> - **错误**：分类器传入浮点标签 `[0.1, 0.2]` → **正确**：分类器标签应为整数 `[0, 1]`，浮点标签用回归器
> - **错误**：直接比较未标准化的多维度特征距离 → **正确**：先标准化再计算距离
> - **错误**：K=1 做预测 → **正确**：K=1极易过拟合，建议通过交叉验证选择最优K

## 6️⃣ 应用场景与扩展

**KNN的实际应用场景**：
- 推荐系统：找到与目标用户最相似的K个用户，基于其偏好进行推荐
- 手写数字/图像识别：经典的入门级图像分类任务
- 医学诊断：基于历史病例的最近邻进行疾病预测
- 异常检测：与正常样本距离过远的样本可能是异常

> [!note] 💡 AI 扩展（基础）
> KNN的"维度灾难"问题：当特征维度很高时（如>20维），任意两个样本之间的距离趋于相近，KNN的区分能力大幅下降。解决方案包括：先做特征降维（PCA）、使用距离加权（距离越近权重越大）、或换用基于树的算法。
