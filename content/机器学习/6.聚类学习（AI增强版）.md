## 聚类算法基础 🔸 核心

### 定义与本质
聚类算法是一种==无监督学习==算法，根据样本之间的相似性，将样本划分到不同的类别中。其目的是在没有先验知识的情况下，自动发现数据集中的内在结构和模式。

**核心特点**：
- 无标签——不需要预先知道类别
- 基于相似性——距离越近的样本越可能被分到同一类
- 常用相似度度量——欧氏距离

### 基础用法

**聚类算法分类**：

按聚类颗粒度分：
- 硬聚类：每个样本只属于一个簇（如 K-Means）
- 软聚类：每个样本以一定概率属于多个簇（如高斯混合模型）

![[image_058.png]]

按实现方法分：

| 方法 | 算法 | 核心思想 |
|------|------|---------|
| 基于质心 | K-Means | 按照质心分类，通用且普遍 |
| 层次聚类 | 凝聚/分裂 | 对数据逐层划分，直到达到类别数 |
| 基于密度 | DBSCAN | 基于密度，可发现任意形状的簇 |
| 基于图论 | 谱聚类 | 基于图论，适用于复杂结构数据 |

```python
from sklearn.cluster import KMeans
model = KMeans(n_clusters=3, random_state=42)
y_pred = model.fit_predict(x)  # 同时计算聚类中心并预测类别
```

> [!info] 🧠 速查卡片：KMeans
> **签名**：`KMeans(n_clusters=8, random_state=None)`
> **核心参数**：`n_clusters`（聚类中心数量，即K值）
> **核心方法**：`fit_predict(X)` — 计算聚类中心并预测类别；`fit(X)` + `predict(X)` — 分步执行
> **属性**：`cluster_centers_`（聚类中心坐标）、`labels_`（聚类标签）、`inertia_`（SSE值）
> 🎯 **最佳场景**：簇形状近似球形、数据量中等、预先大致知道K值范围

### 避坑与局限
- K-Means 对初始质心位置敏感，不同初始化可能产生不同结果（sklearn 默认使用 K-Means++ 优化初始化）
- K-Means 假设簇为凸形（球形），不适合发现非凸形状的簇
- 需要预先指定 K 值，但实际中往往不知道最优 K

## K-Means 算法流程 🔸 核心

### 定义与本质
K-Means 是一种基于质心的划分聚类方法，通过迭代更新质心位置使簇内距离最小化。

### 基础用法

```text
K-Means 算法流程：
1. 事先确定常数 K（最终聚类类别数）
2. 随机选择 K 个样本点作为初始聚类中心（质心）
3. 计算每个样本到 K 个中心的距离，选择最近的中心作为标记类别
4. 根据每个类别中的样本点，重新计算新的聚类中心（取均值）
5. 如果新中心点与原中心点相同（或变化极小），则停止；否则回到步骤3
```

> [!info] 🧠 分步演算：K-Means迭代过程
> 假设 K=2，初始质心为 C1=(1,2), C2=(6,8)，样本为 A(2,3), B(4,6), C(7,7), D(1,1)
>
> | 步骤 | 操作 | A归属 | B归属 | C归属 | D归属 | 新C1 | 新C2 |
> |------|------|-------|-------|-------|-------|------|------|
> | 第1次 | 计算距离，分配到最近质心 | C1 | C2 | C2 | C1 | (1.5,2) | (5.67,7) |
> | 第2次 | 用新质心重新分配 | C1 | C2 | C2 | C1 | (1.5,2) | (5.67,7) |
> | 结果 | 质心不再变化，收敛 | - | - | - | - | - | - |

### 避坑与局限
- 初始质心选择不当可能导致收敛到局部最优（K-Means++ 通过概率选择缓解此问题）
- 算法可能不收敛（质心持续振荡），通常设置最大迭代次数

## 聚类评估方法 🔸 核心

### 定义与本质

**SSE（误差平方和，Sum of Squared Error）**：衡量聚类效果的核心指标。

$$SSE = \sum_{i=1}^{k} \sum_{p \in C_i} \|p - m_i\|^2$$

- Ci 表示第 i 个簇
- p 表示簇内样本
- mi 表示质心
- **SSE 越小**，数据点越接近质心，聚类效果越好

![[image_059.png]]

### 基础用法

**其他聚类评估指标**：

| 指标 | 含义 | 范围 | 越大/小越好 |
|------|------|------|------------|
| SSE | 簇内样本到质心的距离之和 | ≥0 | 越小越好 |
| 轮廓系数 | 衡量簇内紧密度 vs 簇间分离度 | [-1, 1] | 越大越好 |
| CH指数 | 簇间方差 / 簇内方差 | ≥0 | 越大越好 |

**SC 轮廓系数（Silhouette Coefficient）**：

综合考虑簇内的内聚程度与簇间的分离程度。

![[image_060.png]]

- SC 取值范围 [-1, 1]
- **SC 越大**，聚类效果越好
- SC 接近 1 表示样本与自身簇内样本很相似，与其他簇样本差异大
- SC 接近 0 表示样本在两个簇的边界上
- SC 为负表示样本可能被分到了错误的簇

**CH 轮廓系数（Calinski-Harabasz Index）**：

综合考虑簇内的内聚程度、簇间的分离程度、质心的个数。

![[image_061.png]]

- CH 取值范围 ≥0
- **CH 越大**，聚类效果越好
- CH 指数计算效率高，适合大数据集评估

```python
from sklearn.metrics import calinski_harabasz_score, silhouette_score
ch_score = calinski_harabasz_score(x, y_pred)
si_score = silhouette_score(x, y_pred)
print(f'CH指数: {ch_score}')   # 输出：CH指数: 数值越大越好
print(f'轮廓系数: {si_score}') # 输出：轮廓系数: 越接近1越好
```

### 避坑与局限
- SSE 会随 K 值增大而持续减小（K=n 时 SSE=0），因此不能单独用 SSE 选择 K 值
- 轮廓系数计算开销较大，大数据集上使用 CH 指数更高效

## K值确定：肘部法则 🔸 核心

### 定义与本质
肘部法则（Elbow Method）通过绘制不同 K 值下的 SSE 曲线，找到 SSE 下降率突然变缓的"拐点"作为最优 K 值。

### 基础用法

```text
肘部法则步骤：
1. 对 K = 1, 2, 3, ..., n 依次执行 K-Means 聚类
2. 每次记录 SSE 值
3. 绘制 K-SSE 曲线
4. 找到 SSE 下降率突然变缓的拐点（"肘部"）即为最优 K
```

```python
from sklearn.cluster import KMeans
import matplotlib.pyplot as plt

sse_list = []
for k in range(1, 10):
    kmeans = KMeans(n_clusters=k, random_state=42)
    kmeans.fit(x)
    sse_list.append(kmeans.inertia_)  # inertia_ 即 SSE
print(f'SSE列表: {sse_list}')
# 绘制肘部图找拐点
plt.plot(range(1, 10), sse_list, 'bo-')
plt.xlabel('K值')
plt.ylabel('SSE')
plt.show()
```

> [!note] 💡 AI 扩展（基础）
> 肘部法则的局限性：当 SSE 曲线没有明显的"肘部"拐点时（如数据本身没有清晰的自然聚类），该方法难以确定最优 K 值。此时可结合轮廓系数法：遍历不同 K 值，选择轮廓系数最大的 K。此外，DBSCAN 等密度聚类算法不需要预先指定 K 值，适合对 K 值完全未知的数据进行探索性分析。

### 避坑与局限
- 肘部法则依赖主观判断，不同人可能选择不同的"拐点"
- 当数据维度很高时，距离度量失效，肘部法则效果下降
- 在决定什么时候停止训练时，肘形判据同样有效：数据通常有更多的噪音，在增加分类无法带来更多回报时，我们停止增加类别

## K-Means++初始化原理 🔺 难点

### 定义与本质

标准 K-Means 随机初始化质心，容易导致收敛到局部最优。K-Means++ 通过**概率化选择**初始质心来优化，是 sklearn 的默认初始化策略。

### 进阶用法与原理

**K-Means++ 初始化步骤**：

1. 从数据集中随机选择第一个质心
2. 对于每个样本 $x$，计算它到最近已有质心的距离 $D(x)$
3. 以概率 $\frac{D(x)^2}{\sum_{x'}D(x')^2}$ 选择下一个质心（距离越远，被选概率越大）
4. 重复步骤2-3，直到选出 $K$ 个质心

**为什么有效**：

概率与距离平方成正比，使得初始质心彼此远离，分布更均匀。理论证明，K-Means++ 的期望目标函数值（SSE）在 $O(\log K)$ 倍以内接近最优解。

```python
from sklearn.cluster import KMeans
# sklearn 默认即 K-Means++
model = KMeans(n_clusters=3, init='k-means++', random_state=42)
# 对比随机初始化（通常效果更差）
model_random = KMeans(n_clusters=3, init='random', random_state=42)
```

> [!note] 💡 AI 扩展（进阶）
> 对于大规模数据集（样本数 > 10万），K-Means++ 的初始化阶段计算开销较大。此时可使用 **Mini-Batch K-Means**，它在每次迭代中仅使用一小批样本来更新质心，训练速度快 3~5 倍，但 SSE 可能略高。sklearn 中为 `MiniBatchKMeans` 类，用法与 `KMeans` 相同。

### 避坑与局限
- K-Means++ 虽然改善了初始化，但不能保证全局最优
- 对于高维数据，距离度量可能失效，K-Means++ 的优势会减弱

### K-Means 完整案例（make_blobs + 可视化 + 多K值对比）

```python
"""
KMeans完整案例：使用make_blobs生成模拟数据，进行聚类并可视化
步骤：
    1. 导包 sklearn.cluster.KMeans, sklearn.datasets.make_blobs
    2. 创建数据集（make_blobs生成模拟数据）
    3. 实例化Kmeans模型并预测
    4. 展示聚类效果（matplotlib可视化）
    5. 评估聚类效果好坏（CH指数）
"""
import matplotlib.pyplot as plt
from sklearn.datasets import make_blobs
from sklearn.cluster import KMeans
from sklearn.metrics import calinski_harabasz_score

# 创建数据集
# X为样本特征，Y为样本簇类别，共1000个样本，每个样本2个特征，共4个簇，
# 簇中心在[-1,-1], [0,0],[1,1], [2,2]，簇方差分别为[0.4, 0.2, 0.2, 0.2]
X, y = make_blobs(n_samples=1000, n_features=2, 
                  centers=[[-1, -1], [0, 0], [1, 1], [2, 2]],
                  cluster_std=[0.4, 0.2, 0.2, 0.2],
                  random_state=9)

# 数据集可视化
plt.scatter(X[:, 0], X[:, 1], marker='o')
plt.title('原始数据分布')
plt.show()

# 分别尝试 n_clusters=2, 3, 4，查看聚类效果
for k in [2, 3, 4]:
    y_pred = KMeans(n_clusters=k, random_state=9).fit_predict(X)
    plt.scatter(X[:, 0], X[:, 1], c=y_pred)
    plt.title(f'K={k} 的聚类效果')
    plt.show()
    # 用 Calinski-Harabasz Index 评估聚类分数
    ch_score = calinski_harabasz_score(X, y_pred)
    print(f'K={k}, CH指数: {ch_score:.2f}')
```

### K-Means 基础聚类（简化版）

```python
from sklearn.cluster import KMeans
from sklearn.metrics import calinski_harabasz_score
import numpy as np

# 准备数据
x = np.array([[1, 2], [1, 4], [1, 0],
              [4, 2], [4, 4], [4, 0]])

# 创建K-Means模型（K=2）

print(f'CH指数: {ch:.2f}')  # 输出：CH指数: 数值越大越好
```

## 4️⃣ 避坑指南 & 易错对比

> [!warning] 易混概念对比
> | 概念A | 概念B | 区分要点 |
> |-------|-------|----------|
> | 聚类（无监督） | 分类（有监督） | 聚类无标签自动分组，分类有标签预测类别 |
> | K-Means | DBSCAN | K-Means需预设K值且簇为球形，DBSCAN无需K值可发现任意形状簇 |
> | 硬聚类 | 软聚类 | 硬聚类每个样本只属于一个簇，软聚类每个样本以概率属于多个簇 |
> | SSE | CH指数 | SSE越小越好（簇内距离），CH指数越大越好（簇间/簇内方差比） |

> [!danger] 常见错误与规避
> - **错误**：K值设太大或太小 → **正确**：使用肘部法则或轮廓系数法确定最优K
> - **错误**：不做标准化直接聚类 → **正确**：K-Means基于距离度量，特征尺度不同会严重影响结果
> - **错误**：对任意形状数据用K-Means → **正确**：非球形簇应考虑 DBSCAN 或谱聚类
