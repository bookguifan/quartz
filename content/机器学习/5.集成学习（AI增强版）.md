# [day05] 学习笔记｜集成学习（AI 增强版）

## 📌 核心速览
> [!summary] 核心速览
> - **集成学习**：组合多个弱学习器形成强学习器，核心思想是"三个臭皮匠，赛过诸葛亮"，提升整体预测精度。
> - **Bagging策略**：并行训练多个独立学习器，采用有放回抽样构建不同训练集，随机森林是其典型代表。
> - **随机森林**：基于数据与特征的双重随机性构建多棵决策树，通过平权投票或多数表决得出最终结果。
> - **Boosting策略**：串行训练学习器，后一个关注前一个的错误，通过加权投票组合降低模型偏差。
> - **AdaBoost**：通过调整错分样本权重迭代提升，使被错分样本在后续学习器中获得更大关注。
> - **GBDT**：梯度提升决策树，用负梯度拟合残差，逐步减小模型预测误差，实现精准预测。
> - **XGBoost**：在GBDT基础上加入二阶泰勒展开和正则化，训练高效快速，是Kaggle竞赛的王牌算法。

## 集成学习基础 🔸 核心

### 定义与本质
集成学习（Ensemble Learning）通过多个模型的组合形成一个精度更高的模型。参与组合的模型称为==弱学习器==（基学习器），组合后的模型称为强学习器。

**核心思想**：多个分类器组合，实现预测效果更好的集成分类器。

### 基础用法

**集成学习两大策略对比**：

| 维度 | Bagging | Boosting |
|------|---------|----------|
| 训练方式 | 并行（各学习器独立训练） | 串行（依次训练，后一个关注前一个的错误） |
| 抽样方式 | 有放回抽样产生不同训练集 | 调整样本分布（错误样本权重增大） |
| 结合策略 | 平权投票/多数表决 | 加权投票 |
| 代表算法 | 随机森林 | AdaBoost、GBDT、XGBoost |
| 过拟合风险 | 低（多样性降低过拟合） | 较高（需控制迭代次数） |
| 偏差-方差 | 降低方差 | 降低偏差 |

![[image_057.png]]

### 进阶用法与原理

> [!note] 💡 AI 扩展（进阶）
> Boosting 的本质是将多个弱学习器逐步提升为强学习器，其工作机制：（1）从初始训练集训练出第一个基学习器；（2）根据基学习器的表现调整训练样本分布，使被错分的样本在后续获得更大关注；（3）基于调整后的分布训练下一个基学习器；（4）重复 T 次后，将 T 个基学习器加权结合。经典实现包括 AdaBoost（自适应提升）、GBDT（梯度提升树）、XGBoost（极致优化的GBDT）、LightGBM。Boosting 系列算法在 Kaggle 竞赛和工业界中广泛使用，是表格数据任务的首选方案。

### 避坑与局限
- Bagging 要求基学习器之间有差异性，否则组合后无提升
- Boosting 对噪声数据敏感，噪声样本会被反复关注导致过拟合

## 随机森林算法 🔸 核心

### 定义与本质
随机森林（Random Forest）是基于 ==Bagging== 思想实现的一种集成学习算法，采用==决策树==作为每个弱学习器。

### 基础用法

**随机森林的两大随机性**：
1. **数据随机**：有放回抽样（Bootstrap）产生不同的训练子集
2. **特征随机**：每个节点随机选择 n 个特征（n < 总特征数）进行最优划分

**随机森林训练与预测流程**：

```text
训练阶段：
1. 有放回地从原始数据集中抽样，产生 k 个训练子集
2. 每棵树在训练时，每个节点随机选择 n 个候选特征
3. 在候选特征中选择最优特征和划分点进行分裂
4. 重复直到生成 k 棵决策树

预测阶段：
1. 每棵树独立做出预测
2. 分类问题：平权投票，多数表决
3. 回归问题：取所有树预测结果的平均值
```

![[image_041.png]]

```python
from sklearn.ensemble import RandomForestClassifier
model = RandomForestClassifier(n_estimators=100, max_depth=6, random_state=9)
model.fit(x_train, y_train)
print(f'准确率: {model.score(x_test, y_test)}')
```

> [!info] 🧠 速查卡片：RandomForestClassifier
> **签名**：`RandomForestClassifier(n_estimators=100, criterion='gini', max_depth=None, max_features='auto', bootstrap=True)`
> **核心参数**：`n_estimators`（树的数量）、`max_depth`（最大深度）、`max_features`（候选特征数）、`bootstrap`（是否有放回抽样）
> **核心方法**：`fit(X,y)` 训练、`predict(X)` 预测、`score(X,y)` 评分
> 🎯 **最佳场景**：中等规模表格数据、需要比单棵决策树更稳定的效果

### 进阶用法与原理

**随机森林 vs 单棵决策树**：

| 维度   | 单棵决策树   | 随机森林         |
| ---- | ------- | ------------ |
| 稳定性  | 对数据变化敏感 | 鲁棒，结果稳定      |
| 过拟合  | 容易过拟合   | 不易过拟合（双重随机性） |
| 可解释性 | 强（可可视化） | 弱（多棵树的组合）    |
| 训练速度 | 快       | 慢（需训练多棵树）    |

> [!note] 💡 AI 扩展（基础）
> 随机森林的双重随机性（数据随机+特征随机）充分保证了每棵树之间的独立性，使得投票结果更准确。即使每棵决策树没有进行剪枝，随机森林也不会过拟合——因为每棵树只看到部分数据和部分特征，过拟合的部分会被其他树的正确预测"稀释"掉。实践中，`n_estimators` 设为 100~500 是常见选择，`max_features` 默认为 `sqrt(总特征数)`。

### 避坑与局限
- 随机森林的可解释性不如单棵决策树
- 当特征维度非常高时（如文本特征），随机森林可能不如 Boosting 系列算法

## 随机森林核心问题解答

> **为什么要随机抽样训练集？**
> 如果不随机抽样，每棵树训练集都一样，最终分类结果也完全一样，集成没有意义。

> **为什么要有放回地抽样？**
> 如果无放回抽样，每棵树的训练样本完全没有交集，每棵树都是"有偏的、片面的"，投票结果不可靠。有放回抽样使各棵树有差异但不完全不同，保证投票的可靠性。

## AdaBoost算法 🔸 核心

### 定义与本质
AdaBoost（Adaptive Boosting，自适应提升）是基于 Boosting 思想实现的一种集成学习算法。核心思想是**通过逐步提高那些被前一步分类错误的样本的权重来训练一个强分类器**。

**两个核心步骤**：

**权值调整**：提高被前一轮基分类器错误分类样本的权值，降低被正确分类样本的权值。使得没有得到正确分类的样本，由于权值的加大而受到后一轮基分类器的更大关注。

**基分类器组合**：采用加权多数表决的方法——分类误差率较小的弱分类器权值大，在表决中起较大作用；分类误差率较大的弱分类器权值小。

![[image_042.png]]

### 避坑与局限
- AdaBoost 对噪声样本敏感，噪声样本会被反复加大权重
- 弱分类器不能太弱（至少要比随机猜测强）

## GBDT（梯度提升树）🔺 难点

### 定义与本质
GBDT（Gradient Boosting Decision Tree）是提升树的一种改进算法。核心思想是**用损失函数的负梯度来拟合残差**。

### 基础用法

**直观理解**：假如要预测一个人的年龄（30岁），先用20岁去拟合，发现损失（残差）有10岁；再用6岁去拟合剩下的损失，差距还有4岁；第三轮用3岁拟合，差距只剩1岁。每一轮迭代，拟合的误差都会减小，最后将每次拟合的结果加起来就是模型输出。

![[image_001.png]]

**GBDT 的核心机制**：

$$L(y, f_i(x)) = L(y, f_{i-1}(x) + h_i(x))$$

- 前一轮迭代得到的强学习器：$f_{i-1}(x)$
- 本轮迭代的目标弱学习器：$h_i(x)$
- 让本轮损失最小化

当采用**平方损失函数**时：

![[image_002.png]]

损失函数为平方损失，则每个样本要拟合的负梯度为：

![[image_003.png]]

**结论**：GBDT 拟合的负梯度就是残差（回归问题）。如果进行的是分类问题，损失函数变为 logloss，拟合的目标值就是该损失函数的负梯度值。

### 基础用法

**GBDT 算法流程**：
1. 初始化弱学习器（目标值的均值作为预测值）
2. 迭代构建学习器，每一个学习器拟合上一个学习器的负梯度
3. 直到达到指定的学习器个数
4. 输入未知样本时，将所有弱学习器的输出结果组合起来作为强学习器的输出

### 避坑与局限
- GBDT 串行训练，无法并行，训练速度较慢
- 对异常值敏感（平方损失会放大异常值影响）

## XGBoost 🔺 难点

### 定义与本质
XGBoost（Extreme Gradient Boosting，极端梯度提升树）是集成学习方法的王牌，在 Kaggle 数据挖掘比赛中，大部分获胜者用了 XGBoost。

**XGBoost 是对 GBDT 的三大改进**：

| 改进点 | GBDT | XGBoost |
|--------|------|---------|
| 泰勒展开 | 一阶导数（梯度） | **二阶泰勒展开**（梯度+海森矩阵） |
| 正则化 | 无 | 损失函数中加入**正则化项** |
| 分裂指标 | 基于基尼指数/信息增益 | **自创打分函数**，考虑树的复杂度 |

### 基础用法

**目标函数**：

![[image_004.png]]

预测值和真实值经过损失函数计算损失，求解所有样本的平均损失并使其最小。为了降低模型复杂度、防止过拟合，在损失函数中添加正则化项：

![[image_005.png]]

**XGBoost 的树构建方法（打分函数）**：

![[image_006.png]]

该公式从树的损失函数和树的复杂度两个角度衡量一棵树的优劣。构建树时的使用逻辑：

1. 对树中的每个叶子结点尝试进行分裂
2. 计算分裂前 - 分裂后的分数（gain）：
   - 如果 gain > 0，分裂后损失更小，**考虑此次分裂**
   - 如果 gain < 0，分裂后分数更大，**不建议分裂**
3. 停止分裂条件：达到最大深度、叶子结点样本数量低于阈值等

```python
import pandas as pd
from sklearn.model_selection import train_test_split, GridSearchCV
from sklearn.tree import DecisionTreeClassifier
from sklearn.ensemble import RandomForestClassifier
from sklearn.metrics import classification_report

# 1. 加载数据
titanic = pd.read_csv('data/泰坦尼克号.csv')
X = titanic[['Pclass', 'Age', 'Sex']]
y = titanic['Survived']

# 2. 特征工程：填补缺失值 + 独热编码
# inplace原地替换原数据, 带来一定的副作用, 未来的某个版本 这个属性会被移除掉  
# x['Age'].fillna(x['Age'].mean(), inplace=True) # inplace=True  原地修改Age数据  
# x.fillna({ 'Age': x['Age'].mean()}, inplace=True)  
x['Age'] = x['Age'].fillna(x['Age'].mean())
X = pd.get_dummies(X)

# 3. 数据切分
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.25, random_state=22)

# 4. 单棵决策树
dtc = DecisionTreeClassifier()
dtc.fit(X_train, y_train)
print(f'决策树准确率: {dtc.score(X_test, y_test)}')

# 5. 随机森林
rfc = RandomForestClassifier(max_depth=6, random_state=9)
rfc.fit(X_train, y_train)
print(f'随机森林准确率: {rfc.score(X_test, y_test)}')

# 6. 性能评估
print("决策树报告:", classification_report(y_test, dtc.predict(X_test)))
print("随机森林报告:", classification_report(y_test, rfc.predict(X_test)))
```

### XGBoost：红酒品质预测

```python
import joblib
import xgboost as xgb
import pandas as pd
import numpy as np
from sklearn.model_selection import train_test_split, StratifiedKFold, GridSearchCV
from sklearn.metrics import classification_report

def test01():
    # 1. 加载训练数据
    data = pd.read_csv('data/红酒品质分类.csv')
    x = data.iloc[:, :-1]
    y = data.iloc[:, -1] - 3  # 标签偏移：原始标签为3~8，偏移后为0~5，适配XGBoost多分类
    # 2. 数据集分割
    x_train, x_valid, y_train, y_valid = train_test_split(
        x, y, test_size=0.2, stratify=y, random_state=22)
    # 存储数据
    pd.concat([x_train, y_train], axis=1).to_csv('data/红酒品质分类-train.csv')
    pd.concat([x_valid, y_valid], axis=1).to_csv('data/红酒品质分类-valid.csv')

def test02():
    train_data = pd.read_csv('data/红酒品质分类-train.csv')
    valid_data = pd.read_csv('data/红酒品质分类-valid.csv')
    x_train = train_data.iloc[:, :-1]
    y_train = train_data.iloc[:, -1]
    x_valid = valid_data.iloc[:, :-1]
    y_valid = valid_data.iloc[:, -1]
    # XGBoost模型训练
    estimator = xgb.XGBClassifier(
        n_estimators=100,
        objective='multi:softmax',
        eval_metric='merror',
        eta=0.1,
        use_label_encoder=False,
        random_state=22)
    estimator.fit(x_train, y_train)
    # 模型评估
    y_pred = estimator.predict(x_valid)
    print(classification_report(y_true=y_valid, y_pred=y_pred))
    # 模型保存
    joblib.dump(estimator, 'model/xgboost.pth')
```

> [!warning] 本代码依赖外部文件：红酒品质分类.csv

## 4️⃣ 避坑指南 & 易错对比

> [!warning] 易混概念对比
> | 概念A | 概念B | 区分要点 |
> |-------|-------|----------|
> | Bagging | Boosting | Bagging并行+平权投票降低方差，Boosting串行+加权投票降低偏差 |
> | 随机森林 | 决策树 | 随机森林是多棵决策树的集成，比单棵树更稳定、不易过拟合 |
> | 有放回抽样 | 无放回抽样 | 有放回保证各棵树有重叠（~63.2%），无放回导致完全独立"有偏" |
> | `n_estimators` | `max_depth` | 树的数量（越多越稳定但越慢）、树的最大深度（控制过拟合） |

> [!danger] 常见错误与规避
> - **错误**：`pd.read.csv()` → **正确**：`pd.read_csv()`（方法名错误）
> - **错误**：随机森林树数量太少（如10棵） → **正确**：一般设 100~500 棵
> - **错误**：不处理缺失值直接训练 → **正确**：先填补或删除缺失值
