# [day03] 学习笔记｜线性回归（AI 增强版）

## 📌 核心速览
> [!summary] 核心速览
> - **线性回归**：有监督学习中经典的回归算法，通过线性方程建模自变量与因变量的关系。
> - **一元/多元回归**：一元回归处理单特征，多元回归处理多特征，方程形式为 y = wx + b。
> - **损失函数**：MSE、RMSE、MAE用于衡量预测值与真实值之间的误差大小。
> - **梯度下降**：通过迭代更新参数最小化损失函数，包含FGD、SGD、mini-batch三种变体。
> - **学习率**：控制参数更新步长的关键超参数，过大导致震荡，过小收敛缓慢。
> - **正则化**：L1(Lasso)和L2(Ridge)通过约束权重防止过拟合，提升模型泛化能力。
> - **sklearn实现**：`LinearRegression`基于正规方程直接求解，`SGDRegressor`基于梯度下降迭代优化。
> - **停止条件**：梯度下降通过 max_iter、tol、梯度≈0、早停四种条件判断收敛，防止无限迭代。
> - **数值演算**：通过具体样本的梯度下降迭代，直观理解参数从初始值向最优解逼近的完整过程。

## 线性回归基础 🔸 核心

### 定义与本质
线性回归（Linear Regression）是利用回归方程对一个或多个自变量（特征值）和因变量（目标值）之间的关系进行建模的一种分析方式。属于==有监督学习==，有特征, 有标签, 标签为连续值。

### 基础用法

**一元线性回归**：1个特征 + 1个标签

$$y = wx + b$$

- w（Weight）：权重（数学中的斜率 k）
- b（Bias）：偏置（数学中的截距）

**多元线性回归**：多个特征 + 1个标签

$$y = w_1x_1 + w_2x_2 + \dots + w_nx_n + b = W^T X + b$$

```python
from sklearn.linear_model import LinearRegression
x_train = [[160], [166], [172], [174], [180]]
y_train = [56.3, 60.6, 65.1, 68.5, 75]
model = LinearRegression()
model.fit(x_train, y_train)
print(f'权重: {model.coef_}')       # 输出：权重: [0.9294]
print(f'偏置: {model.intercept_}')  # 输出：偏置: -93.2735
y_pre = model.predict([[176]])
print(f'预测值: {y_pre}')           # 输出：预测值: [70.3048]
```

- 线性回归假设特征与目标之间是线性关系，非线性关系需要特征工程或换用多项式回归

## 损失函数 🔸 核心

### 定义与本质
损失函数衡量模型预测值与真实值之间的差距，目标是通过优化方法==最小化==损失函数。**用于描述 每个样本点 和 其预测值之间关系的, 让损失函数最小, 就是让 误差和小, 线性回归效率, 评估就越高.**

**误差概念**：预测值 ŷ 与真实值 y 的差（y - ŷ）
**损失函数**：衡量每个样本预测值与真实值效果的函数  (Loss Function, 也叫成本函数, 代价函数, 目标函数, Cost Function)

```text
思路1: 正规方程法.(求导,求偏导,矩阵运算)
思路2: 梯度下降法.(全梯度下降FGD,随机梯度下降SGD,小批量梯度下降mini-batch,随机平均梯度下降算法)
  流程:1.给定初始位置,步长(学习率)
    2.计算该点当前的梯度的负方向
    3.向该负梯度方向移动步长
    4.重复2-3步,直至结束
```

| 损失函数         | 公式               | 特点                          |
| ------------ | ---------------- | --------------------------- |
| 最小二乘         | Σ(yi - ŷi)²      | 每个样本点误差平方和，最基础              |
| MSE (均方误差)   | Σ(yi - ŷi)² / n  | 最小二乘的的平方和 / 样本个数，最常用        |
| RMSE (均方根误差) | √MSE             | 均方误差开平方根,与原始数据同量纲，更直观       |
| MAE (均绝对误差)  | Σ\|yi - ŷi\| / n | 每个样本点误差的绝对值和 / 样本个数,对异常值不敏感 |

### 进阶用法与原理

> [!note] 💡 AI 扩展（基础）
> 实际评估中，MAE 和 RMSE 是最常用的两个指标。MAE 反映"真实"的平均误差大小，不受个别大误差的过度影响；RMSE 会将大误差的数据点的影响放大（因为是先平方再开根），对异常数据更敏感。当业务中需要特别关注大误差（如金融风控），优先看 RMSE；当需要稳定评估整体偏差，优先看 MAE。

### 避坑与局限
- MSE 对异常值非常敏感（平方放大），如果数据中有极端异常值，考虑使用 MAE 或 Huber Loss
- 单一指标不足以全面评估模型，建议 MAE + RMSE 组合使用

## 回归模型的评估方法 🔸 核心

### 定义与本质
为什么要进行线性回归模型的评估？我们希望衡量预测值和真实值之间的差距，会用到 **MAE、MSE、RMSE** 多种测评函数进行评价。

### 基础用法

**1. 平均绝对误差 Mean Absolute Error（MAE）**

$$MAE = \frac{1}{n}\sum_{i=1}^{n}|y_i - \hat{y}_i|$$

其中 n 为样本数量，y 为实际值，ŷ 为预测值。MAE 越小，模型预测越准确。

**2. 均方误差 Mean Squared Error（MSE）**

$$MSE = \frac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2$$

MSE 越小，模型预测越准确。

**3. 均方根误差 Root Mean Squared Error（RMSE）**

$$RMSE = \sqrt{MSE}$$

RMSE 是 MSE 的平方根，某些情况下比 MSE 更有用。RMSE 越小，模型预测越准确。

### 避坑与局限
- 结论：一般使用 MAE 和 RMSE 这两个指标
- MAE 反应的是"真实"的平均误差，RMSE 会将误差大的数据点放大
- MAE 不能体现出误差大的数据点，RMSE 放大误差大的数据点对指标的影响，但对异常数据比较敏感

## 梯度下降算法 🔺 难点

### 定义与本质
梯度下降是求解损失函数最小值的迭代优化方法。==梯度的方向是函数值上升最快的方向==，因此沿梯度的负方向移动可以找到函数最小值。
梯度：单变量函数就是导数；多变量函数就是偏导数
单变量函数中，梯度就是某一点切线斜率（某一点的导数）；有方向为函数增长最快的方向
多变量函数中，梯度就是某一个点的偏导数；有方向：偏导数分量的向量方向

### 基础用法

**梯度下降公式**：

$$w = w - \alpha \cdot \nabla Loss(w)$$
(w_1, w_2, ... w_n) = (w_1, w_2, ... w_n) - a * (梯度1, 梯度2... 梯度n)
- α（学习率/步长）：控制每次更新的幅度，通常取 0.001~0.01
- 梯度：单变量为导数，多变量为偏导数

![[image_049.png]]

```text
梯度下降流程：
1. 给定初始权重位置和学习率α
2. 计算当前位置的梯度（偏导数）
3. 沿梯度的负方向移动α步长
4. 检查停止条件（满足任一即停）：
   - 达到最大迭代次数 max_iter
   - 损失变化 < tol 阈值
   - 梯度范数 ≈ 0
   - 验证集连续多轮不提升（早停）
5. 重复步骤 2-4
```

### 完整数值演算

> 本节用 **2 个样本、2 轮迭代** 的具体数值，完整展示"预测 → 算损失 → 求梯度 → 更新参数"的闭环过程。

#### 已知条件
$\theta_0$截距   $\theta_1$斜率
**模型**：$\hat{y} = \theta_0 + \theta_1 x$

**数据集**（m = 2）：
- 样本 1：$x^{(1)} = 1,\ y^{(1)} = 3$
- 样本 2：$x^{(2)} = 2,\ y^{(2)} = 5$

**超参数**：$\theta_0 = 0,\ \theta_1 = 0,\ \alpha = 0.1$

**损失函数**：$J = \frac{1}{2m}\sum(\hat{y} - y)^2$

**梯度**：$\frac{\partial J}{\partial \theta_0} = \frac{1}{m}\sum(\hat{y} - y)$，$\frac{\partial J}{\partial \theta_1} = \frac{1}{m}\sum(\hat{y} - y)x$

**更新**：$\theta_j := \theta_j - \alpha \cdot \frac{\partial J}{\partial \theta_j}$

---

#### 状态追踪表

| 变量                                     | 初始   | 第 1 轮后   | 第 2 轮后     | 变化解读          |
| -------------------------------------- | ---- | -------- | ---------- | ------------- |
| $\theta_0$                             | 0    | **0.4**  | **0.6625** | 截距向最优值 1 逼近   |
| $\theta_1$                             | 0    | **0.65** | **1.0775** | 斜率向最优值 2 逼近   |
| $\hat{y}^{(1)}$                        | 0    | 1.05     | —          | 预测值逐渐接近真实值 3  |
| $\hat{y}^{(2)}$                        | 0    | 1.70     | —          | 预测值逐渐接近真实值 5  |
| 误差⁽¹⁾                                  | -3   | -1.95    | —          | 误差绝对值减小，预测仍偏低 |
| 误差⁽²⁾                                  | -5   | -3.30    | —          | 误差绝对值减小，预测仍偏低 |
| $\frac{\partial J}{\partial \theta_0}$ | -4   | -2.625   | —          | 梯度绝对值减小，坡度变缓  |
| $\frac{\partial J}{\partial \theta_1}$ | -6.5 | -4.275   | —          | 梯度绝对值减小，坡度变缓  |
| 损失 $J$                                 | 8.5  | 2.72     | 1.10       | 损失持续下降，收敛趋势明显 |

> [!note] 观察规律
> - 每轮参数都向最优解 $(\theta_0=1, \theta_1=2)$ 靠近
> - 梯度绝对值越来越小，说明坡度在变缓
> - 损失 $J$ 从 8.5 → 2.72 → 1.10，下降速度逐渐减慢

---

#### 收敛趋势

| 轮次 | $\theta_0$ | $\theta_1$ | 损失 $J$ |
|------|-----------|-----------|---------|
| 初始 | 0 | 0 | 8.5 |
| 第 1 轮 | 0.4 | 0.65 | 2.72 |
| 第 2 轮 | 0.6625 | 1.0775 | 1.10 |
| **最优解** | **1** | **2** | **0** |

> [!tip] 收敛判断：若继续迭代，损失将无限接近 0，参数无限接近 $(1, 2)$。实际训练中当损失变化 < tol 或达到 max_iter 时停止。


### 三者关联推导

> 从理论层面建立"线性回归 → MSE 损失 → 梯度下降"的三层绑定关系。

```mermaid
flowchart LR
    A["输入 x<br>参数 θ"] -->|"线性回归<br>ŷ = θ₀ + θ₁x"| B["预测值 ŷ"]
    B -->|"计算误差<br>(ŷ - y)"| C["误差项"]
    C -->|"MSE损失<br>J = ¹⁄₂ₘΣ(ŷ-y)²"| D["损失值 J"]
    D -->|"求偏导<br>∂J/∂θ"| E["梯度 ∇J"]
    E -->|"梯度下降<br>θ := θ - α·∇J"| F["更新后的 θ"]
    F -->|"循环"| A
```

#### 公式链总览

| 阶段 | 输入 | 核心公式 | 输出 | 作用 |
|------|------|---------|------|------|
| **预测层** | $x, \theta$ | $\hat{y} = \theta_0 + \theta_1 x$ | 预测值 $\hat{y}$ | 根据当前参数计算预测结果 |
| **评价层** | $\hat{y}, y$ | $J = \frac{1}{2m}\sum(\hat{y} - y)^2$ | 损失值 $J$ | 衡量预测与真实的差距 |
| **优化层** | $J, \alpha$ | $\theta := \theta - \alpha \cdot \nabla J$ | 更新后的 $\theta$ | 沿负梯度方向微调参数 |

---
#### 第 1 步：预测层（线性回归）
**功能**：给定参数 $\theta$ 和特征 $x$，输出预测值 $\hat{y}$。
$\hat{y}^{(i)} = \theta_0 + \theta_1 x^{(i)}$
- $\hat{y}$ 对 $\theta$ 的依赖是**线性的**
- 参数 $\theta$ 的好坏无法直接判断，需要进入评价层打分
#### 第 2 步：评价层（MSE 损失）
**功能**：将预测值 $\hat{y}$ 与真实标签 $y$ 对比，构造关于 $\theta$ 的目标函数 $J(\theta)$。
$J(\theta_0, \theta_1) = \frac{1}{2m}\sum_{i=1}^m \big(\hat{y}^{(i)} - y^{(i)}\big)^2$
**关键设计**：
- 将线性回归的 $\hat{y}^{(i)} = \theta_0 + \theta_1 x^{(i)}$ 代入，$J$ 就变为**仅关于 $\theta$ 的函数**
- $\frac{1}{2}$ 是计算技巧：求导后刚好消去平方项的系数 2
- 目标：找到使 $J(\theta)$ 最小的 $\theta$
#### 第 3 步：优化层（梯度下降）
**功能**：对 $J(\theta)$ 求偏导得到梯度，迭代更新 $\theta$。
**梯度（对损失函数求偏导）**：
$\frac{\partial J}{\partial \theta_0} = \frac{1}{m}\sum_{i=1}^m \big(\hat{y}^{(i)} - y^{(i)}\big)$
$\frac{\partial J}{\partial \theta_1} = \frac{1}{m}\sum_{i=1}^m \big(\hat{y}^{(i)} - y^{(i)}\big) x^{(i)}$
**参数更新**：
$\theta_0 := \theta_0 - \alpha \cdot \frac{\partial J}{\partial \theta_0}$
$\theta_1 := \theta_1 - \alpha \cdot \frac{\partial J}{\partial \theta_1}$
> ⚠️ **同步更新**：必须先算完两个梯度，再同时更新 $\theta_0$ 和 $\theta_1$。

---
#### 三层绑定关系

| 层级 | 角色 | 公式中的体现 |
|------|------|-------------|
| **线性回归** | 预测工具 | $\theta_0 + \theta_1 x^{(i)}$ = 预测值 $\hat{y}$ |
| **MSE 损失** | 评价尺子 | $(\hat{y} - y)$ = 误差项，构造目标函数 |
| **梯度下降** | 优化算法 | $\theta = \theta - \alpha \cdot$ 梯度 = 参数更新 |

> [!note] 一句话总结：**梯度下降是用来最小化线性回归 MSE 损失函数的数值求解算法**。


### 避坑与局限
- 学习率太小学得太慢，太大会导致震荡甚至发散（错过最优解）
- 局部最优解问题：损失函数非凸时可能陷入局部最小值

### 停止条件与 sklearn 参数对应

| 停止条件 | 含义 | sklearn 参数 |
|---------|------|-------------|
| 最大迭代次数 | 达到指定轮数强制停止 | `max_iter` |
| 损失变化阈值 | 提升小于 tol 认为收敛 | `tol` |
| 梯度接近 0 | 参数更新幅度趋于 0 | 内部判断 |
| 验证集早停 | 验证集连续不提升则停 | `early_stopping` + `n_iter_no_change` |

```python
from sklearn.linear_model import SGDRegressor

estimator = SGDRegressor(
    max_iter=1000,      # 最多训练 1000 轮
    tol=1e-3,           # 提升 < 0.001 认为收敛
    early_stopping=True,
    n_iter_no_change=5  # 连续 5 轮验证集不提升则停
)
```

> [!warning] 数据集分工
> - **训练集**：训练参数
> - **验证集**：判断停止时机（早停看的是验证集，不是测试集）
> - **测试集**：最终评估（不参与训练/早停决策）

> [!tip] 记忆口诀
> 次数到了要停，损失不降要停，梯度变小要停，验证变差要停。

## 过拟合与正则化 🔺 难点

### 定义与本质

| 问题 | 表现 | 原因 | 解决方法 |
|------|------|------|---------|
| 欠拟合 | 训练集差 + 测试集差 | 模型太简单 | 添加特征、添加多项式特征 |
| 过拟合 | 训练集好 + 测试集差 | 模型太复杂、数据太少 | 清洗数据、增大数据量、正则化、降维 |

**欠拟合解决方法详解**：
1. **添加其他特征项**：有时出现欠拟合是因为特征项不够导致的，可以添加其他特征项来解决
2. **添加多项式特征**：模型过于简单时的常用套路，例如将线性模型通过添加二次项或三次项使模型泛化能力更强

**过拟合解决方法详解**：
1. **重新清洗数据集**：导致过拟合的一个原因有可能是数据不纯，如果出现了过拟合就需要重新清洗数据
2. **增大数据的训练量**：还有一个原因就是我们用于训练的数据量太小导致的，训练数据占总数据的比例过小
3. **正则化概念（出现的原因）**：在模型训练时，数据中有些特征影响模型复杂度、或者某个特征的异常值较多，所以要尽量减少这个特征的影响（甚至删除某个特征的影响），这就是正则化。为了减少过拟合的影响，控制模型的参数，尤其是高次项的权重参数。

**正则化**：在损失函数中增加惩罚项，限制模型复杂度
4. **减少特征维度**：去除不重要的特征，降低模型复杂度

### 基础用法

**正则化概念（出现的原因）**：在模型训练时，数据中有些特征影响模型复杂度、或者某个特征的异常值较多，所以要尽量减少这个特征的影响（甚至删除某个特征的影响），这就是正则化。为了减少过拟合的影响，控制模型的参数，尤其是高次项的权重参数。

**正则化**：在损失函数中增加惩罚项，限制模型复杂度

| 正则化 | 公式 | 效果 | 对应算法 |
|--------|------|------|---------|
| L1正则化 | Loss + λΣ\|wi\| | 将高次项系数压缩为0（特征选择） | Lasso回归 |
| L2正则化 | Loss + λΣwi² | 将高次项系数压缩到很小 | 岭回归（Ridge） |

```python
from sklearn.linear_model import Lasso, Ridge
# L1正则化（Lasso）- 可将部分特征权重压缩为0
lasso = Lasso(alpha=0.1)

# L2正则化（Ridge）- 将权重压缩到很小的值
ridge = Ridge(alpha=0.1)
ridge.fit(X_train, y_train)
ridge.predict(X_train)
```

**L2正则化优点**：越小的参数说明模型越简单，越简单的模型则越不容易产生过拟合现象。

### 进阶用法与原理

> [!note] 💡 AI 扩展（进阶）
> L1正则化的几何直觉：L1约束形成一个菱形可行域，最优解更容易落在菱形的顶点上（某些维度为0），因此具有==特征选择==的天然能力。L2正则化的约束形成一个圆形可行域，最优解落在圆上但各维度一般不为0。实践中，如果不确定用哪个，可使用 ElasticNet（L1+L2混合），sklearn 中为 `ElasticNet` 类，通过 `l1_ratio` 参数控制 L1 和 L2 的比例。

### 避坑与局限
- 正则化系数 λ（alpha）的选择需要调优，太大导致欠拟合，太小则正则化效果不足
- L1正则化在特征高度相关时可能随机选择一个特征而丢弃其他相关特征

### 过拟合和欠拟合可视化

```python
import numpy as np
import matplotlib.pyplot as plt
from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_squared_error

def demo_underfitting():
    # 1. 准备数据（添加噪声）
    np.random.seed(666)
    x = np.random.uniform(-3, 3, size=100)
    # y = 0.5x² + x + 2 + 噪声（真实关系是二次的）
    y = 0.5 * x ** 2 + x + 2 + np.random.normal(0, 1, size=100)

    # 2. 用线性模型拟合二次数据（必然欠拟合）
    model = LinearRegression()
    X = x.reshape(-1, 1)
    model.fit(X, y)

    # 3. 预测
    y_predict = model.predict(X)
    print(f'均方误差: {mean_squared_error(y, y_predict):.4f}')

    # 4. 可视化
    plt.scatter(x, y, label='真实数据')         # 散点图
    plt.plot(x, y_predict, color='r', label='线性拟合（欠拟合）')
    plt.legend()
    plt.title('欠拟合示例：用线性模型拟合二次数据')
    plt.show()

if __name__ == '__main__':
    demo_underfitting()
```

## 4️⃣ 避坑指南 & 易错对比

> [!warning] 易混概念对比
> | 概念A | 概念B | 区分要点 |
> |-------|-------|----------|
> | MSE | RMSE | RMSE = √MSE，与原始数据同量纲更直观 |
> | MSE | MAE | MSE对异常值敏感（先平方），MAE更鲁棒 |
> | 正规方程 | 梯度下降 | 正规方程解析解（快但不可扩展），梯度下降迭代解（可扩展） |
> | L1正则化 | L2正则化 | L1压缩至0（特征选择），L2压缩至很小（权重衰减） |
> | 权重w | 偏置b | w是特征前的系数，b是截距项（独立于特征） |

> [!danger] 常见错误与规避
> - **错误**：学习率设太大（如0.5）导致梯度震荡 → **正确**：通常从 0.001~0.01 开始尝试
> - **错误**：未做标准化直接用SGD → **正确**：SGD对特征尺度敏感，必须先标准化
