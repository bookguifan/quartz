## 📌 核心速览

- **计算机五大核心部件**：运算器、控制器、存储器、输入设备、输出设备，共同构成冯·诺依曼体系结构。CPU = 运算器 + 控制器，是计算机的"大脑"。

## 1️⃣ 完整知识库

### 计算机组成原理 🔹 基础

> 理解计算机如何工作是学习编程的第一步。程序本质上是交给 CPU 执行的一系列指令，而数据在运行时必须 resides 在内存中。

#### 定义与本质

**计算机五大核心部件**

| 部件 | 英文 | 功能 | 类比 |
| :--- | :--- | :--- | :--- |
| **运算器** | ALU | 执行算术运算（加减乘除）和逻辑运算（与或非） | 计算器 |
| **控制器** | CU | 指挥协调其他部件工作，按指令顺序执行 | 交通指挥员 |
| **存储器** | Memory | 存储程序和数据 | 办公桌（内存）/ 文件柜（硬盘） |
| **输入设备** | Input | 将外部信息传入计算机 | 眼睛、耳朵 |
| **输出设备** | Output | 将结果呈现给用户 | 嘴巴、手 |

**计算机存储单元**

- **位（bit）**：计算机中最小的存储单位，表示一个二进制数位（0 或 1）。
- **字节（Byte，简写 B）**：计算机基本的存储单元，`1 Byte = 8 bit`。
- 常见单位换算：`1 KB = 1024 B`，`1 MB = 1024 KB`，`1 GB = 1024 MB`。

**CPU（中央处理器）**

- **组成**：运算器 + 控制器 + 寄存器（临时高速存储）。
- **核心指标**：主频（GHz，每秒时钟周期数）、核心数、缓存大小。
- **工作原理**：取指令 → 译码 → 执行 → 访存 → 写回，周而复始。

**GPU（图形处理器）与 CPU 对比**

| 特性 | CPU | GPU |
| :--- | :--- | :--- |
| **计算方式** | 串行计算，擅长复杂运算 | 并行计算，擅长大量简单运算 |
| **核心数** | 较少（通常 4~64 核） | 极多（通常数百至数千核） |
| **适用场景** | 通用计算、逻辑判断、复杂任务调度 | 图形渲染、矩阵运算、AI 模型训练与推理 |
| **架构特点** | 单核性能强，缓存大 | 单核性能弱，但并行吞吐量极高 |

> 💡 **AI 场景**：训练大型 AI 模型时，模型的全部参数需要加载到显存（GPU 专用内存）中，利用 GPU 的并行能力进行高速矩阵运算。

**内存（RAM）与硬盘（Disk）**

| 特性 | 内存（RAM） | 硬盘（HDD/SSD） |
| :--- | :--- | :--- |
| **速度** | 纳秒级（快） | 毫秒级（慢，SSD 比 HDD 快 10~100 倍） |
| **持久性** | 断电数据丢失（易失性） | 断电数据保留（非易失性） |
| **容量** | 通常 8~64 GB | 通常 256 GB ~ 数 TB |
| **价格** | 贵 | 便宜 |
| **作用** | 程序运行时的临时工作区 | 长期存储文件、程序、数据 |

**显存（VRAM）**

显存是显卡的专用内存。在 AI 场景中，如果想把一个模型跑起来，模型的全部参数（即模型的"知识"）必须完整地放进显存里。显存容量直接决定了能运行多大的模型，是 GPU 除算力外的另一核心指标。

> [!note] 💡 AI 扩展（基础）为什么程序必须先加载到内存？因为 CPU 只能直接访问内存。硬盘上的程序文件（如 `.exe`、`.py`）是"静态"的，操作系统需要将其读取到内存中变成"进程"，CPU 才能执行。这也是内存不足时程序会卡顿的原因。

#### 避坑与局限

- **内存不够 ≠ 硬盘空间不够**：程序报错"内存不足"是指 RAM 不够，和硬盘剩余空间无关。
- **程序未保存就断电**：如果数据只存在于内存（如正在编辑的文档未保存），断电后数据会丢失。

### Python 语言基础 🔸 核心

> Python 的设计哲学是"优雅""明确""简单"。其语法接近自然语言，是编程入门的最佳选择。

#### 定义与本质

**Python 的核心特点**

- **解释型**：无需编译，源代码直接由解释器逐行执行，开发效率高。
- **面向对象**：一切皆对象，支持封装、继承、多态。
- **动态类型**：变量类型在运行时确定，无需显式声明（如 `int a = 10`）。
- **丰富的标准库**：内置大量实用模块，"自带电池"。

**Python 解释器**

- 官方解释器是 **CPython**（C 语言实现），也是最常用的版本。
- 其他实现：PyPy（性能优化）、Jython（运行在 JVM 上）、IronPython（运行在 .NET 上）。

#### 基础用法

**注释**

```python
# 这是单行注释

"""
这是多行注释（文档字符串）
常用于函数或模块的说明文档
"""
```

**Python 的三种开发方式**

| 方式 | 说明 | 适用场景 |
| :--- | :--- | :--- |
| **命令行开发** | 在 Python 交互式解释器中逐行输入执行 | 快速验证代码、学习语法、调试小段逻辑 |
| **文件开发** | 将代码写入 `.py` 文件，通过 `python 文件名.py` 执行 | 编写独立脚本、小型工具 |
| **项目开发** | 使用 IDE（如 PyCharm、VS Code）管理多文件工程 | 中大型应用、团队协作、复杂工程 |

**变量与数据类型**

```python
# Python 中变量是"标签"，指向内存中的对象
name = "Alice"      # 字符串 str
age = 25            # 整数 int
height = 1.68       # 浮点数 float
is_student = True   # 布尔 bool

# 查看类型
print(type(name))   # <class 'str'>
print(type(age))    # <class 'int'>

# 同时赋值多个变量
a, b, c = 1, 2, 3
```

**变量命名规范**

变量命名应做到"见名知义"，Python 社区推荐使用以下三种命名法：

| 命名法 | 格式 | 示例 | 使用场景 |
| :--- | :--- | :--- | :--- |
| **大驼峰命名法** | 每个单词首字母大写 | `MyName` | 类名（Class） |
| **小驼峰命名法** | 首单词小写，后续单词首字母大写 | `myName` | 部分其他语言常用 |
| **下划线命名法** | 单词间用下划线连接 | `my_name` | **Python 推荐**：变量名、函数名 |

> ✅ **Python 官方建议**：变量和函数使用下划线命名法（`snake_case`），类名使用大驼峰命名法（`PascalCase`）。

**`input()` 用户输入**

```python
# input() 返回的永远是字符串！
name = input("请输入你的名字：")
print(f"你好，{name}！")

# 如果需要数字，必须手动转换
age = int(input("请输入你的年龄："))  # 转换为整数
price = float(input("请输入价格："))  # 转换为浮点数
```

> 📝 **注意**：`input()` 会中止当前代码的继续执行，处于等待状态，直到用户输入完成并按下回车。如果用户没有输入任何内容，程序将一直停留在该处。

> ⚠️ **避坑**：忘记类型转换是新手最常见错误。如果写 `age = input("年龄：")`，然后 `print(age + 1)`，会报错——因为字符串不能和整数相加。

**`print()` 输出**

```python
# 基础输出
print("Hello, World!")

# 输出多个值，默认用空格分隔
print("姓名：", "Alice", "年龄：", 25)

# 自定义分隔符和结束符
print("A", "B", "C", sep="-", end="!")  # 输出：A-B-C!
```

**格式化输出（三种方式）**

```python
name = "Alice"
age = 25

# 方式1：% 格式化（老旧，不推荐）
print("姓名：%s，年龄：%d" % (name, age))

# 方式2：str.format()（较新，但已过时）
print("姓名：{}，年龄：{}".format(name, age))
print("姓名：{0}，年龄：{1}".format(name, age))

# 方式3：f-string（推荐，Python 3.6+）
print(f"姓名：{name}，年龄：{age}")
print(f"明年年龄：{age + 1}")  # 支持表达式
print(f"圆周率：{3.14159:.2f}")  # 保留两位小数
```

**字符串转义符**

转义符以反斜杠 `\` 开头，用于表示那些无法直接输入或具有特殊含义的字符。

| 转义符 | 名称 | 说明 | 示例 |
| :--- | :--- | :--- | :--- |
| `\t` | 制表符（Tab） | 水平制表位，通常等于 4 个空格的距离 | `print("A\tB")` → `A    B` |
| `\n` | 换行符 | 将光标移到下一行开头 | `print("第一行\n第二行")` → 分两行显示 |
| `\\` | 反斜杠 | 表示一个普通的反斜杠字符 | `print("C:\\Users")` → `C:\Users` |
| `\'` | 单引号 | 在单引号字符串中表示单引号 | `'It\'s OK'` |
| `\"` | 双引号 | 在双引号字符串中表示双引号 | `"他说：\"你好\""` |

```python
# 转义符示例
print("姓名：\tAlice")
print("年龄：\t25")
print("第一行\n第二行")

# 原始字符串（r前缀）：忽略转义
path = r"C:\Users\Alice\Documents"
print(path)  # C:\Users\Alice\Documents
```

> 💡 **注意**：Windows 文件路径中大量使用反斜杠，为避免与转义符冲突，可使用原始字符串 `r"..."` 或在路径中使用正斜杠 `/`。

#### 进阶用法与原理

**变量赋值的内存本质**

```python
a = 10
b = a
print(id(a))  # 查看内存地址，如 140735893619280
print(id(b))  # 和 a 相同！说明 b 也指向同一个对象

a = 20        # a 指向新的对象 20
print(id(a))  # 地址变了
print(id(b))  # b 仍然指向 10
```

- Python 中变量保存的是对象的**引用（内存地址）**，而非对象本身。
- 小整数（-5 ~ 256）会被缓存复用，`a = 10` 和 `b = 10` 可能指向同一个对象。

> [!note] 💡 AI 扩展（进阶）
> ** intern 机制与缓存**：
> - 小整数缓存：CPython 启动时预分配 -5~256 的整数对象，所有引用这些值的变量共享同一对象。
> - 字符串驻留：部分短字符串（如标识符）也会被缓存，`a = "hello"` 和 `b = "hello"` 可能 `id(a) == id(b)`。
> - 这是 Python 内存优化的重要手段，也是 `is` 和 `==` 可能结果不同的原因：`==` 比较值，`is` 比较内存地址。

#### 避坑与局限

| 坑点 | 说明 | 正确做法 |
| :--- | :--- | :--- |
| **变量未定义就使用** | `print(x)` 但前面没有 `x = ...` | 确保先赋值再使用 |
| **整数除法变浮点** | `5 / 2` 结果是 `2.5`（不是 `2`） | 需要整除用 `//`：`5 // 2 == 2` |
| **input 返回字符串** | `age = input()` 得到的是字符串 | 需要数值时用 `int()` 或 `float()` 转换 |
| **浮点数精度问题** | `0.1 + 0.2 == 0.3` 为 `False` | 用 `round()` 或 `decimal` 模块处理精度敏感场景 |

> **案例2：简单的 BMI 计算器**

```python
height = float(input("请输入身高（米）："))

print(f"您的 BMI 值为：{bmi:.2f}")
if bmi < 18.5:
    print("体重过轻")
elif bmi < 24:
    print("体重正常")
else:
    print("体重过重")
```
