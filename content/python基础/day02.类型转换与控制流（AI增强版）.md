# [day02] 学习笔记｜类型转换、运算符与控制流（AI 增强版）

## 📌 核心速览

- **类型转换**：`int()`、`float()`、`str()`、`bool()` 等内置函数实现类型显式转换；转换失败会抛出 `ValueError` 或 `TypeError`。
- **运算符**：算术运算符（`+`、`-`、`*`、`/`、`//`、`%`、`**`）、赋值运算符（`=`、`+=` 等）、比较运算符（`==`、`!=`、`>` 等）、逻辑运算符（`and`、`or`、`not`）。
- **控制流**：`if-elif-else` 条件判断、`while` 循环、`for` 循环、`break`（跳出循环）、`continue`（跳过本轮）、`return`（函数返回）。
- **短路求值**：`and` 和 `or` 在逻辑运算中具有短路特性，可用于设置默认值和条件执行。
- **可迭代对象**：`for` 循环的本质是遍历可迭代对象（Iterable），如列表、字符串、`range()` 等。

## 1️⃣ 完整知识库

### 类型转换 🔹 基础

> Python 是动态类型语言，但类型转换无处不在——尤其是 `input()` 返回字符串，必须转换后才能进行数学运算。

#### 定义与本质

**显式类型转换（强制转换）**

| 函数 | 作用 | 示例 | 失败场景 |
| :--- | :--- | :--- | :--- |
| `int(x)` | 转换为整数 | `int("10")` → `10` | `int("abc")` → `ValueError` |
| `float(x)` | 转换为浮点数 | `float("3.14")` → `3.14` | `float("")` → `ValueError` |
| `str(x)` | 转换为字符串 | `str(100)` → `"100"` | 几乎不会失败 |
| `bool(x)` | 转换为布尔值 | `bool(0)` → `False` | 几乎不会失败 |
| `list(x)` | 转换为列表 | `list("abc")` → `['a','b','c']` | 参数不可迭代时失败 |
| `tuple(x)` | 转换为元组 | `tuple([1,2,3])` → `(1,2,3)` | 参数不可迭代时失败 |

**隐式类型转换**

```python
# 整数 + 浮点数 → 浮点数（自动提升）
result = 3 + 2.5   # 结果：5.5（int 被隐式转为 float）

# 布尔值在算术运算中视为 0/1
print(True + True)  # 2
print(True * 5)     # 5
```

> **常量命名约定**：Python 没有真正的常量类型，但程序员约定俗成使用**全部大写**的变量名表示常量（多个单词用下划线分割），如 `MAX_SIZE = 100`。这是一种代码规范而非语法约束。

#### 避坑与局限

- **`int()` 截断而非四舍五入**：`int(3.9)` 结果是 `3`，不是 `4`。需要四舍五入用 `round(3.9)`。
- **空字符串转 `bool`**：`bool("")` 为 `False`，但 `bool("0")` 为 `True`（非空字符串都是 `True`）。
- **`float` 转 `int` 丢失精度**：`int(3.14)` 变成 `3`，小数部分直接丢弃。
- **浮点数运算精度问题**：`float` 采用 IEEE 754 表示，某些运算会产生微小误差（如 `0.1 + 0.2` → `0.30000000000000004`）。
  - 需要**可控精度**的十进制运算时，使用 `decimal` 模块：`from decimal import Decimal, getcontext`。可设定有效位数，但仍是近似小数。
  - 需要**数学上绝对精确**的有理数表示时，使用 `fractions.Fraction`：`from fractions import Fraction`。以分子/分母形式存储，无任何舍入误差。

### 运算符 🔸 核心

#### 基础用法

**算术运算符**

| 运算符 | 名称 | 示例 | 结果 |
| :--- | :--- | :--- | :--- |
| `+` | 加 | `5 + 3` | `8` |
| `-` | 减 | `5 - 3` | `2` |
| `*` | 乘 | `5 * 3` | `15` |
| `/` | 除（浮点） | `5 / 2` | `2.5` |
| `//` | 整除（地板除） | `5 // 2` | `2` |
| `%` | 取余（模） | `5 % 2` | `1` |
| `**` | 幂运算 | `2 ** 3` | `8` |

> ⚠️ **重点区分**：`/` 永远返回浮点数，即使能整除（`4 / 2` → `2.0`）；`//` 返回整数或浮点数（取决于操作数类型）。

**赋值运算符**

```python
a = 10      # 基础赋值
a += 5      # 等价于 a = a + 5，结果 15
a -= 3      # 等价于 a = a - 3，结果 12
a *= 2      # 等价于 a = a * 2，结果 24
a //= 4     # 等价于 a = a // 4，结果 6
```

**比较运算符**

| 运算符 | 含义 | 示例 |
| :--- | :--- | :--- |
| `==` | 等于 | `5 == 5` → `True` |
| `!=` | 不等于 | `5 != 3` → `True` |
| `>` | 大于 | `5 > 3` → `True` |
| `<` | 小于 | `5 < 3` → `False` |
| `>=` | 大于等于 | `5 >= 5` → `True` |
| `<=` | 小于等于 | `5 <= 3` → `False` |

> ⚠️ **避坑**：`==` 是比较运算符（判断相等），`=` 是赋值运算符。`if a = 5:` 会报错！
> ⚠️ **避坑**：**字符串和整数不能直接比较**。Python 3 中 `'5' > 10` 会抛出 `TypeError: '>' not supported between instances of 'str' and 'int'`。比较前务必先进行类型转换：`int('5') > 10`。

**成员运算符**

| 运算符 | 含义 | 示例 | 结果 |
| :--- | :--- | :--- | :--- |
| `in` | 判断元素是否在序列/集合中 | `3 in [1, 2, 3]` | `True` |
| `not in` | 判断元素是否不在序列/集合中 | `4 not in [1, 2, 3]` | `True` |

```python
# 成员判断示例
fruits = ["apple", "banana", "cherry"]
print("apple" in fruits)      # True
print("grape" not in fruits)  # True

# 字符串子串判断
print("Py" in "Python")       # True

# 字典中判断键
person = {"name": "Alice", "age": 25}
print("name" in person)       # True
```

**逻辑运算符**

| 运算符 | 含义 | 短路规则 |
| :--- | :--- | :--- |
| `and` | 与（两边都为真才为真） | 左边为假，直接返回左边（不评估右边） |
| `or` | 或（一边为真即为真） | 左边为真，直接返回左边（不评估右边） |
| `not` | 非（取反） | 无短路 |

```python
# 短路求值示例
x = 0 and 10    # 0（左边为假，直接返回 0，不评估 10）
y = 5 or 10     # 5（左边为真，直接返回 5，不评估 10）

# 实用技巧：设置默认值
name = user_input or "匿名"   # 如果 user_input 为空字符串（假值），则使用默认值
```

#### 进阶用法与原理

**运算符优先级**（从高到低）

1. `()` 括号（最高优先级，可改变顺序）
2. `**` 幂运算
3. `*`、`/`、`//`、`%`
4. `+`、`-`
5. `>`、`<`、`>=`、`<=`、`==`、`!=`
6. `not`
7. `and`
8. `or`（最低优先级）

```python
# 例子
result = 2 + 3 * 4       # 14，不是 20（先乘后加）
result = (2 + 3) * 4     # 20，括号改变优先级
result = not True or False  # False（not 优先级高于 or：False or False）
```

> [!note] 💡 AI 扩展（进阶）
> **布尔值的底层本质**：
> Python 中以下值被视为"假值"（Falsy）：`False`、`0`、`0.0`、`""`（空字符串）、`[]`（空列表）、`{}`（空字典）、`None`、`()`（空元组）、`set()`（空集合）。其余皆为"真值"（Truthy）。
> `and` 和 `or` 不一定会返回 `True`/`False`，而是返回**最后一个被评估的操作数**：
> ```python
> print(3 and 5)   # 5（两边都真，返回最后一个）
> print(0 and 5)   # 0（左边假，直接返回）
> print(3 or 5)    # 3（左边真，直接返回）
> print(0 or 5)    # 5（左边假，返回右边）
> ```

### 控制流 🔸 核心

#### 基础用法

**`if-elif-else` 条件判断**

```python
score = int(input("请输入成绩："))

if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
elif score >= 70:
    grade = "C"
elif score >= 60:
    grade = "D"
else:
    grade = "F"

print(f"等级：{grade}")
```

> **注意**：`elif` 是"else if"的缩写，Python 中不能写 `else if`（必须连写）。条件从上到下依次判断，满足一个后后面的条件不再检查。

**`pass` 语句**

`pass` 是 Python 的空语句占位符，语法上需要语句但逻辑上暂时不需要执行任何操作时使用，不影响程序运行。

```python
if condition:
    pass  # 暂不处理，占位避免语法错误
else:
    print("其他情况")
```

**`while` 循环**

```python
# 计算 1 + 2 + ... + 100
i = 1
sum_val = 0
while i <= 100:
    sum_val += i
    i += 1
print(sum_val)  # 5050
```

**死循环（无限循环）**

> 在编程中，一个靠自身控制**无法终止**的程序称为"死循环"。死循环会无限重复执行同一段代码，导致程序永远无法正常结束。

最常见的死循环形式：

```python
while True:
    print("这段代码会永远执行下去...")
```

⚠️ **CPU 消耗警告**：死循环会**持续占用 CPU 资源**。如果循环体内没有 `time.sleep()` 等暂停操作，CPU 使用率会飙升到接近 100%，导致电脑发热、卡顿甚至死机。在编写 `while True` 时，必须确保循环内部有明确的 `break` 终止条件或其他退出机制。

```python
import time

# 安全的无限循环示例（带有退出条件）
while True:
    user_input = input("请输入内容（输入 'exit' 退出）：")
    if user_input == "exit":
        print("程序已退出")
        break
    time.sleep(0.1)  # 短暂休眠，降低 CPU 占用
```

**`for` 循环**

```python
# 遍历列表
for i in [1, 2, 3, 4, 5]:
    print(i)

# 遍历字符串
for char in "Hello":
    print(char)

# range() 生成数字序列
for i in range(5):       # 0, 1, 2, 3, 4
    print(i)

for i in range(1, 10, 2):  # 1, 3, 5, 7, 9（起始，结束，步长）
    print(i)
```

**循环控制关键字**

| 关键字 | 作用 | 使用场景 |
| :--- | :--- | :--- |
| `break` | 立即终止整个循环 | 找到目标后提前退出 |
| `continue` | 跳过本轮剩余代码，进入下一轮 | 跳过某些不符合条件的项 |
| `return` | 从函数中返回，同时终止循环（如果在函数内） | 函数返回值并结束 |

```python
# break 示例：找到第一个能被 7 和 13 整除的数
for i in range(1, 1000):
    if i % 7 == 0 and i % 13 == 0:
        print(f"找到：{i}")  # 91
        break

# continue 示例：打印 1~10 中的奇数
for i in range(1, 11):
    if i % 2 == 0:
        continue
    print(i)  # 1, 3, 5, 7, 9
```

#### 进阶用法与原理

**`for-else` 结构**

```python
# 当循环正常结束（没有被 break）时，执行 else 块
for i in range(2, 10):
    if i % 5 == 0:
        print(f"找到能被5整除的数：{i}")
        break
else:
    print("没有找到能被5整除的数")  # 不会执行，因为 5 会被找到并 break
```

> [!note] 💡 AI 扩展（进阶）
> **`while-else` 同理**，但使用频率较低。`for-else` 最常见的场景是**搜索**：判断一个列表中是否包含某个元素，如果不包含则执行特定逻辑。
> ```python
> # 判断质数
> n = 17
> for i in range(2, int(n**0.5) + 1):
>     if n % i == 0:
>         print(f"{n} 不是质数")
>         break
> else:
>     print(f"{n} 是质数")
> ```

#### 避坑与局限

| 坑点 | 说明 | 正确做法 |
| :--- | :--- | :--- |
| **死循环** | `while True:` 没有终止条件 | 确保循环条件最终会变为 `False` |
| `range()` 左闭右开 | `range(5)` 是 0~4，不包含 5 | 需要包含末尾时加 1 |
| **缩进错误** | Python 用缩进表示代码块 | 统一使用 4 个空格 |
| **`break` 只跳出一层循环** | 嵌套循环中 `break` 只退出当前层 | 需要全跳出时设置标志变量或使用函数返回 |

## 4️⃣ 避坑指南 & 易错对比

**概念易混对比**

| 对比组 | 区分要点 |
| :--- | :--- |
| `=` vs `==` | `=` 是赋值，`==` 是判断是否相等 |
| `/` vs `//` | `/` 是浮点除法，`//` 是整除（地板除） |
| `and` vs `&` | `and` 是逻辑与（用于布尔判断），`&` 是按位与（用于二进制位运算） |
| `break` vs `continue` | `break` 终止整个循环，`continue` 只跳过本轮 |
| `is` vs `==` | `is` 判断内存地址是否相同，`==` 判断值是否相等 |

**常见错误与规避**

1. **修改正在遍历的列表**
   ```python
   # 错误！遍历中删除元素会导致跳过某些项
   nums = [1, 2, 3, 4, 5]
   for n in nums:
       if n % 2 == 0:
           nums.remove(n)  # 危险！
   ```
   - **正确做法**：遍历副本 `for n in nums[:]:`，或用列表推导式重建列表。

2. **`range()` 不可变**
   ```python
   # 错误！
   for i in range(5):
       i += 10  # 这只修改了局部变量 i，不影响循环
   ```

> **案例3：质数判断（平方根优化）**

```python
def is_prime(n):
    """判断一个数是否为质数"""
    if n <= 1:
        return False
    if n == 2:
        return True
    if n % 2 == 0:
        return False

    # 核心优化：只需检查到平方根即可
    # 原理：如果 n 有大于其平方根的因数，那它一定对应有一个小于平方根的因数
    # 因数总是成对出现，例如 12 的因数：1×12, 2×6, 3×4
    for i in range(3, int(n ** 0.5) + 1, 2):
        if n % i == 0:
            return False
    return True

# 测试
num = int(input("请输入一个大于1的整数："))
if is_prime(num):
    print(f"{num} 是质数")
else:
    print(f"{num} 不是质数")
```

> 💡 **平方根优化原理**：假设 `n = a × b`，如果 `a` 和 `b` 都大于 `√n`，那么 `a × b > n`，矛盾。因此至少有一个因数小于或等于 `√n`。只需遍历到 `int(√n) + 1` 即可，时间复杂度从 O(n) 降为 O(√n)。

> 使用 `for-else` 结构的简洁写法：
> ```python
> n = int(input("请输入一个大于1的整数："))
> if n <= 1:
>     print("请输入大于1的整数")
> else:
>     for i in range(2, int(n ** 0.5) + 1):
>         if n % i == 0:
>             print(f"{n} 不是质数")
>             break
>     else:
>         print(f"{n} 是质数")
> ```
