## 📌 核心速览

- **虚拟机（Virtual Machine）**：通过软件模拟的完整计算机系统，在物理机上运行多个独立的操作系统实例。常用工具：VMware、VirtualBox。
- **Linux**：开源的类 Unix 操作系统，是服务器领域的绝对主流。发行版：Ubuntu（新手友好）、CentOS（企业服务器）、Debian（稳定）。
- **文件系统**：Linux 一切皆文件，采用层级目录结构。重要目录：`/`（根目录）、`/home`（用户目录）、`/etc`（配置文件）、`/var`（日志文件）、`/usr`（系统软件）。
- **基础命令**：`ls`、`cd`、`pwd`、`mkdir`、`rm`、`cp`、`mv`、`cat`、`grep`、`chmod`、`ps`、`top`、`ssh`。
- **权限管理**：三类权限（读 `r`、写 `w`、执行 `x`）× 三类用户（所有者、所属组、其他用户）。

## 1️⃣ 完整知识库

### 虚拟机 🔹 基础

#### 定义与本质

- **宿主机（Host）**：运行虚拟化软件的物理计算机。
- **客户机（Guest）**：虚拟机中运行的操作系统。
- **Hypervisor**：虚拟化层，分为 Type 1（直接运行在硬件上，如 VMware ESXi）和 Type 2（运行在宿主操作系统上，如 VMware Workstation、VirtualBox）。

**为什么使用虚拟机？**

| 场景 | 说明 |
| :--- | :--- |
| 学习 Linux | 在 Windows 上安全地体验 Linux，不影响主系统 |
| 环境隔离 | 不同项目需要不同操作系统或软件版本 |
| 服务器模拟 | 本地搭建多节点集群测试环境 |
| 安全测试 | 在隔离环境中运行可疑程序 |

**虚拟机网络模式**

| 模式 | 说明 | 特点 |
| :--- | :--- | :--- |
| **NAT 模式** | 虚拟机将 Windows 电脑当做路由器上网，借助 Windows 电脑上网 | 推荐模式，配置简单，虚拟机可上网但独立于外部网络 |
| **桥接模式** | 虚拟机将自己接入 Windows 电脑所在的网络中，借助同一台路由器上网 | 虚拟机与宿主机在同一局域网，拥有独立 IP |
| **仅主机模式** | 虚拟机无法上网，只能和 Windows 电脑互通 | 完全隔离，适合搭建封闭测试环境 |

### Linux 文件系统 🔹 基础

#### 定义与本质

Linux 采用单一树形目录结构，所有文件和目录都从根目录 `/` 开始。

**重要目录**

| 目录 | 内容 | 类比 Windows |
| :--- | :--- | :--- |
| `/` | 根目录，一切起点 | `C:\` |
| `/home` | 普通用户的家目录 | `C:\Users` |
| `/root` | root 用户的家目录 | `C:\Users\Administrator` |
| `/etc` | 系统配置文件 | `C:\Windows\System32` |
| `/var` | 日志、缓存等可变数据 | — |
| `/usr` | 用户程序和库 | `C:\Program Files` |
| `/tmp` | 临时文件 | `%TEMP%` |
| `/bin` / `/sbin` | 系统命令 | `C:\Windows\System32` |

**Linux 与 Windows 目录结构对比**

| 对比项 | Windows | Linux |
| :--- | :--- | :--- |
| **根目录** | 可以有多个，如 `C:\`、`D:\`、`E:\` | 只有一个 `/`，所有文件都在它下面 |
| **路径分隔符** | 反斜杠 `\` | 正斜杠 `/` |
| **路径举例** | `C:\目录1\目录2\文件` | `/目录1/目录2/文件` |

**绝对路径说明**

在 Linux 绝对路径中，第一个 `/` 代表根目录，后面其他的 `/` 代表两个目录之间的间隔符。  
举例：`/home/binzi` 中，开头的 `/` 是根目录，后续的 `/` 是目录分隔符。

**家目录 `~`**

- `~` 表示当前用户的家目录。
- **普通用户**：家目录是 `/home/普通用户名`
- **root 用户**：家目录是 `/root`
- **快速回家**：`cd ~`，注意 `~` 可以省略（直接输入 `cd` 即可回到家目录）

### Linux 基础命令 🔸 核心

#### 基础用法

**文件与目录操作**

| 命令 | 作用 | 常用选项 |
| :--- | :--- | :--- |
| `ls` | 列出目录内容 | `-l` 详细列表，`-a` 显示隐藏文件，`-h` 人类可读大小 |
| `cd` | 切换目录 | `cd ..` 上级，`cd ~` 回家目录 |
| `pwd` | 显示当前路径 | — |
| `mkdir` | 创建目录 | `-p` 递归创建 |
| `rm` | 删除文件/目录 | `-r` 递归删除目录，`-f` 强制删除 |
| `cp` | 复制 | `-r` 复制目录 |
| `mv` | 移动/重命名 | — |
| `cat` | 查看文件内容 | — |
| `less` / `more` | 分页查看 | — |
| `head` / `tail` | 查看开头/结尾 | `-n 20` 指定行数，`tail -f` 实时追踪 |
| `find` | 查找文件 | `find / -name "*.py"` |
| `grep` | 文本搜索 | `-i` 忽略大小写，`-r` 递归搜索 |

```bash
# 常用组合
ls -lah          # 详细列表，显示隐藏文件，人类可读大小
grep -r "error" /var/log   # 在日志目录递归搜索 "error"
find . -name "*.py" -type f   # 查找当前目录下所有 Python 文件
```

**命令行小技巧**

- **`tab` 补全**：输入命令或文件名的前几个字母后按 `Tab` 键，系统自动补全文件名称或命令，提高输入效率。
- **`history`**：显示已执行过的历史命令记录。
- **上下箭头 `↑` / `↓`**：快速翻看历史命令，无需重新输入。

**系统管理**

| 命令 | 作用 |
| :--- | :--- |
| `ps` | 查看进程 |
| `top` / `htop` | 实时系统资源监控 |
| `df -h` | 查看磁盘空间 |
| `free -h` | 查看内存使用 |
| `kill PID` | 终止进程 |
| `chmod` | 修改文件权限 |
| `chown` | 修改文件所有者 |

### 权限管理 🔸 核心

#### 定义与本质

Linux 文件权限由 10 个字符表示：

```
-rwxr-xr--  1 alice dev  1234 Jan 15 10:00 script.py
│└┬┘└┬┘└┬┘
│ │  │  └── 其他用户权限: r--
│ │  └───── 所属组权限: r-x
│ └──────── 所有者权限: rwx
└────────── 文件类型: - 普通文件, d 目录, l 链接
```

**权限数值**

| 权限 | 二进制 | 数值 |
| :--- | :--- | :--- |
| `r--` | 100 | 4 |
| `-w-` | 010 | 2 |
| `--x` | 001 | 1 |
| `rw-` | 110 | 6 |
| `rwx` | 111 | 7 |

```bash
chmod 755 script.py    # rwxr-xr-x
chmod 644 file.txt     # rw-r--r--
chmod +x script.py     # 添加执行权限
```

## 4️⃣ 避坑指南 & 易错对比

| 对比组 | 区分要点 |
| :--- | :--- |
| `rm` vs `rmdir` | 前者删除文件或目录（需 `-r`），后者只能删除空目录 |
| `cp` vs `mv` | 前者复制（原文件保留），后者移动/重命名（原文件消失） |
| `cat` vs `less` | 前者一次性输出全部，后者分页查看（适合大文件） |
| `chmod 777` 风险 | 给予所有用户全部权限，存在严重安全隐患 |
| `sudo` vs `su` | 前者以 root 权限执行单个命令，后者切换到 root 用户 |
| 绝对路径 vs 相对路径 | `/home/alice/file` vs `./file` 或 `../file` |

## 3️⃣ 应用场景与扩展

> **案例：自动化训练脚本**

```bash
#!/bin/bash
# train.sh

echo "=== 开始训练 ==="
cd /home/user/project

# 激活环境
source ~/miniconda3/etc/profile.d/conda.sh
conda activate pytorch

# 检查 GPU
nvidia-smi

# 运行训练（后台运行，输出到日志）
nohup python train.py \
    --epochs 100 \
    --batch_size 32 \
    --lr 0.001 \
    > logs/train_$(date +%Y%m%d_%H%M%S).log 2>&1 &

echo "训练已在后台启动，PID: $!"
echo "查看日志: tail -f logs/train_*.log"
```
