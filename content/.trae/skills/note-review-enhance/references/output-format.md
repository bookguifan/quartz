# Markdown 输出格式规范

> **适用范围**：AI 增强后的学习笔记 Markdown 文件输出。
> **前置依赖**：必须先完成概念完整性核对、审核纠错、补齐增强。

---

## 一、输出格式骨架

### 1.1 文件命名

```
N.主题名（AI增强版）.md
```

- `N`：数字序号（对应原笔记顺序，如 1, 2, 3...）
- `主题名`：核心主题，如"机器学习概述"、"KNN算法"
- 示例：`1.机器学习概述（AI增强版）.md`

### 1.2 头部信息

```markdown
# [dayXX] 学习笔记｜主题名（AI 增强版）

**📅 日期**：[原笔记日期或"未标注"] **⏱ 学习时长**：[原笔记时长或"未标注"] **🔧 AI 审核版本**：[版本号]
**🏷 标签**：[相关标签，如 #学习笔记 #机器学习]
```

### 1.3 模块顺序（固定）

```markdown
## 📌 核心速览
[summary callout：一段话总结全文核心内容]

## 1️⃣ 完整知识库
[多个主题块，每个主题按四段式组织]

## 2️⃣ 修正与删除记录
[bug callout：记录所有修正和删除]

## 3️⃣ 代码库
[超过20行的综合代码案例，或注明"省略"]

## 4️⃣ 避坑指南 & 易错对比
[warning callout：易混概念对比表]
[danger callout：常见错误与规避]

## 5️⃣ 知识网络
[课内联动、前后衔接、AI/实战落地]

## 8️⃣ AI 附加说明
[组织方式、结构调整说明、重要性判断摘要、难度标签分布、扩展块统计、代码库使用情况、可能遗漏主题、不确定项、自检声明]
```

**模块编号固定**：`1️⃣→2️⃣→3️⃣→4️⃣→5️⃣→8️⃣`，不得更改或省略。

## 二、主题块结构（四段式）

每个知识主题按以下四级标题组织：

```markdown
## 主题名 [难度标签]

### 定义与本质
[概念定义、核心机制、本质说明、包含关系等]

### 基础用法
[基本语法、核心参数、简单示例、常见用法]

### 进阶用法与原理（可选）
[深层原理、复杂用法、性能分析、算法推导]

### 避坑与局限
[常见误区、使用限制、注意事项]
```

### 2.1 难度标签规则

| 标签 | 含义 | 使用场景 |
|------|------|---------|
| 🔹 基础 | 术语定义、常识性概念 | 简单术语、基本概念 |
| 🔸 核心 | 核心算法、关键流程、必会知识 | 核心知识点、重点算法 |
| 🔺 难点 | 复杂推导、易混淆概念、深度理解 | 难点、易错点 |

### 2.2 AI 扩展块插入规则

扩展块使用 `[!note] 💡 AI 扩展（级别）` callout：

| 级别 | 触发条件 | 内容类型 |
|------|---------|---------|
| 基础 | 概念需要补充背景知识 | 背景解释、实际意义、应用场景 |
| 进阶 | 概念需要深入原理 | 算法原理、性能分析、工程实践 |

**插入位置**：紧跟在需要扩展的概念后面，通常在"定义与本质"或"基础用法"之后。

**数量规则**：
- 基础扩展：N≤3→1个, N=4~6→1~2个, N≥7→2~3个
- 进阶扩展：N≥3→1个, N≥6→1~2个
（N = 本笔记中独立知识主题的数量）

### 2.3 关键概念 ==高亮== 规范

- 每主题块的"定义与本质"段落中用 ==高亮== 标记核心术语
- 每个主题块高亮不超过 3 处
- 仅高亮一词或短语，不整句高亮
- 示例：`KNN是一种 ==惰性学习== 算法`、`特征尺度差异会 ==支配== 预测结果`

### 2.4 速查卡片规范

**触发条件**：主题涉及常用 API（如 sklearn 类），在"基础用法"后插入速查卡片。

**格式**：

```markdown
> [!info] 🧠 速查卡片：API名
> **签名**：`API(主要参数)`
> **核心方法**：`fit()` | `predict()` | `fit_transform()`
> **属性**：`属性1`（说明）、`属性2`（说明）
> 🎯 **最佳场景**：一句话说明适用场景
```

**示例**：

```markdown
> [!info] 🧠 速查卡片：StandardScaler
> **签名**：`StandardScaler()`
> **核心方法**：`fit_transform(X)` — 计算均值标准差并转换；`transform(X)` — 用已有参数转换
> **属性**：`mean_`（均值）、`scale_`（标准差）、`var_`（方差）
> 🎯 **最佳场景**：大数据集、含异常值的数据
```

### 2.5 公式 LaTeX 化规范

- 数学公式保留原文文本形式，在下方提供 LaTeX 展示
- LaTeX 使用 `$$...$$` 块级公式

```markdown
> 原文公式：x' = (当前值 - 该列平均值) / 标准差

**标准化公式**：

$$x' = \frac{x - \mu}{\sigma}$$
```

## 三、Callout 使用规范

| 类型 | 语法 | 使用场景 |
|------|------|---------|
| `[!summary]` | `> [!summary] 核心速览` | 开篇总览 |
| `[!note]` | `> [!note] 💡 AI 扩展（基础/进阶）` | AI 补充知识 |
| `[!tip]` | `> [!tip] 算法选择原则` | 提示/建议 |
| `[!info]` | `> [!info] 🧠 速查卡片` | API 速查 |
| `[!warning]` | `> [!warning] 核心理念` | 警告/重要理念 |
| `[!danger]` | `> [!danger] 常见错误与规避` | 严重错误规避 |
| `[!bug]` | `> [!bug] 修正记录` | 原笔记错误修正 |

## 四、表格使用指南

### 4.1 表格优先原则

多维度对比、步骤流程、术语列举、属性说明等内容**优先使用表格呈现**，避免纯文本列表。当内容适合用表格结构化时（≥2个维度的对比或列举），应尽量使用表格；若内容不适合表格化（如单一概念定义），可不强制要求。

### 4.2 必须使用表格的场景

| 场景 | 示例 | 表格列设计 |
|------|------|-----------|
| 多维度对比 | 有监督 vs 无监督 vs 半监督 | 维度A\|维度B\|维度C\|... |
| 步骤流程 | 建模六步流程 | 步骤\|操作内容\|目的\|关键注意点 |
| 术语列举 | 样本、特征、标签定义 | 术语\|英文\|定义\|示例 |
| 属性说明 | sklearn API 参数 | 参数\|类型\|默认值\|说明 |
| 状态诊断 | 欠拟合 vs 过拟合 | 状态\|表现A\|表现B\|原因\|解决方案 |
| 硬件/工具对比 | CPU vs GPU vs TPU | 硬件\|定位\|适用场景\|特点 |

### 4.3 原始内容保留规则

- 原始笔记中的表格应尽量保留为表格形式
- 仅在以下情况允许转换：
  - 表格列数不一致、无表头分隔行（格式错误）
  - 表格内容需重组为更合理的分类方式
- 多维度对比信息（如"有监督 vs 无监督"的5+个维度）**必须**用表格呈现，禁止拆为列表
- 原始笔记中的代码示例不得删除，应放入"基础用法"段落或"3️⃣ 代码库"模块
- 原始笔记中的 `[[笔记名]]` 内部链接必须保留，不得改为纯文本

### 4.4 表格设计规范

- 每行同一主题，每列同一维度
- 表头必须明确（不能省略）
- 单元格内容保持简洁，复杂解释放正文或 callout
- 同一笔记中，同类表格的列维度尽量保持一致

## 附录A：主题内分层归纳指南

当单个主题下知识点较多时，使用中间分类节点承上启下：

```markdown
## 算法名 🔸 核心

### 定义与本质
### 基础用法
#### 分类1（如：参数说明）
#### 分类2（如：返回值说明）
### 进阶用法与原理
#### 算法原理
#### 复杂度分析
### 避坑与局限
```

**中间分类命名建议**：
- 参数说明 / API 说明 / 返回值说明
- 算法原理 / 数学推导 / 复杂度分析
- 应用场景 / 最佳实践 / 性能优化

## 附录B：难度标签判定矩阵

| 判定维度 | 🔹 基础 | 🔸 核心 | 🔺 难点 |
|---------|--------|--------|--------|
| **理解深度** | 知道是什么 | 知道怎么用 | 知道为什么 |
| **记忆要求** | 名词+定义 | 流程+参数 | 原理+推导 |
| **出错代价** | 低（影响小） | 中（影响结果） | 高（导致错误结论） |
| **考试频率** | 偶尔考查 | 必考重点 | 拉分题目 |
| **对应处理** | 表格速查 | 代码示例+对比表 | 分步演算+扩展块 |

**判定流程**：
1. 该概念是否涉及复杂数学推导？→ 是 → 🔺 难点
2. 该概念是否是课程核心考点？→ 是 → 🔸 核心
3. 其他 → 🔹 基础
