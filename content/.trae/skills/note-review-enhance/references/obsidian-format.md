# Obsidian 格式适配与验收规范

> **适用范围**：AI 增强后的 Markdown 笔记在 Obsidian 中的语法正确性与逻辑完整性验收。
> **执行时机**：结构化输出前（Step 5）必须逐项自检，全部通过后方可输出。

---

## 一、Obsidian 格式适配规范

### 1.1 链接语法

| 类型 | 语法 | 示例 | 注意 |
|------|------|------|------|
| 内部链接（笔记） | `[[笔记名]]` | `[[2.KNN算法]]` | 笔记名不含扩展名 |
| 图片嵌入 | `![[文件名.png]]` | `![[Pasted image.png]]` | 必须包含扩展名 |
| WikiLink 误报 | `[[笔记名]]` | `[[机器学习概述]]` | **不是图片**，正则匹配时需过滤 |

**图片引用规则**：
- 使用 `![[filename.ext]]` 格式（Obsidian WikiLink 图片语法）
- 不使用标准 Markdown `![alt](path)`（除非跨目录引用）
- 图片文件应位于同目录 `asset/` 文件夹中

### 1.2 标签语法

```markdown
#标签名  #多词标签
```

- 标签前有空格或行首
- 多词标签直接连写或使用驼峰
- 标签放在头部信息行或文末标签云

### 1.3 高亮语法

```markdown
==高亮文本==
```

- 用于标记核心概念、关键结论
- 适度使用，每主题不超过 3 处

### 1.4 Callout 语法

```markdown
> [!type] 标题
> 内容行1
> 内容行2
```

**支持的类型**：
- `[!summary]` — 核心速览
- `[!note]` — AI 扩展
- `[!tip]` — 提示建议
- `[!info]` — 速查卡片
- `[!warning]` — 重要警告
- `[!danger]` — 严重错误规避
- `[!bug]` — 修正记录

### 1.5 代码块

````markdown
```python
# 代码必须有语言标注
代码内容...
```
````

- 所有代码块必须标注语言类型（python、java、bash 等）
- 行内代码用 `` ` `` 包裹

### 1.6 嵌入语法

```markdown
![[其他笔记名]]        # 嵌入整个笔记
![[其他笔记名#标题]]    # 嵌入指定标题块
```

- 用于知识网络中的联动引用

## 二、语法验收标准（7项）

生成最终笔记前，必须全部通过：

| 序号 | 验收项 | 通过标准 | 失败示例 |
|------|--------|---------|---------|
| 1 | 代码块语言标注 | 所有 ``` 后均有语言类型 | 无标注的代码块 |
| 2 | 图片引用格式 | 使用 `![[文件名.扩展名]]` | 使用 `![](path)` 或缺失扩展名 |
| 3 | 内部链接有效性 | `[[笔记名]]` 指向已存在笔记 | 指向不存在的笔记 |
| 4 | 标签格式 | `#标签` 前有空格或位于行首 | `#标签` 紧跟文字 |
| 5 | Callout 语法 | 正确的 `> [!type]` 格式 | 缺少 `>` 或 `[!]` |
| 6 | 标题层级连续 | H1→H2→H3，无跳级 | H1 后直接 H4 |
| 7 | 表格格式 | 每行列数一致，有表头分隔行 | 列数不一致 |

## 三、笔记逻辑验收标准（11项）

生成最终笔记前，必须全部通过：

| 序号 | 验收项 | 通过标准 | 优先级 |
|------|--------|---------|--------|
| 1 | **概念完整性** | 增强版内容 ≥ 原始笔记，无压缩/丢失 | P0 |
| 2 | 难度标签覆盖 | 每个主题均有 🔹/🔸/🔺 标签 | P1 |
| 3 | 扩展块数量 | 符合 N 值区间规则 | P1 |
| 4 | 修正记录一致 | 修正记录与实际修改一一对应 | P1 |
| 5 | 代码可运行 | 代码可独立运行或已标注依赖等级 | P1 |
| 6 | 易混对比表 | N≥3 时至少 1 组对比表 | P2 |
| 7 | 知识网络链接 | 所有 `[[链接]]` 指向有效笔记 | P2 |
| 8 | 结构调整说明 | 若重组了原结构，必须在 AI 附加说明中记录 | P2 |
| 9 | 图片无悬空 | 所有图片引用均有实际文件 | P2 |
| 10 | 无占位符残留 | 无 `[[filename.png]]` 等无效占位符 | P2 |
| 11 | 核心速览准确 | 一段话准确概括全文核心内容 | P2 |
| 12 | **表格完整性** | 原始笔记中的表格未被打散为列表/段落（除非原表格式错误）；多维度对比必须用表格 | P1 |
| 13 | **链接完整性** | 原始笔记中的 `[[笔记名]]` 链接未改为纯文本；课内联动必须使用 WikiLink | P2 |
| 14 | **概念深化** | 原文薄弱的主题块已补全定义/原理/用法/局限四层；有补充内容的主题块 ≥ 总数的 1/3 | P1 |

**概念完整性详细校验（第1项）**：

- 逐条对照原始笔记，确认每个概念在增强版中有**等价或更详细**的表达
- **禁止行为**：
  - 将正文描述压缩为 callout
  - 仅保留名词不保留解释
  - 删除流程步骤
  - 丢失 A=B 对应关系
- **强制保留**：原始笔记中"定义+用法+关联+局限"的完整链条

## 四、质量自检清单

输出最终笔记前，AI 必须执行以下自检：

### 语法层面

- [ ] 所有代码块均有语言标注（7项-1）
- [ ] 所有图片引用格式正确（7项-2）
- [ ] 所有内部链接指向有效（7项-3）
- [ ] 标签格式正确（7项-4）
- [ ] Callout 语法正确（7项-5）
- [ ] 标题层级连续（7项-6）
- [ ] 表格格式正确（7项-7）

### 逻辑层面

- [ ] **增强版未比原始笔记简略**（11项-1，最关键！）
- [ ] 每个主题均有难度标签（11项-2）
- [ ] 扩展块数量符合规则（11项-3）
- [ ] 修正记录与实际修改一致（11项-4）
- [ ] 代码可运行或已标注依赖（11项-5）
- [ ] 易混概念对比表 ≥1 组（11项-6，当N≥3）
- [ ] 知识网络链接有效（11项-7）
- [ ] 结构调整有说明（11项-8）
- [ ] 图片引用无悬空（11项-9）
- [ ] 无占位符残留（11项-10）
- [ ] 核心速览准确（11项-11）
- [ ] 原始表格未被打散为列表（12项）
- [ ] 课内联动使用 `[[笔记名]]` WikiLink 格式（13项）
- [ ] 原文薄弱主题已深化补充（14项）
- [ ] 核心概念有 ==高亮== 标记（每主题 ≤3 处）
- [ ] API类主题有 `[!info]` 速查卡片
- [ ] 数学公式有 LaTeX 展示

### 资源层面

- [ ] 图片已迁移到 `asset/` 目录
- [ ] 已验证图片文件存在
- [ ] 已删除未引用图片
- [ ] 无 `[[filename.png]]` 占位符

**自检声明模板**（放在 AI 附加说明末尾）：

```markdown
- **自检声明**：已按语法验收标准（7项）和笔记逻辑验收标准（14项）逐项自检确认。
```
