# Obsidian 格式适配与验收规范

> **适用范围**：AI 增强后的 Markdown 笔记在 Obsidian 中的语法正确性与逻辑完整性验收。
> **执行时机**：结构化输出前（Step 5）必须逐项自检，全部通过后方可输出。

---

## 一、Obsidian 格式适配规范

### 1.1 链接语法

| 类型 | 语法 | 示例 | 注意 |
|------|------|------|------|
| 内部链接（笔记） | `[[笔记名]]` | `[[2.KNN算法]]` | 笔记名不含扩展名 |
| 图片嵌入 | `![[文件名.png]]` | `![[Pasted image.png]]` | 必须包含扩展名 |
| WikiLink 误报 | `[[笔记名]]` | `[[机器学习概述]]` | **不是图片**，正则匹配时需过滤 |

**图片引用规则**：
- 使用 `![[filename.ext]]` 格式（Obsidian WikiLink 图片语法）
- 不使用标准 Markdown `![alt](path)`（除非跨目录引用）
- 图片文件应位于同目录 `asset/` 文件夹中

### 1.2 标签语法

```markdown
#标签名  #多词标签
```

- 标签前有空格或行首
- 多词标签直接连写或使用驼峰
- 标签放在头部信息行或文末标签云

### 1.3 高亮语法

```markdown
==高亮文本==
```

- 用于标记核心概念、关键结论
- 适度使用，每主题不超过 3 处

### 1.4 Callout 语法

```markdown
> [!type] 标题
> 内容行1
> 内容行2
```

**支持的类型**：
- `[!summary]` — 核心速览
- `[!note]` — AI 扩展
- `[!tip]` — 提示建议
- `[!info]` — 速查卡片
- `[!warning]` — 重要警告
- `[!danger]` — 严重错误规避
- `[!bug]` — 修正记录

### 1.5 代码块

````markdown
```python
# 代码必须有语言标注
代码内容...
```
````

- **所有代码块必须标注语言类型**，禁止空标注（即 ``` 后直接换行）
- **代码块结束标记规则**：结束标记必须为**纯 ```**（无任何后缀）。若一行匹配 `^```(\w+)\s*$` 且该行之后为空行或新段落/标题，则该行为**错误结束标记**，必须替换为纯 ```
  - ❌ 错误：` ```python ` 作为结束标记
  - ❌ 错误：` ```bash ` 作为结束标记
  - ❌ 错误：` ```sql ` 作为结束标记
  - ✅ 正确：开始用 ` ```python `，结束用纯 ` ``` `
- 语言类型对照表：
  | 内容类型 | 标注 |
  |---------|------|
  | Python 代码 | `python` |
  | Shell/Bash 命令 | `bash` |
  | JSON/YAML/TOML | `json` / `yaml` / `toml` |
  | 文本流程/算法步骤/纯文本 | `text` |
  | 数学公式块 | `math` 或嵌入 LaTeX |
  | Mermaid 图表 | `mermaid` |
- 行内代码用 `` ` `` 包裹
- **输出标注标准**：
  - 可运行代码块：在最后一行或紧邻代码块下方，使用行内注释标注预期输出 `# 输出：...`
  - 不可运行代码块（依赖外部文件）：在代码块上方使用 `> [!warning]` 标注依赖说明，如 `> [!warning] 本代码依赖外部文件：xxx.csv`

### 1.6 嵌入语法

```markdown
![[其他笔记名]]        # 嵌入整个笔记
![[其他笔记名#标题]]    # 嵌入指定标题块
```

- 用于知识网络中的联动引用

### 1.7 YAML Frontmatter（Properties）

增强版笔记必须以 YAML frontmatter 开头，用 `---` 分隔，作为 Obsidian Properties 的数据源。

**基本规则**：
- frontmatter 必须位于文件最开头（第 1 行为 `---`，最后一行为 `---`）
- 属性值中含中文时无需加引号，但含特殊字符（冒号、`#` 等）时用双引号包裹
- `tags` 必须使用 YAML 列表格式（每项前加 `- `），不使用行内数组 `[a, b]`
- `date` 无法推断时留空（`date: `），不填"未标注"等文字
- 属性详细定义参见 `references/output-format.md` 1.2 节

## 二、语法验收标准（8项）

生成最终笔记前，必须全部通过：

| 序号 | 验收项 | 通过标准 | 失败示例 |
|------|--------|---------|---------|
| 1 | 代码块语言标注 | 所有代码块**开始标记**后均有语言类型；所有**结束标记**必须为纯 ```（禁止 ` ```python ` 等带后缀的结束标记） | 开始用空标注 ` ``` ` 后换行；结束用 ` ```python ` 等带语言后缀 |
| 2 | 图片引用格式 | 使用 `![[文件名.扩展名]]`；文件名建议仅含字母、数字、下划线、连字符 | 使用 `![](path)`、缺失扩展名、含中文或特殊字符 |
| 3 | 内部链接有效性 | `[[笔记名]]` 指向已存在笔记 | 指向不存在的笔记 |
| 4 | 标签格式 | `#标签` 前有空格或位于行首 | `#标签` 紧跟文字 |
| 5 | Callout 语法 | 正确的 `> [!type]` 格式 | 缺少 `>` 或 `[!]` |
| 6 | 标题层级连续 | H1→H2→H3，无跳级 | H1 后直接 H4 |
| 7 | 表格格式 | 每行列数一致，有表头分隔行 | 列数不一致 |
| 8 | YAML frontmatter | `---` 分隔符正确；必填属性齐全（title/review_version/publish/category/difficulty/tags/status）；tags 为列表格式；正文中无残留的行内元数据行（`**📅 日期**`等） | 缺少 frontmatter、tags 用行内数组、正文中保留旧行内元数据 |

## 三、笔记逻辑验收标准（11项）

生成最终笔记前，必须全部通过：

| 序号 | 验收项 | 通过标准 | 优先级 |
|------|--------|---------|--------|
| 1 | **概念完整性** | 增强版内容 ≥ 原始笔记，无压缩/丢失 | P0 |
| 2 | 难度标签覆盖 | 每个主题均有 🔹/🔸/🔺 标签 | P1 |
| 3 | 扩展块数量 | 符合 N 值区间规则 | P1 |
| 4 | 修正记录一致 | 修正记录与实际修改一一对应 | P1 |
| 5 | 代码可运行 | 代码可独立运行或已标注依赖等级 | P1 |
| 6 | 易混对比表 | N≥3 时至少 1 组对比表 | P2 |
| 7 | 知识网络链接 | 所有 `[[链接]]` 指向有效笔记 | P2 |
| 8 | 结构调整说明 | 若重组了原结构，必须在 AI 附加说明中记录 | P2 |
| 9 | 图片无悬空 | 所有图片引用均有实际文件 | P2 |
| 10 | 无占位符残留 | 无 `[[filename.png]]` 等无效占位符 | P2 |
| 11 | 核心速览准确 | 一段话准确概括全文核心内容 | P2 |

**概念完整性详细校验（第1项）**：

- 逐条对照原始笔记，确认每个概念在增强版中有**等价或更详细**的表达
- **禁止行为**：
  - 将正文描述压缩为 callout
  - 仅保留名词不保留解释
  - 删除流程步骤
  - 丢失 A=B 对应关系
- **强制保留**：原始笔记中"定义+用法+关联+局限"的完整链条

## 四、质量自检清单

输出最终笔记前，AI 必须执行以下自检：

### 语法层面

- [ ] 所有代码块均有语言标注（7项-1）
- [ ] 所有图片引用格式正确（7项-2）
- [ ] 所有内部链接指向有效（7项-3）
- [ ] 标签格式正确（7项-4）
- [ ] Callout 语法正确（7项-5）
- [ ] 标题层级连续（7项-6）
- [ ] 表格格式正确（7项-7）
- [ ] YAML frontmatter 完整（8项-8）：`---` 分隔符正确、必填属性齐全、tags 为列表格式、正文无残留行内元数据

### 逻辑层面

- [ ] **增强版未比原始笔记简略**（11项-1，最关键！）
- [ ] 每个主题均有难度标签（11项-2）
- [ ] 扩展块数量符合规则（11项-3）
- [ ] 修正记录与实际修改一致（11项-4）
- [ ] 代码可运行或已标注依赖（11项-5）
- [ ] 易混概念对比表 ≥1 组（11项-6，当N≥3）
- [ ] 知识网络链接有效（11项-7）
- [ ] 结构调整有说明（11项-8）
- [ ] 图片引用无悬空（11项-9）
- [ ] 无占位符残留（11项-10）
- [ ] 核心速览准确（11项-11）
- [ ] 代码示例分流正确：需要代码的知识点已提供 ≤10 行可运行示例，无需代码的知识点未硬塞代码
- [ ] 重要函数速查卡片完整（签名、参数、≥3 组示例、场景标签）
- [ ] 触发分步演算条件的复杂逻辑已提供调用栈展开或状态变化表
- [ ] 扩展块内容符合限制（含原理/实战/性能/进阶中至少两项，不含完整函数/类实现）

### 资源层面

- [ ] 图片已迁移到 `asset/` 目录
- [ ] 已验证图片文件存在（所有 `![[...]]` 引用均有对应文件）
- [ ] 已删除未引用图片（`asset/` 中不存在被任何 `.md` 引用的冗余文件）
- [ ] 无 `[[filename.png]]` 占位符
- [ ] 图片文件名不含中文或特殊字符（建议仅使用字母、数字、下划线、连字符）

**自检声明模板**（放在 AI 附加说明末尾）：

```markdown
- **自检声明**：已按语法验收标准（8项）和笔记逻辑验收标准（11项）逐项自检确认。
```
