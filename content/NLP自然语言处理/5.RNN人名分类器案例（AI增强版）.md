> [!summary] 核心速览

> - **张量形状链**：`name` 字符串 → `(seq, 57)` → `(1, seq, 57)` → `(1, seq, 128)` → `(1, 18)` → softmax。

## 任务背景与流程总览 🔹 基础

### 定义与本质

**任务目标**：给定一个人名（如 "Ang"、"Yuasa"），判定其属于 18 个国家类别中的哪一个。这是一个典型的**文本分类任务**，更准确地说是**多分类任务（18 分类）**。

**数据格式**：每行包含两列，第一列是人名，第二列是国家类别，中间用制表符 `\t` 隔开。

```text
Ang	Chinese
AuYong	Chinese
Yuasa	Japanese
Yuhara	Japanese
Yunokawa	Japanese
```

### 基础用法

**任务实现流程**（完整五步）：

```text
1. 获取数据：案例中是直接给定的 txt 文件
2. 数据预处理：脏数据清洗、数据格式转换、Dataset 构造、Dataloader 构造
3. 模型搭建：RNN、LSTM、GRU 一系列模型
4. 模型训练和评估（测试）：NLLLoss + Adam
5. 模型上线：API 接口（后续课程讲解）
```

> [!note] 💡 AI 扩展（基础）
> **为什么选 RNN 做人名分类？**
> - 人名是**字符序列**（如 "Ang" = ['A','n','g']），RNN 天然适合处理变长序列。
> - 与 CNN 不同，RNN 不要求输入定长，可以处理任意长度的人名。
> - 国家类别信息往往分布在人名的字符组合中（如 "Yukawa" 中的 "wa" 是日语后缀），RNN 能在序列最后位置聚合整个序列的信息。
> **实战意义**：人名分类是 NLP 入门经典案例，能覆盖"数据预处理 → 模型搭建 → 训练 → 预测"完整机器学习工作流。
> **应用场景**：用户画像中的国籍推断、跨境反欺诈、电商多语言客服路由等。

## 数据预处理 🔸 核心

### 定义与本质

数据预处理是模型训练前的关键步骤，本案例包含三个子任务：
1. **read_data**：从 txt 文件读取人名与国家标签
2. **NameClassDataset**：将人名转为 one-hot 张量，封装为 PyTorch Dataset
3. **DataLoader**：将 Dataset 封装为可迭代、可批处理、可打乱的迭代器

### 基础用法

#### 4.1 读取 txt 文档数据

**目的**：将 txt 文档中的数据读取到内存，将人名与国家类别分别存入两个列表。

```python
def read_data(filename):
    # 1. 初始化两个空列表，分别存人名 x 和国家标签 y
    my_list_x, my_list_y = [], []
    # 2. 读取文件内容
    with open(filename, 'r', encoding='utf-8') as fr:
        for line in fr.readlines():
            # 跳过长度过短的脏数据行（≤5 个字符认为格式不完整）
            if len(line) <= 5:
                continue
            # strip() 默认去除字符串首尾空白字符
            # split('\t') 按制表符分割为人名和国家
            x, y = line.strip().split('\t')
            my_list_x.append(x)
            my_list_y.append(y)

    return my_list_x, my_list_y
```

> [!warning] 本代码依赖外部文件：./data/name_classfication.txt，需提前准备数据集才能运行。

#### 4.2 构建自定义数据源 Dataset

**目的**：使用 PyTorch 框架的标准做法——继承 `torch.utils.data.Dataset`，构造数据源供模型使用。

**Dataset 魔法方法说明**：

| 方法 | 作用 | 调用方式 |
|------|------|---------|
| `__init__` | 初始化数据（人名列表、标签列表、样本总数） | 构造时自动调用 |
| `__len__` | 返回样本总个数 | `len(dataset)` |
| `__getitem__` | 按索引取出样本（人名张量 + 标签张量） | `dataset[index]` |

```python
from torch.utils.data import Dataset
import torch

# all_letters: 所有可能出现的字符集合（如 a-z + 空格等）
# n_letter: 字符集合大小（如 57）
# categorys: 国家类别列表（如 ['Chinese', 'Japanese', ...] 共 18 个）

class NameClassDataset(Dataset):
    def __init__(self, mylist_x, mylist_y):
        # 保存人名列表和标签列表
        self.mylist_x = mylist_x
        self.mylist_y = mylist_y
        # 样本总数
        self.sample_len = len(mylist_x)

    # 魔法方法 __len__：返回样本总数
    def __len__(self):
        return self.sample_len

    # 魔法方法 __getitem__：按索引取样本
    def __getitem__(self, index):
        # 1. index 异常值处理：限制在合法范围内
        index = min(max(index, 0), self.sample_len - 1)
        # 2. 根据 index 取出人名和国家名
        x = self.mylist_x[index]
        y = self.mylist_y[index]
        # 3. 对人名进行 one-hot 编码：
        #    思路：针对每个人名的字符做 one-hot，然后按字符位置拼接成 (seq_len, n_letter) 矩阵
        tensor_x = torch.zeros(len(x), n_letter)
        for li, letter in enumerate(x):
            tensor_x[li][all_letters.find(letter)] = 1
        # 4. 获取标签：将国家字符串转为类别索引（0~17）
        tensor_y = torch.tensor(categorys.index(y), dtype=torch.long)
        return tensor_x, tensor_y
```

#### 4.3 构建数据迭代器 DataLoader

**目的**：将 Dataset 进一步封装为可迭代器，支持：
- 自动按 `batch_size` 增维
- 自动随机打乱样本顺序（`shuffle=True`）
- 支持 `for x, y in dataloader` 循环

```python
from torch.utils.data import DataLoader

filename = './data/name_classfication.txt'
my_list_x, my_list_y = read_data(filename)
mydataset = NameClassDataset(mylist_x=my_list_x, mylist_y=my_list_y)
# batch_size=1：每次取 1 个样本；shuffle=True：每轮打乱顺序
my_dataloader = DataLoader(dataset=mydataset, batch_size=1, shuffle=True)
```

> [!note] 💡 AI 扩展（基础）
> **PyTorch Dataset/Dataloader 数据流图解（从原始文本到模型输入）**：
>
> ```text
> ┌──────────────────────────────────────────────────────────────┐
> │                  原始数据文件（txt）                         │
> │  Ang\tChinese                                              │
> │  Yuasa\tJapanese                                           │
> │  Yuhara\tJapanese                                         │
> │  Yunokawa\tJapanese                                       │
> └──────────────┬───────────────────────────────────────────────┘
>                │ read_data(filename)
>                │ 读取txt → 按行split('\t') → 过滤短行
>                ▼
> ┌──────────────────────────────────────────────────────────────┐
> │            两个列表（内存中）                                │
> │  my_list_x = ['Ang', 'Yuasa', 'Yuhara', ...]                │
> │  my_list_y = ['Chinese', 'Japanese', 'Japanese', ...]       │
> └──────────────┬───────────────────────────────────────────────┘
>                │ NameClassDataset(mylist_x, mylist_y)
>                │ 封装为 PyTorch Dataset 对象
>                ▼
> ┌──────────────────────────────────────────────────────────────┐
> │              Dataset 对象                                   │
> │  __len__()    → 返回样本总数 N                              │
> │  __getitem__(i) → 返回 (tensor_x, tensor_y)                │
> │    ├─ tensor_x: one-hot 编码 [seq_len, 57]                  │
> │    └─ tensor_y: 类别索引 [1] (0~17)                        │
> └──────────────┬───────────────────────────────────────────────┘
>                │ DataLoader(dataset, batch_size, shuffle)
>                │ 封装为可迭代器，自动增 batch 维 + 打乱
>                ▼
> ┌──────────────────────────────────────────────────────────────┐
> │           DataLoader 迭代器                                 │
> │  for x, y in dataloader:                                   │
> │    x 形状: [batch_size, seq_len, 57]  如 [1, 3, 57]        │
> │    y 形状: [batch_size]               如 [1]               │
> └──────────────┬───────────────────────────────────────────────┘
>                │ 直接送入模型
>                ▼
> ┌──────────────────────────────────────────────────────────────┐
> │              RNN / LSTM / GRU 模型                          │
> │  输入: x [1, seq_len, 57]                                   │
> │  输出: softmax 概率 [1, 18] → 18 个国家类别                 │
> └──────────────────────────────────────────────────────────────┘
> ```
>
> - **原理**：Dataset 负责"如何取一个样本"（__getitem__），DataLoader 负责"如何组织一批样本"（batch + shuffle）。二者职责分离，是 PyTorch 数据管线的标准设计。
> - **实战**：本案例 batch_size=1 仅用于教学；生产场景通常设为 32/64，此时 DataLoader 自动将多个样本的 tensor_x 堆叠为 `[batch, seq_len, 57]`，需配合 collate_fn 处理变长序列的 padding。
> - **进阶**：多进程加载（`num_workers > 0`）可加速数据读取；`pin_memory=True` 可加速 CPU→GPU 数据传输。

### 进阶用法与原理

#### 数据流张量形状变化表

下表展示一个人名 "Ang"（3 个字符）从原始字符串到模型输入的完整形状变化：

| 步骤 | 操作 | 张量形状 | 说明 |
|------|------|---------|------|
| 1 | 原始人名字符串 `"Ang"` | `(3,)` | 字符列表 ['A', 'n', 'g'] |
| 2 | one-hot 编码 `tensor_x` | `(3, 57)` | seq_len=3, n_letter=57 |
| 3 | DataLoader 增 batch 维 | `(1, 3, 57)` | batch_size=1, batch_first=True |
| 4 | 送入 RNN 的 input | `(1, 3, 57)` | (batch, seq_len, input_size) |
| 5 | RNN 输出 output | `(1, 3, 128)` | (batch, seq_len, hidden_size) |
| 6 | RNN 输出 hn | `(1, 1, 128)` | (num_layers, batch, hidden_size) |

> [!note] 💡 AI 扩展（进阶）
> **batch_first 参数详解**：
> - **默认 `batch_first=False`**：输入形状为 `(seq_len, batch_size, input_size)`，PyTorch 内部按 CuDNN 优化路径处理。
> - **设为 `batch_first=True`**：输入形状为 `(batch_size, seq_len, input_size)`，与 DataLoader 的输出第一维对齐。
> - **关键陷阱**：DataLoader 返回的张量第一维始终是 `batch_size`，而 `nn.RNN/LSTM/GRU` 默认 `batch_first=False` 期望第一维是 `seq_len`，**形状不匹配会报错**。
> - **解决方案**：① 设 `batch_first=True`（推荐）；② 在 forward 中手动 `input.transpose(0, 1)` 转置。
> **teacher_forcing 策略说明**：
> - teacher_forcing 是 **Seq2Seq 任务**（如机器翻译）中的训练技巧：训练时不用上一时间步的预测输出作为下一时间步输入，而是直接用**真实的标签**作为下一时间步输入。
> - **本案例是人名分类任务，输入是人名序列，输出是单一国家类别，并非 Seq2Seq 任务，因此不使用 teacher_forcing**。
> - teacher_forcing 的典型场景是 day05 的 Seq2Seq 翻译任务，待后续课程展开。
> **性能实践**：
> - 实际工业任务中，`batch_size` 通常设为 32 或 64 而非 1，可大幅提升训练速度。
> - 人名长度不一，使用 `batch_size > 1` 时需配合 `collate_fn` 进行 padding，使同一 batch 内所有序列等长。
> - one-hot 编码维度高（57 维）但稀疏，可以考虑用 `nn.Embedding` 替代以降低维度。

### 避坑与局限

- **`n_letter` 和 `all_letters` 是全局变量**：必须在调用 Dataset 前定义，否则 `__getitem__` 会报 NameError。
- **`categorys` 是国家类别列表**：必须是固定顺序的列表，否则 `categorys.index(y)` 返回的索引与训练时不同导致预测错误。
- **短行过滤**：原代码 `if len(line) <= 5: continue` 会跳过过短的人名，但 5 这个阈值需要根据数据集调整，过短会丢失样本。
- **batch_size=1**：训练效率低，仅用于教学；生产场景应增大 batch_size 并处理 padding。

## 模型搭建 🔸 核心

### 定义与本质

本案例搭建三种模型（`MyRNN` / `MyLSTM` / `MyGRU`），结构高度统一：

```text
输入 (batch, seq, input_size)
  ↓
RNN / LSTM / GRU 层（输出 (batch, seq, hidden_size) + 末位隐状态）
  ↓
取最后一个时间步的隐状态 → Linear 层 → (batch, output_size=18)
  ↓
LogSoftmax(dim=-1) → 输出对数概率 (batch, 18)
```

上图展示 RNN 单步前向计算中输入 `x0` 与隐状态 `h0` 各自经过线性变换后相加，得到输出 `out` 与新隐状态 `h1` 的过程，标注了每一步张量的形状。

**注意事项**：

> [!warning] RNN 模型在实例化时默认 `batch_first=False`，因此需要小心输入数据的形状。
> DataLoader 返回的 x 形状是 `[batch_size, seq_len, input_size]`，所以课堂上代码与讲义稍有不同：讲义是默认的 `batch_first=False`，而本案例代码设为 `batch_first=True`，目的就是直接承接 DataLoader 的输出。

**代码实现**：

```python
import torch
import torch.nn as nn

class MyRNN(nn.Module):
    def __init__(self, input_size, hidden_size, ouput_size, num_layers=1):
        super().__init__()
        # input_size：词嵌入维度（即 n_letter=57，one-hot 的字符维度）
        self.input_size = input_size
        # hidden_size：RNN 隐藏层维度（如 128）
        self.hidden_size = hidden_size
        # output_size：国家类别数（18）
        self.ouput_size = ouput_size
        self.num_layers = num_layers

        # 定义 RNN 网络层
        # batch_first=True：使输入第一维为 batch_size，可直接承接 DataLoader 输出
        self.rnn = nn.RNN(self.input_size, self.hidden_size,
                          num_layers=self.num_layers, batch_first=True)

        # 定义输出全连接层：将 hidden_size 映射到 output_size（18 类）
        self.linear = nn.Linear(self.hidden_size, self.ouput_size)

        # 定义 LogSoftmax 层（与 NLLLoss 搭配使用）
        self.softmax = nn.LogSoftmax(dim=-1)

    def forward(self, input, hidden):
        # input 形状: [batch_size, seq_len, input_size]，如 [1, 9, 57]
        # hidden 形状: [num_layers, batch_size, hidden_size]，如 [1, 1, 128]

        # 将 input 和 hidden 送入 RNN 模型
        # rnn_output 形状: [1, 9, 128]——所有时间步的隐状态
        # rnn_hn 形状: [1, 1, 128]——最后一个时间步的隐状态
        rnn_output, rnn_hn = self.rnn(input, hidden)

        # 取最后一个时间步的隐状态：rnn_output[0] 取第一个样本，[-1] 取最后时间步
        # 结果形状: [128]，再 unsqueeze(0) 增加第 0 维 → [1, 128]
        tmep = rnn_output[0][-1].unsqueeze(0)

        # 经 Linear 层映射到 18 维 → [1, 18]
        output = self.linear(tmep)

        # 经 LogSoftmax 转为对数概率 → [1, 18]
        return self.softmax(output), rnn_hn

    def inithidden(self):
        # 初始化全零隐状态: [num_layers, batch_size, hidden_size]
        return torch.zeros(self.num_layers, 1, self.hidden_size)
```

上图展示 LSTM 单元的三门（遗忘门、输入门、输出门）+ 候选状态 + 细胞状态的内部结构。

**Bi-LSTM 双向结构（补充说明）**：

![[day06_classifier_05.png]]

上图展示 Bi-LSTM 的双向处理流程：正向 LSTM 和反向 LSTM 各自处理一遍序列，最后通过拼接（绿色虚线"拼接"）合并为每个位置的最终表示。本案例只用单向 LSTM，双向结构留作扩展。

**注意事项**：

> [!warning] LSTM 模型在实例化时默认 `batch_first=False`，输入形状需小心。本案例同样设为 `batch_first=True`。
> LSTM 比 RNN 多一个细胞状态 `c0`，`forward` 必须同时传入 `(h0, c0)`，返回也是 `(hn, cn)`。

**代码实现**：

```python
class MyLSTM(nn.Module):
    def __init__(self, input_size, hidden_size, ouput_size, num_layers=1):
        super().__init__()
        self.input_size = input_size
        self.hidden_size = hidden_size
        self.ouput_size = ouput_size
        self.num_layers = num_layers

        # 定义 LSTM 网络层
        # batch_first=True：直接承接 DataLoader 输出
        self.lstm = nn.LSTM(self.input_size, self.hidden_size,
                            num_layers=self.num_layers, batch_first=True)

        self.linear = nn.Linear(self.hidden_size, self.ouput_size)
        self.softmax = nn.LogSoftmax(dim=-1)

    def forward(self, input, hidden, c0):
        # input 形状: [batch_size, seq_len, input_size]，如 [1, 9, 57]
        # hidden 形状: [num_layers, batch_size, hidden_size]，如 [1, 1, 128]
        # c0 形状: 同 hidden，[1, 1, 128]

        # LSTM 前向：注意返回值是 (output, (hn, cn)) 元组
        # lstm_output 形状: [1, 9, 128]
        # lstm_hn 形状: [1, 1, 128]
        # lstm_cn 形状: [1, 1, 128]
        lstm_output, (lstm_hn, lstm_cn) = self.lstm(input, (hidden, c0))

        # 取最后一个时间步的隐状态
        tmep = lstm_output[0][-1].unsqueeze(0)  # [1, 128]

        output = self.linear(tmep)               # [1, 18]
        return self.softmax(output), lstm_hn, lstm_cn

    def inithidden(self):
        # LSTM 需要同时初始化 h0 和 c0
        h0 = torch.zeros(self.num_layers, 1, self.hidden_size)
        c0 = torch.zeros(self.num_layers, 1, self.hidden_size)
        return h0, c0
```

#### 5.3 MyGRU 模型

**GRU 单元结构**：

![[day06_classifier_06.png]]

上图展示 GRU 单元的两个门：**重置门（reset gate）** 和 **更新门（update gate）**，结构比 LSTM 简洁，无独立的细胞状态。

**注意事项**：

> [!warning] GRU 模型在实例化时默认 `batch_first=False`。本案例设为 `batch_first=True`。
> GRU 的 `forward` 返回 `(output, hn)`，**与 RNN 相同，无细胞状态 c**。

**代码实现**：

```python
class MyGRU(nn.Module):
    def __init__(self, input_size, hidden_size, ouput_size, num_layers=1):
        super().__init__()
        self.input_size = input_size
        self.hidden_size = hidden_size
        self.ouput_size = ouput_size
        self.num_layers = num_layers

        # 定义 GRU 网络层
        # batch_first=True：直接承接 DataLoader 输出
        self.gru = nn.GRU(self.input_size, self.hidden_size,
                          num_layers=self.num_layers, batch_first=True)

        self.linear = nn.Linear(self.hidden_size, self.ouput_size)
        self.softmax = nn.LogSoftmax(dim=-1)

    def forward(self, input, hidden):
        # input 形状: [batch_size, seq_len, input_size]，如 [1, 9, 57]
        # hidden 形状: [num_layers, batch_size, hidden_size]，如 [1, 1, 128]

        # GRU 前向：返回 (output, hn)，无 c 状态
        # gru_output 形状: [1, 9, 128]
        # gru_hn 形状: [1, 1, 128]
        gru_output, gru_hn = self.gru(input, hidden)

        # 取最后一个时间步的隐状态
        tmep = gru_output[0][-1].unsqueeze(0)  # [1, 128]

        output = self.linear(tmep)              # [1, 18]
        return self.softmax(output), gru_hn

    def inithidden(self):
        # GRU 只需初始化 h0
        return torch.zeros(self.num_layers, 1, self.hidden_size)
```

### 进阶用法与原理

#### 三种模型 forward 形状对比表

| 模型 | 初始状态 | forward 输入 | forward 返回 | 末位隐状态获取 |
|------|---------|--------------|--------------|----------------|
| `MyRNN` | `h0` | `(input, hidden)` | `(output, hn)` | `rnn_output[0][-1]` |
| `MyLSTM` | `(h0, c0)` | `(input, hidden, c0)` | `(output, hn, cn)` | `lstm_output[0][-1]` |
| `MyGRU` | `h0` | `(input, hidden)` | `(output, hn)` | `gru_output[0][-1]` |

#### 关键代码片段解读：`rnn_output[0][-1].unsqueeze(0)`

```text
rnn_output 形状: [1, 9, 128]   # (batch=1, seq_len=9, hidden_size=128)
       ↓ [0]
[9, 128]                       # 取第 0 个样本的所有时间步
       ↓ [-1]
[128]                          # 取最后一个时间步的隐状态
       ↓ .unsqueeze(0)
[1, 128]                       # 增加第 0 维，恢复 batch 维度，供 Linear 层处理
```

**为什么取最后一个时间步？**
- RNN 是单向从左到右处理序列，最后一个时间步的隐状态 `h_T` 包含了整个序列的信息。
- 对于分类任务，我们只需要一个固定维度的特征向量送入分类头，取末位隐状态是标准做法。

> [!note] 💡 AI 扩展（进阶）
> **为什么用 LogSoftmax + NLLLoss 而不是 Softmax + CrossEntropyLoss？**
> - `LogSoftmax + NLLLoss` 与 `CrossEntropyLoss` 在数学上等价：`CrossEntropyLoss = LogSoftmax + NLLLoss`。
> - PyTorch 的 `nn.CrossEntropyLoss` 内部已经包含了 `LogSoftmax`，所以如果用 `CrossEntropyLoss`，模型 forward 末尾就**不应再加** `LogSoftmax`，否则会做两次 log。
> - 本案例显式用 `LogSoftmax + NLLLoss` 是教学做法，便于理解计算流程；工程实践中通常直接用 `CrossEntropyLoss`。
> **性能实践**：
> - `LogSoftmax(dim=-1)` 中 `dim=-1` 表示在最后一个维度做 softmax，等价于 `dim=1`（当输出形状为 `[batch, 18]` 时）。
> - 用 `dim=-1` 更通用，无论 batch 维度在前面有几维都能正确归一化。

- **类名不一致**：原笔记中 `MyRNN`（5.1 节定义）与训练代码中的 `My_RNN`（带下划线）不一致，运行会报 `NameError`。应统一为 `MyRNN`（见修正记录）。

## 模型训练与预测 🔸 核心

### 定义与本质

模型训练是将数据通过 DataLoader 喂入模型，计算损失、反向传播、更新参数的循环过程。模型预测是加载训练好的参数，对新数据进行前向推理，并通过 `torch.topk` 获取 Top-K 预测结果。

### 基础用法

#### 训练流程总览

```text
1. 获取数据
2. 构建数据源 Dataset
3. 构建数据迭代器 DataLoader
4. 加载自定义的模型
5. 实例化损失函数对象（NLLLoss）
6. 实例化优化器对象（Adam）
7. 定义打印日志参数
8. 开始训练
   8.1 实现外层大循环 epoch
   8.2 内部遍历数据迭代器 dataloader
   8.3 将数据送入模型得到输出结果
   8.4 计算损失
   8.5 梯度清零: optimizer.zero_grad()
   8.6 反向传播: loss.backward()
   8.7 参数更新（梯度更新）: optimizer.step()
   8.8 打印训练日志
9. 保存模型: torch.save(model.state_dict(), "model_path")
```

```python
import time
import json
from tqdm import tqdm
import torch.optim as optim

my_lr = 1e-3     # 学习率
epochs = 1        # 训练轮数

def train_rnn():
    # 1. 读取数据
    my_list_x, my_list_y = read_data(filepath='./data/name_classfication.txt')
    # 2. 实例化 dataset 数据源对象
    my_dataset = NameClassDataset(my_list_x, my_list_y)
    # 3. 实例化模型
    #    n_letters=57, hidden_size=128, 类别总数 output_size=18
    my_rnn = MyRNN(input_size=57, hidden_size=128, output_size=18)
    # 4. 实例化损失函数对象（与 LogSoftmax 搭配）
    my_nll_loss = nn.NLLLoss()
    # 5. 实例化优化器对象
    my_optim = optim.Adam(my_rnn.parameters(), lr=my_lr)
    # 6. 定义打印日志的参数
    start_time = time.time()
    total_iter_num = 0       # 当前已经训练的样本总数
    total_loss = 0           # 已经训练的累计损失
    total_loss_list = []     # 每隔 100 样本保存平均损失
    total_acc_num = 0        # 预测正确的样本个数
    total_acc_list = []      # 每隔 100 样本保存平均准确率

    # 7. 开始训练
    for epoch_idx in range(epochs):
        # 每个 epoch 重新实例化 DataLoader，shuffle=True 会重新打乱
        my_dataloader = DataLoader(dataset=my_dataset, batch_size=1, shuffle=True)
        # 内部迭代数据，逐 batch 送入模型
        for i, (x, y) in enumerate(tqdm(my_dataloader)):
            # x 形状: [1, seq_len, 57]，y 形状: [1]
            # 注意：x[0] 取出 batch 维度后变成 [seq_len, 57]
            # 但因为 batch_first=True，应直接送入 x 而非 x[0]
            # 原代码这里用 x[0]，可能是 batch_first=False 的写法，需根据模型设置调整
            output, hn = my_rnn(input=x[0], hidden=my_rnn.inithidden())
            # output 形状: [1, 18]

            # 8. 计算损失
            my_loss = my_nll_loss(output, y)

            # 9. 梯度清零（必须！否则梯度会累加）
            my_optim.zero_grad()
            # 10. 反向传播
            my_loss.backward()
            # 11. 梯度更新
            my_optim.step()

            # 12. 统计已训练样本总数
            total_iter_num += 1
            # 13. 统计累计损失
            total_loss += my_loss.item()
            # 14. 统计预测正确的样本数
            i_predict_num = 1 if torch.argmax(output).item() == y.item() else 0
            total_acc_num += i_predict_num

            # 每隔 100 次训练保存一次平均损失和准确率
            if total_iter_num % 100 == 0:
                avg_loss = total_loss / total_iter_num
                total_loss_list.append(avg_loss)
                avg_acc = total_acc_num / total_iter_num
                total_acc_list.append(avg_acc)
            # 每隔 2000 次训练打印一次日志
            if total_iter_num % 2000 == 0:
                temp_loss = total_loss / total_iter_num
                temp_acc = total_acc_num / total_iter_num
                temp_time = time.time() - start_time
                print('轮次:%d, 损失:%.6f, 时间:%d, 准确率:%.3f'
                      % (epoch_idx + 1, temp_loss, temp_time, temp_acc))
        # 每个 epoch 结束保存模型
        torch.save(my_rnn.state_dict(),
                   './save_model/ai20_rnn_%d.bin' % (epoch_idx + 1))

    # 15. 计算总耗时
    total_time = int(time.time() - start_time)
    print('训练总耗时：', total_time)
    # 16. 将结果保存到 JSON 文件
    dict1 = {"avg_loss": total_loss_list,
             "all_time": total_time,
             "avg_acc": total_acc_list}
    with open('./save_results/ai_rnn.json', 'w') as fw:
        fw.write(json.dumps(dict1))

    return total_loss_list, total_time, total_acc_list
```

### 进阶用法与原理

#### 损失函数 NLLLoss 解读

**NLLLoss（Negative Log Likelihood Loss，负对数似然损失）**：

```text
NLLLoss(output, target) = -output[target]
```

- `output`：模型经过 `LogSoftmax` 后的对数概率，形状 `[batch, num_classes]`
- `target`：真实类别的索引，形状 `[batch]`
- 计算方式：取出 `output` 中 `target` 索引位置的对数概率，取负号

由于 `LogSoftmax` 已经取过 log，NLLLoss 实际只做"取目标位置 + 取负"，等价于交叉熵损失。

#### 优化器 Adam 解读

**Adam（Adaptive Moment Estimation）** 结合了 Momentum（动量）和 RMSProp 的优点：

```text
m_t = β1 * m_(t-1) + (1 - β1) * g_t          # 一阶矩（动量）
v_t = β2 * v_(t-1) + (1 - β2) * g_t²         # 二阶矩（自适应学习率）
m_hat = m_t / (1 - β1^t)                      # 偏差修正
v_hat = v_t / (1 - β2^t)
θ_t = θ_(t-1) - lr * m_hat / (√v_hat + ε)     # 参数更新
```

- 默认参数：`lr=1e-3`、`β1=0.9`、`β2=0.999`、`ε=1e-8`
- 优点：自适应学习率，对超参数不敏感，收敛快
- 适用场景：大多数 NLP 任务的首选优化器

#### torch.topk 详解

```python
torch.topk(input, k, dim=None, largest=True, sorted=True)
```

| 参数 | 含义 | 本案例取值 |
|------|------|----------|
| `input` | 输入张量 | `output`，形状 `[1, 18]` |
| `k` | 取前 K 个 | `3` |
| `dim` | 在哪个维度比较 | `-1`（最后一个维度） |
| `largest` | `True` 取最大，`False` 取最小 | `True` |
| `sorted` | 是否按降序返回 | `True`（默认） |

**返回值**：命名元组 `(values, indices)`，分别为 Top-K 的值和对应的索引。

| `total_acc_list` | 平均准确率列表 | 绘制准确率曲线 |

### 避坑与局限

- **`x[0]` 与 `x` 的形状差异**：原训练代码中 `my_rnn(input=x[0], hidden=...)`，由于 `batch_size=1`，`x[0]` 会去掉 batch 维度，形状变为 `[seq_len, 57]`，但模型设了 `batch_first=True`，需要 `[1, seq_len, 57]`。这是一处潜在 bug，应直接传 `x` 或在 `x[0]` 后 `unsqueeze(0)`。
- **`save_model` 目录不存在**：训练前需 `os.makedirs('./save_model', exist_ok=True)`，否则 `torch.save` 会报错。
- **`load_state_dict` 路径**：预测时加载的模型路径必须与训练保存路径一致，且模型结构（参数名）必须完全一致。
- **`torch.no_grad()` 必须包裹推理过程**：否则会保留中间梯度，浪费内存且影响速度。
- **Top-K 索引取值**：`indexes[0][i]` 而非 `indexes[i]`，因为 indexes 形状为 `[1, 3]`（带 batch 维）。
- **类别列表顺序**：预测时 `categorys[index]` 必须用与训练时**完全相同的 `categorys` 列表**，否则索引与国家类别不对应。

## 3️⃣ 代码库

> [!warning] 以下完整代码依赖外部数据文件 `./data/name_classfication.txt`，需提前准备数据集才能运行。代码中使用 `MyRNN` 类名（与原笔记训练代码的 `My_RNN` 不同，已统一修正）。

```python
"""
RNN 人名分类器完整案例
依赖：torch, tqdm
数据：./data/name_classfication.txt
"""
import time
import json
import torch
import torch.nn as nn
import torch.optim as optim
from torch.utils.data import Dataset, DataLoader
from tqdm import tqdm


# ============ 全局常量 ============
import string
all_letters = string.ascii_letters + " .,;'"     # 所有可能字符
n_letter = len(all_letters)                       # 字符总数（约 57）
categorys = ['Chinese', 'Japanese', 'Korean', 'Vietnamese',
             'Russian', 'Arabic', 'Czech', 'German',
             'Dutch', 'Polish', 'Spanish', 'Greek',
             'Italian', 'Portuguese', 'French', 'English',
             'Scottish', 'Irish']                  # 18 个国家类别


# ============ 1. 数据预处理 ============
def read_data(filename):
    my_list_x, my_list_y = [], []
    with open(filename, 'r', encoding='utf-8') as fr:
        for line in fr.readlines():
            if len(line) <= 5:
                continue
            x, y = line.strip().split('\t')
            my_list_x.append(x)
            my_list_y.append(y)
    return my_list_x, my_list_y


class NameClassDataset(Dataset):
    def __init__(self, mylist_x, mylist_y):
        self.mylist_x = mylist_x
        self.mylist_y = mylist_y
        self.sample_len = len(mylist_x)

    def __len__(self):
        return self.sample_len

    def __getitem__(self, index):
        index = min(max(index, 0), self.sample_len - 1)
        x = self.mylist_x[index]
        y = self.mylist_y[index]
        tensor_x = torch.zeros(len(x), n_letter)
        for li, letter in enumerate(x):
            tensor_x[li][all_letters.find(letter)] = 1
        tensor_y = torch.tensor(categorys.index(y), dtype=torch.long)
        return tensor_x, tensor_y


def get_dataloader(filepath='./data/name_classfication.txt', batch_size=1):
    my_list_x, my_list_y = read_data(filepath)
    my_dataset = NameClassDataset(my_list_x, my_list_y)
    return DataLoader(dataset=my_dataset, batch_size=batch_size, shuffle=True)


# ============ 2. 模型搭建 ============
class MyRNN(nn.Module):
    def __init__(self, input_size, hidden_size, ouput_size, num_layers=1):
        super().__init__()
        self.input_size = input_size
        self.hidden_size = hidden_size
        self.ouput_size = ouput_size
        self.num_layers = num_layers
        self.rnn = nn.RNN(self.input_size, self.hidden_size,
                          num_layers=self.num_layers, batch_first=True)
        self.linear = nn.Linear(self.hidden_size, self.ouput_size)
        self.softmax = nn.LogSoftmax(dim=-1)

    def forward(self, input, hidden):
        rnn_output, rnn_hn = self.rnn(input, hidden)
        tmep = rnn_output[0][-1].unsqueeze(0)
        output = self.linear(tmep)
        return self.softmax(output), rnn_hn

    def inithidden(self):
        return torch.zeros(self.num_layers, 1, self.hidden_size)


# ============ 3. 模型训练 ============
def train_rnn():
    my_dataset = NameClassDataset(*read_data('./data/name_classfication.txt'))
    my_rnn = MyRNN(input_size=57, hidden_size=128, output_size=18)
    my_nll_loss = nn.NLLLoss()
    my_optim = optim.Adam(my_rnn.parameters(), lr=1e-3)
    start_time = time.time()
    total_iter_num = 0
    total_loss = 0
    total_acc_num = 0
    total_loss_list, total_acc_list = [], []

    for epoch_idx in range(1):
        my_dataloader = DataLoader(dataset=my_dataset, batch_size=1, shuffle=True)
        for i, (x, y) in enumerate(tqdm(my_dataloader)):
            # batch_first=True 时直接传 x（而非 x[0]）
            output, hn = my_rnn(input=x, hidden=my_rnn.inithidden())
            my_loss = my_nll_loss(output, y)
            my_optim.zero_grad()
            my_loss.backward()
            my_optim.step()
            total_iter_num += 1
            total_loss += my_loss.item()
            total_acc_num += 1 if torch.argmax(output).item() == y.item() else 0
            if total_iter_num % 100 == 0:
                total_loss_list.append(total_loss / total_iter_num)
                total_acc_list.append(total_acc_num / total_iter_num)
            if total_iter_num % 2000 == 0:
                print('轮次:%d, 损失:%.6f, 准确率:%.3f'
                      % (epoch_idx + 1,
                         total_loss / total_iter_num,
                         total_acc_num / total_iter_num))
        torch.save(my_rnn.state_dict(), './save_model/ai20_rnn_%d.bin' % (epoch_idx + 1))
    print('训练总耗时：', int(time.time() - start_time))


# ============ 4. 模型预测 ============
def line2tensor(x):
    tensor_x = torch.zeros(len(x), n_letter)
    for li, letter in enumerate(x):
        tensor_x[li][all_letters.find(letter)] = 1
    return tensor_x


def rnn_predict(x):
    tensor_x = line2tensor(x).unsqueeze(0)   # [1, seq_len, 57]
    my_rnn = MyRNN(input_size=57, hidden_size=128, output_size=18)
    my_rnn.load_state_dict(torch.load('./save_model/ai20_rnn_1.bin'))
    with torch.no_grad():
        output, hn = my_rnn(tensor_x, my_rnn.inithidden())
        values, indexes = torch.topk(output, k=3, dim=-1, largest=True)
        for i in range(3):
            print(f'Top-{i+1}: 值 {values[0][i]:.4f}, 国家 {categorys[indexes[0][i]]}')


if __name__ == '__main__':
    # 取消下方注释即可运行
    # train_rnn()
    # rnn_predict("Ang")
    pass
```

## 4️⃣ 避坑指南 & 易错对比

> [!warning] 核心理念
> - 本案例的关键不在模型多复杂，而在**张量形状的精确控制**。每一处维度变换都要心里有数。
> - `batch_first` 是 PyTorch RNN 系列最容易踩坑的参数，必须在 API 实例化、forward 调用、形状判断三处保持一致。
> - 训练循环中的 `optimizer.zero_grad()` 不可省略，否则梯度会跨 batch 累加，导致训练发散。

### 易混概念对比表

| 对比维度 | `MyRNN` | `MyLSTM` | `MyGRU` |
|---------|---------|----------|---------|
| 网络层 | `nn.RNN` | `nn.LSTM` | `nn.GRU` |
| 初始状态 | `h0` | `(h0, c0)` | `h0` |
| forward 参数 | `(input, hidden)` | `(input, hidden, c0)` | `(input, hidden)` |
| forward 返回 | `(output, hn)` | `(output, hn, cn)` | `(output, hn)` |
| `inithidden` 返回 | `h0` | `(h0, c0)` | `h0` |
| 训练时每个样本耗时 | 最短 | 最长 | 中等 |

| 对比维度 | `LogSoftmax + NLLLoss` | `CrossEntropyLoss` |
|---------|------------------------|---------------------|
| 模型末尾 | 需加 `LogSoftmax` | 不加 `LogSoftmax` |
| 损失函数 | `nn.NLLLoss()` | `nn.CrossEntropyLoss()` |
| 数学等价性 | 等价 | 等价 |
| 工程实践 | 教学/调试 | 工业主流 |

| 对比维度 | `batch_first=False` | `batch_first=True` |
|---------|---------------------|---------------------|
| 输入形状 | `(seq_len, batch, input_size)` | `(batch, seq_len, input_size)` |
| 是否匹配 DataLoader | 不匹配，需转置 | 直接匹配 |
| 性能 | 略快（CuDNN 优化） | 略慢（新版本差距小） |
| 教学推荐 | - | 推荐 |

> [!danger] 常见错误与规避
> - **错误 1**：DataLoader 返回 `x` 形状为 `[1, seq, 57]`，但传入 `x[0]`（形状 `[seq, 57]`）给 `batch_first=True` 的 RNN，导致形状不匹配报错。
>   - 规避：`batch_first=True` 时直接传 `x`；若一定要传 `x[0]`，则需 `x[0].unsqueeze(0)` 恢复 batch 维度。
> - **错误 2**：LSTM 的 `forward` 返回值解包写成 `output, hn = lstm(...)`，但实际返回是 `(output, (hn, cn))`，导致 `hn` 是元组而非张量。
>   - 规避：写 `output, (hn, cn) = lstm(...)`。
> - **错误 3**：训练时未调用 `optimizer.zero_grad()`，梯度累加导致 loss 不降反升。
>   - 规避：每个 batch 反向传播前必调 `optimizer.zero_grad()`。
> - **错误 4**：预测时忘记 `torch.no_grad()`，导致显存爆炸或速度变慢。
>   - 规避：推理代码包裹在 `with torch.no_grad():` 中。
> - **错误 5**：`torch.topk` 返回 `(values, indexes)`，取值时写成 `values[i]` 而非 `values[0][i]`，导致索引错误。
>   - 规避：`values` 形状为 `[1, 3]`（带 batch 维），必须用 `values[0][i]`。
> - **错误 6**：训练时 `categorys` 列表与预测时 `categorys` 顺序不一致，导致索引对应的类别错误。
>   - 规避：将 `categorys` 定义为全局常量，训练与预测共用。
