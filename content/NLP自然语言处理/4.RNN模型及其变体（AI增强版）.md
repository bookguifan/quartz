# [day04] 学习笔记｜RNN模型及其变体（AI 增强版）

## 📌 核心速览

> [!summary] 核心速览
> - **循环神经网络（RNN）**：当前时间步输出由当前输入与上一时间步隐状态共同决定，擅长处理序列数据。
> - **RNN 四种结构**：N vs N（等长序列）、N vs 1（分类）、1 vs N（生成）、N vs M（seq2seq，翻译）。
> - **传统 RNN**：结构简单、计算快，但长序列易梯度消失/爆炸，适合短文本任务。
> - **LSTM 三门一态**：遗忘门、输入门、输出门 + 细胞状态 C_t，通过门控缓解长依赖问题。
> - **GRU 两门**：重置门 + 更新门，参数比 LSTM 少 1/3，效果相当，训练更快。
> - **Bi-LSTM/Bi-GRU**：双向结构，正向反向各跑一次再拼接，捕获双向上下文信息。
> - **PyTorch API**：`nn.RNN` / `nn.LSTM` / `nn.GRU` 三者参数一致，核心是 `input_size`、`hidden_size`、`num_layers`、`batch_first`。
> - **形状约定**：默认 `batch_first=False`，输入为 `(seq_len, batch_size, input_size)`；设为 `True` 时变为 `(batch_size, seq_len, input_size)`。

## 1️⃣ 完整知识库

## RNN 总览：定义与分类 🔹 基础

### 定义与本质

**循环神经网络（Recurrent Neural Network, RNN）** 是一种专门处理序列数据的神经网络。其核心思想是：当前时间步的输出不仅依赖当前输入，还依赖上一时间步的隐状态（hidden state），从而在时间维度上"记忆"历史信息。

公式表达：

```text
h_t = tanh(W_xh · x_t + W_hh · h_(t-1) + b_h)
```

- `x_t`：当前时间步输入
- `h_(t-1)`：上一时间步的隐状态
- `h_t`：当前时间步的隐状态
- `W_xh`、`W_hh`、`b_h`：权重与偏置参数

### 基础用法

RNN 在 NLP 中应用广泛，主要包括：文本翻译、文本分类、情感分析、命名实体识别、图片描述生成等。

**根据输入与输出结构，RNN 可分为四类**：

| 类型 | 输入 | 输出 | 特点 | 典型应用 |
|------|------|------|------|---------|
| N vs N | 长度 N | 长度 N | 输入输出等长 | 诗歌生成、对联、词性标注 |
| N vs 1 | 长度 N | 1 个 | 输出唯一 | 文本分类、情感分析 |
| 1 vs N | 1 个 | 长度 N | 输入唯一 | 图片描述生成、音乐生成 |
| N vs M | 长度 N | 长度 M | 输入输出不等长（seq2seq） | 机器翻译、文本摘要 |

**根据内部结构，RNN 可分为**：传统 RNN、LSTM、Bi-LSTM、GRU、Bi-GRU。本笔记聚焦传统 RNN、LSTM、GRU 三大主流结构。

> [!note] 💡 AI 扩展（基础）
> **RNN 四种输入输出结构的应用场景对比**：
>
> | 类型 | 结构特点 | 典型 NLP 应用 | 补充说明 |
> |------|---------|-------------|---------|
> | **N-to-N** | 每个时间步都有输入和输出，等长 | 词性标注、字符级文本生成 | 输入和输出序列一一对应 |
> | **N-to-1** | 多步输入，最后一步输出 | 文本分类、情感分析、意图识别 | 只取最后时间步的 `h_T` 作为句子表示 |
> | **1-to-N** | 单步输入，展开为多步输出 | 图片描述生成、音乐生成 | 固定输入经 RNN 展开生成长序列 |
> | **N-to-M**（seq2seq） | 编码器-解码器架构，不等长 | 机器翻译、文本摘要、对话系统 | 编码器压缩为上下文向量，解码器生成目标序列 |
>
> 选型关键：先确定"输入几步、输出几步"，再匹配合适的 RNN 结构。

### 避坑与局限

- RNN 的"记忆"是隐式的（隐状态），并非真正的长期记忆，长序列中早期信息会被逐步稀释。
- 选择结构时应优先考虑序列长度：短文本（如对联）→ RNN；中长文本（如分类）→ LSTM/GRU；超长序列（如机器翻译）→ LSTM/GRU + 注意力机制。

## 传统 RNN 模型 🔸 核心

### 定义与本质

传统 RNN（Vanilla RNN）是最朴素的循环神经网络。其内部结构的核心是：在每一时间步，将当前输入 `x_t` 与上一时间步的隐状态 `h_(t-1)` 拼接后，经过一个线性层和 `tanh` 激活函数，得到新的隐状态 `h_t`。

![[day04_rnn_structure.png]]

**结构图解读**：
- 图中红色矩形为输入 `x_t` 当前时间步的输入
- 蓝色矩形为隐状态 `h_t`
- 黄色矩形为输出 `o_t`（可选，由 `h_t` 经过线性层得到）
- U、W、V 为对应的权重矩阵

### 基础用法

#### PyTorch API：`nn.RNN`

```python
nn.RNN(input_size, hidden_size, num_layers=1,
       nonlinearity='tanh', bias=True, batch_first=False,
       dropout=0, bidirectional=False)
```

| 参数 | 含义 | 典型值 |
|------|------|--------|
| `input_size` | 输入特征维度（词向量维度） | 5 / 57 / 128 |
| `hidden_size` | 隐藏层神经元个数 | 6 / 128 / 256 |
| `num_layers` | 隐藏层层数（堆叠 RNN） | 1 / 2 / 3 |
| `nonlinearity` | 激活函数 | `'tanh'`（默认）/ `'relu'` |
| `batch_first` | 输入第 0 维是否为 batch_size | `True`（推荐） |
| `bidirectional` | 是否双向 | `False`（Bi-RNN 时设 `True`） |

#### 输入输出形状

```text
input  形状: (seq_len, batch_size, input_size)     # batch_first=False 时
h0     形状: (num_layers * num_directions, batch_size, hidden_size)
output 形状: (seq_len, batch_size, hidden_size * num_directions)
hn     形状: (num_layers * num_directions, batch_size, hidden_size)
```

#### 多层 RNN 结构

`num_layers > 1` 时，下一层的输入是上一层的输出：

![[day04_rnn_multilayer.png]]

#### 完整代码示例

```python
import torch
import torch.nn as nn

def dm_rnn_base():
    # input_size: 输入张量 x 的维度（词向量维度）
    # hidden_size: 隐藏层神经元个数
    # num_layers: 隐藏层数量，默认为 1
    rnn = nn.RNN(input_size=5, hidden_size=6, num_layers=1)

    # input 三参数含义：
    #   seq_len: 输入序列长度（一个句子的词或字符个数）= 4，对应 "what time is it"
    #   batch_size: 批次样本数量 = 3
    #   input_size: 输入张量 x 的维度 = 5
    input = torch.randn(4, 3, 5)

    # h0 三参数含义：
    #   num_layers * num_directions: 默认 1
    #   batch_size: 3
    #   hidden_size: 6
    h0 = torch.randn(1, 3, 6)

    # 方式一：一次性输入整个序列
    output, hn = rnn(input, h0)
    print(f'一次性输入 output 形状: {output.shape}, hn 形状: {hn.shape}')
    # 输出：output 形状: torch.Size([4, 3, 6]), hn 形状: torch.Size([1, 3, 6])

    # 方式二：逐时间步输入（适合实时场景或 teacher forcing）
    h = h0
    s_len = input.shape[0]
    for idx in range(s_len):
        tmp = input[idx].unsqueeze(0)  # (1, batch_size, input_size)
        output, h = rnn(tmp, h)
        print(f'第 {idx+1} 步 output 形状: {output.shape}, hn 形状: {h.shape}')
        # 输出：第 1 步 output 形状: torch.Size([1, 3, 6]), hn 形状: torch.Size([1, 3, 6])
        # ...
```

> [!note] 💡 AI 扩展（基础）
> **两种输入方式的本质差异**：
> - **一次性输入**：PyTorch 内部自动按时间步展开 RNN，效率高，但无法在中途干预（如 teacher forcing 或 beam search）。
> - **逐时间步输入**：每次只喂一个时间步，可在每步根据需要修改下一输入（teacher forcing）或下一隐状态（如注意力重写），更灵活但更慢。
> **实战建议**：训练阶段如果不需要干预，优先用一次性输入；解码阶段（如 seq2seq 生成）必须逐时间步输入。
> **性能提示**：`batch_first=False` 时 RNN 内部走 CuDNN 优化路径，比 `batch_first=True` 略快；但 `batch_first=True` 更易理解，PyTorch 较新版本中两者性能差距已很小。

### 进阶用法与原理

#### RNN 的反向传播：BPTT

RNN 的训练使用**沿时间反向传播（Backpropagation Through Time, BPTT）**。将 RNN 沿时间维度展开后，所有时间步共享同一组权重，梯度需要对所有时间步累乘：

```text
∂L/∂W = Σ (t=1..T) ∂L_t/∂W
```

#### 梯度消失与梯度爆炸的成因

在反向传播时，由于权重共享，梯度需要连乘多次雅可比矩阵：

```text
∂h_t/∂h_0 = Π (i=1..t) W_hh · diag(1 - tanh²(...))
```

- 当 `|W_hh|` 特征值 > 1 → 多次连乘后梯度指数级放大 → **梯度爆炸**
- 当 `|W_hh|` 特征值 < 1 → 多次连乘后梯度指数级衰减 → **梯度消失**

`tanh` 的导数最大为 1，且大部分时候小于 1，进一步加剧梯度消失问题，导致 RNN 难以学习长距离依赖（通常 > 10 步就明显退化）。

#### 梯度爆炸的缓解：梯度裁剪

```python
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=5)
```

将梯度的整体范数裁剪到 `max_norm` 以内，避免爆炸。但梯度消失无法用此方法解决，需依赖 LSTM/GRU 的门控结构。

### 避坑与局限

- **优点**：计算资源占用小，网络结构简单，对短文本表现优异。
- **缺点**：处理长文本时容易梯度消失或梯度爆炸，难以捕捉长距离依赖。
- **使用建议**：序列长度 < 10 时可考虑传统 RNN；更长序列应使用 LSTM 或 GRU。
- **常见坑**：`h0` 形状必须为 `(num_layers * num_directions, batch_size, hidden_size)`，多设或漏设都会报形状错误。

## LSTM 模型 🔸 核心

### 定义与本质

**长短期记忆网络（Long Short-Term Memory, LSTM）** 由 Hochreiter 和 Schmidhuber 于 1997 年提出，通过引入**细胞状态（cell state）** 和三个**门控机制**，有效缓解了传统 RNN 的梯度消失问题。

LSTM 的核心创新是：
- **细胞状态 C_t**：一条贯穿整个序列的"传送带"，信息增删受门控调节。
- **三个门**：遗忘门、输入门、输出门，每个门都是一个 `sigmoid` 层（输出 0~1），控制信息流的通过比例。

![[day04_lstm_structure.png]]

### 基础用法

#### LSTM 三门一态详解

下表为 LSTM 每个门的作用和数学公式（含变量说明）：

| 名称                   | 作用                          | 数学公式                                      | 变量说明                             |
| -------------------- | --------------------------- | ----------------------------------------- | -------------------------------- |
| **遗忘门（Forget Gate）** | 决定从上一细胞状态 `C_(t-1)` 中丢弃多少信息 | `f_t = σ(W_f · [h_(t-1), x_t] + b_f)`     | `σ`：sigmoid 函数；`W_f`：权重；`b_f`：偏置 |
| **输入门（Input Gate）**  | 决定当前候选信息 `C̃_t` 中有多少写入细胞状态  | `i_t = σ(W_i · [h_(t-1), x_t] + b_i)`     | `i_t`：输入门的开关量                    |
| **候选细胞状态**           | 当前时间步生成的候选信息                | `C̃_t = tanh(W_c · [h_(t-1), x_t] + b_c)` | `C̃_t`：候选新信息                     |
| **细胞状态更新**           | 综合遗忘门和输入门更新细胞状态             | `C_t = f_t * C_(t-1) + i_t * C̃_t`        | `*`：逐元素相乘                        |
| **输出门（Output Gate）** | 决定细胞状态中多少信息作为隐状态输出          | `o_t = σ(W_o · [h_(t-1), x_t] + b_o)`     | `o_t`：输出门的开关量                    |
| **隐状态更新**            | 由细胞状态经 tanh 和输出门共同决定        | `h_t = o_t * tanh(C_t)`                   | `h_t`：当前隐状态，传入下一时间步              |
|                      |                             |                                           |                                  |
$$
\begin{align}
\boldsymbol{f}_t &= \sigma( W_f [\boldsymbol{h}_{t-1}, \boldsymbol{x}_t] + \boldsymbol{b}_f ) \\
\boldsymbol{i}_t &= \sigma( W_i [\boldsymbol{h}_{t-1}, \boldsymbol{x}_t] + \boldsymbol{b}_i ) \\
\boldsymbol{o}_t &= \sigma( W_o [\boldsymbol{h}_{t-1}, \boldsymbol{x}_t] + \boldsymbol{b}_o ) \\
\tilde{\boldsymbol{C}}_t &= \tanh( W_C [\boldsymbol{h}_{t-1}, \boldsymbol{x}_t] + \boldsymbol{b}_C ) \\
\boldsymbol{C}_t &= \boldsymbol{f}_t \cdot \boldsymbol{C}_{t-1} + \boldsymbol{i}_t \cdot \tilde{\boldsymbol{C}}_t \\
\boldsymbol{h}_t &= \boldsymbol{o}_t \cdot \tanh( \boldsymbol{C}_t )
\end{align}
$$
> [!note] 💡 AI 扩展（进阶）
> **为什么 LSTM 能缓解梯度消失？**
> - 关键在细胞状态 `C_t = f_t * C_(t-1) + i_t * C̃_t` 这条公式：`C_t` 对 `C_(t-1)` 的梯度为 `f_t`，**不再连乘 W_hh**。
> - 如果 `f_t ≈ 1`（遗忘门保持打开），梯度可以直接从 `C_t` 流回 `C_0`，几乎不衰减，这条路径被称为"常数误差流（Constant Error Carousel, CEC）"。
> - 门控的开关量是可学习的，模型可以自主决定何时记忆、何时遗忘。
> **性能实践**：
> - LSTM 参数量约为 RNN 的 4 倍（4 个线性层：3 门 + 1 候选状态），计算量也约为 4 倍。
> - 工业界常用技巧：将 `input_size` 较大的输入先用一层线性投影降维再进 LSTM，可大幅减少参数量。
> - `dropout` 参数只作用于多层 LSTM 的层间，**不作用于同一层的不同时间步**，否则会破坏序列记忆。

#### PyTorch API：`nn.LSTM`

```python
nn.LSTM(input_size, hidden_size, num_layers=1,
        bias=True, batch_first=False, dropout=0, bidirectional=False)
```

与 `nn.RNN` API 形参完全一致，但 `forward` 需要同时传入 `(h0, c0)`：

```python
lstm = nn.LSTM(input_size=5, hidden_size=6, num_layers=1)
input = torch.randn(8, 3, 5)   # (seq_len=8, batch=3, input_size=5)
h0 = torch.randn(1, 3, 6)      # (num_layers, batch, hidden_size)
c0 = torch.randn(1, 3, 6)      # (num_layers, batch, hidden_size)

output, (hn, cn) = lstm(input, (h0, c0))
print(f'output 形状: {output.shape}, hn 形状: {hn.shape}, cn 形状: {cn.shape}')
# 输出：output 形状: torch.Size([8, 3, 6]), hn 形状: torch.Size([1, 3, 6]), cn 形状: torch.Size([1, 3, 6])
```

#### 完整代码实现

```python
import torch
import torch.nn as nn

def dm_test_lstm():
    # 实例化 LSTM 对象
    #   input_size: 输入数据维度 = 5
    #   hidden_size: 隐藏层维度 = 6
    #   num_layers: 隐藏层层数 = 1
    lstm = nn.LSTM(5, 6, 1)

    # 输入张量：(seq_len=8, batch_size=3, input_size=5)
    input = torch.randn(8, 3, 5)

    # 初始隐状态和细胞状态
    h0 = torch.randn(1, 3, 6)   # (num_layers, batch_size, hidden_size)
    c0 = torch.randn(1, 3, 6)

    # 前向传播：注意返回值为 output, (hn, cn)
    output, (hn, cn) = lstm(input, (h0, c0))

    print(f'output 形状: {output.shape}')
    print(f'hn 形状: {hn.shape}')
    print(f'cn 形状: {cn.shape}')
    # 输出：output 形状: torch.Size([8, 3, 6])
    #       hn 形状: torch.Size([1, 3, 6])
    #       cn 形状: torch.Size([1, 3, 6])
```

### 进阶用法与原理

#### Bi-LSTM（双向 LSTM）

**Bi-LSTM** 是将同一个 LSTM 沿两个方向各跑一次：正向（从 t=1 到 t=T）和反向（从 t=T 到 t=1），然后将两个方向的隐状态**拼接**作为最终输出。

```text
h_t = [h_t_forward ; h_t_backward]    # 拼接后维度为 2 * hidden_size
```

- 本质：内部网络结构未变，只是把模型应用了两次且方向不同，最后拼接两次的输出。
- 优点：能同时利用过去和未来的上下文信息（如分词、命名实体识别等任务）。
- 限制：要求整个序列一次性可见，**不能用于在线流式预测**。

PyTorch 中只需 `bidirectional=True`：

```python
bilstm = nn.LSTM(input_size=5, hidden_size=6, num_layers=1, bidirectional=True)
# 输出维度变为 6 * 2 = 12
```

### 避坑与局限

- **优点**：长序列问题中能有效缓解梯度消失或梯度爆炸，相比传统 RNN 在长序列上表现更好。
- **缺点**：内部结构复杂，相同算力下相比传统 RNN 计算效率低下。
- **常见坑**：
  - `forward` 返回的是元组 `(hn, cn)`，与 `nn.RNN` 返回单个 `hn` 不同，解包时容易出错。
  - 双向 LSTM 输出维度会翻倍，下游 Linear 层的 `in_features` 需同步调整为 `2 * hidden_size`。
  - `c0` 必须与 `h0` 形状一致，遗漏会导致报错。

## GRU 与双向变体 🔸 核心

### 定义与本质

**门控循环单元（Gated Recurrent Unit, GRU）** 由 Cho 等人于 2014 年提出，是 LSTM 的简化版。GRU 将 LSTM 的"遗忘门 + 输入门"合并为一个**更新门**，并合并了细胞状态和隐状态，结构更精简。

GRU 内部包含两个门：
- **重置门（Reset Gate, r_t）**：控制上一隐状态 `h_(t-1)` 中有多少信息参与候选状态的计算，用于"遗忘"过去。
- **更新门（Update Gate, z_t）**：决定上一隐状态 `h_(t-1)` 与候选状态 `h̃_t` 的混合比例，相当于 LSTM 的遗忘门 + 输入门。

> [!note] 💡 AI 扩展（进阶）
> **LSTM 与 GRU 的对比**：
>
> | 维度 | LSTM | GRU |
> |------|------|-----|
> | 门数量 | 3 门 + 候选状态 = 4 个线性层 | 2 门 + 候选状态 = 3 个线性层 |
> | 状态变量 | 隐状态 h_t + 细胞状态 C_t | 仅有隐状态 h_t |
> | 参数量 | 约 4 × (input_size + hidden_size) × hidden_size | 约 3 × (input_size + hidden_size) × hidden_size |
> | 训练速度 | 较慢 | 快约 1/3 |
> | 长依赖效果 | 略优 | 与 LSTM 相当 |
> | 适用场景 | 长序列、需精细控制记忆 | 数据量较少、计算资源受限 |
> **性能实践**：
> - 工业实践中，LSTM 与 GRU 在大多数 NLP 任务上效果相近，GRU 更快、参数更少。
> - 数据量较小时优先尝试 GRU；数据量充足且追求极限精度时尝试 LSTM。
> - 若 LSTM 与 GRU 都不达预期，应转向 **Transformer（自注意力）** 架构。

### 基础用法

#### GRU 数学公式

下表为 GRU 的完整前向计算公式：

| 名称 | 数学公式 | 变量说明 |
|------|---------|---------|
| **重置门** | `r_t = σ(W_r · [h_(t-1), x_t] + b_r)` | `r_t`：重置门开关量，决定保留多少历史信息 |
| **更新门** | `z_t = σ(W_z · [h_(t-1), x_t] + b_z)` | `z_t`：更新门开关量，决定新旧状态混合比例 |
| **候选隐状态** | `h̃_t = tanh(W_h · [r_t * h_(t-1), x_t] + b_h)` | `*`：逐元素相乘；`r_t * h_(t-1)` 表示按需遗忘后的历史 |
| **隐状态更新** | `h_t = (1 - z_t) * h_(t-1) + z_t * h̃_t` | 通过 `z_t` 在新旧状态间做插值 |

**对比 LSTM 关键差异**：
- LSTM 用 `f_t` 和 `i_t` 两个独立门分别控制"遗忘"与"写入"；GRU 用 `z_t` 一个门统一控制：`1 - z_t` 决定保留旧状态的比例，`z_t` 决定写入新信息的比例，二者之和为 1（互斥）。
- GRU 没有"细胞状态"这一独立通道，直接对隐状态做插值更新。

#### PyTorch API：`nn.GRU`

```python
nn.GRU(input_size, hidden_size, num_layers=1,
       bias=True, batch_first=False, dropout=0, bidirectional=False)
```

与 `nn.RNN` API 完全一致，`forward` 也只返回 `(output, hn)`，**不需要 c0**：

```python
gru = nn.GRU(input_size=5, hidden_size=6, num_layers=1)
input = torch.randn(8, 3, 5)
h0 = torch.randn(1, 3, 6)

output, hn = gru(input, h0)
print(f'output 形状: {output.shape}, hn 形状: {hn.shape}')
# 输出：output 形状: torch.Size([8, 3, 6]), hn 形状: torch.Size([1, 3, 6])
```

#### 三种 RNN 变体的 API 对比

| API | 输入 | 初始状态 | 返回值 | 状态变量数 |
|-----|------|---------|--------|-----------|
| `nn.RNN` | `(seq_len, batch, input_size)` | `h0` | `(output, hn)` | 1（h） |
| `nn.LSTM` | `(seq_len, batch, input_size)` | `(h0, c0)` | `(output, (hn, cn))` | 2（h, c） |
| `nn.GRU` | `(seq_len, batch, input_size)` | `h0` | `(output, hn)` | 1（h） |

### 进阶用法与原理

#### Bi-GRU（双向 GRU）

与 Bi-LSTM 类似，将 GRU 沿两个方向各跑一次再拼接：

```python
bigru = nn.GRU(input_size=5, hidden_size=6, num_layers=1, bidirectional=True)
# 输出维度变为 6 * 2 = 12
```

#### 选型决策树

```text
1. 序列长度 < 10 且追求极致速度？        → 传统 RNN
2. 是否需要双向上下文（NER、分词）？     → Bi-LSTM 或 Bi-GRU
3. 数据量较少或算力受限？                → GRU（参数少 1/3）
4. 数据量充足、追求极限精度？            → LSTM
5. 序列超长且依赖全局对齐？              → LSTM + Attention 或 Transformer
```

### 避坑与局限

- **优点**：结构比 LSTM 简单，参数更少，训练更快，效果通常与 LSTM 相当。
- **缺点**：在超长序列上略逊于 LSTM，因无独立的细胞状态通道，长依赖信息更易被覆盖。
- **常见坑**：
  - GRU 的 `forward` 与 `nn.RNN` 完全相同，**返回 `(output, hn)` 而非 `(output, (hn, cn))`**，迁移代码时容易混淆。
  - 重置门和更新门都是 sigmoid 输出，初始训练时若 `z_t` 始终接近 0，模型只会复用旧状态，难以学到新信息；可通过减小初始化方差缓解。

## 3️⃣ 代码库

```python
"""
RNN/LSTM/GRU 三大循环神经网络对比示例
依赖：torch >= 1.8
"""
import torch
import torch.nn as nn


def demo_rnn():
    """传统 RNN：只返回 (output, hn)，状态仅 h"""
    rnn = nn.RNN(input_size=5, hidden_size=6, num_layers=1, batch_first=True)
    # batch_first=True 时输入形状为 (batch_size, seq_len, input_size)
    x = torch.randn(3, 8, 5)         # (batch=3, seq_len=8, input_size=5)
    h0 = torch.randn(1, 3, 6)        # (num_layers, batch, hidden_size)
    output, hn = rnn(x, h0)
    print(f'[RNN]  output: {output.shape}, hn: {hn.shape}')
    # 输出：[RNN]  output: torch.Size([3, 8, 6]), hn: torch.Size([1, 3, 6])


def demo_lstm():
    """LSTM：返回 (output, (hn, cn))，状态有 h 和 c"""
    lstm = nn.LSTM(input_size=5, hidden_size=6, num_layers=1, batch_first=True)
    x = torch.randn(3, 8, 5)
    h0 = torch.randn(1, 3, 6)
    c0 = torch.randn(1, 3, 6)
    output, (hn, cn) = lstm(x, (h0, c0))
    print(f'[LSTM] output: {output.shape}, hn: {hn.shape}, cn: {cn.shape}')
    # 输出：[LSTM] output: torch.Size([3, 8, 6]), hn: torch.Size([1, 3, 6]), cn: torch.Size([1, 3, 6])


def demo_gru():
    """GRU：返回 (output, hn)，状态仅 h（与 RNN 相同）"""
    gru = nn.GRU(input_size=5, hidden_size=6, num_layers=1, batch_first=True)
    x = torch.randn(3, 8, 5)
    h0 = torch.randn(1, 3, 6)
    output, hn = gru(x, h0)
    print(f'[GRU]  output: {output.shape}, hn: {hn.shape}')
    # 输出：[GRU]  output: torch.Size([3, 8, 6]), hn: torch.Size([1, 3, 6])


def demo_bilstm():
    """Bi-LSTM：bidirectional=True，输出维度翻倍"""
    bilstm = nn.LSTM(input_size=5, hidden_size=6, num_layers=1,
                     batch_first=True, bidirectional=True)
    x = torch.randn(3, 8, 5)
    # 双向时 num_layers * num_directions = 1 * 2 = 2
    h0 = torch.randn(2, 3, 6)
    c0 = torch.randn(2, 3, 6)
    output, (hn, cn) = bilstm(x, (h0, c0))
    print(f'[BiLSTM] output: {output.shape}, hn: {hn.shape}')
    # 输出：[BiLSTM] output: torch.Size([3, 8, 12]), hn: torch.Size([2, 3, 6])


if __name__ == '__main__':
    demo_rnn()
    demo_lstm()
    demo_gru()
    demo_bilstm()
```

## 4️⃣ 避坑指南 & 易错对比

> [!warning] 核心理念
> - RNN 系列模型共享权重，所有时间步用同一组参数，这是它能处理变长序列的根本。
> - LSTM/GRU 的"门"本质上是一个 sigmoid 层，输出 0~1 的开关量，而非硬开关。
> - **batch_first 的隐式陷阱**：`nn.RNN/LSTM/GRU` 默认 `batch_first=False`，但 DataLoader 返回的第一维是 batch_size，二者不匹配时需要手动转置或设置 `batch_first=True`。

### 易混概念对比表

| 对比维度 | 传统 RNN | LSTM | GRU |
|---------|---------|------|-----|
| 提出年份 | 1986 | 1997 | 2014 |
| 门数量 | 0 | 3（遗忘/输入/输出） | 2（重置/更新） |
| 状态变量 | 仅 h | h + C | 仅 h |
| 参数量（单层） | 1× | 4× | 3× |
| 梯度缓解 | 无 | 通过 CEC 缓解 | 通过更新门缓解 |
| 训练速度 | 最快 | 最慢 | 中等 |
| 长序列表现 | 差 | 优 | 优 |
| 适用场景 | 短序列 | 长序列、精细控制 | 数据少、算力紧 |

| 对比维度 | `nn.RNN` API | `nn.LSTM` API | `nn.GRU` API |
|---------|-------------|---------------|--------------|
| 初始状态 | `h0` | `(h0, c0)` | `h0` |
| 返回值 | `(output, hn)` | `(output, (hn, cn))` | `(output, hn)` |
| 与 RNN 兼容性 | - | 不兼容（需改 unpack） | 兼容（结构一致） |

> [!danger] 常见错误与规避
> - **错误 1**：把 `nn.LSTM` 的返回值当成 `(output, hn)` 解包，导致 `hn` 实际是元组 `(hn, cn)`，下游调用 `hn.shape` 报错。
>   - 规避：写 `output, (hn, cn) = lstm(...)`。
> - **错误 2**：`h0` 形状写成 `(batch_size, hidden_size)`，少了 `num_layers` 维度。
>   - 规避：始终三维 `(num_layers * num_directions, batch_size, hidden_size)`。
> - **错误 3**：双层 LSTM 用 `num_layers=2` 但 `h0` 第一维仍为 1。
>   - 规避：`h0` 第一维 = `num_layers * num_directions`。
> - **错误 4**：双向 LSTM 输出维度未翻倍处理，下游 Linear 层报形状不匹配。
>   - 规避：`in_features = 2 * hidden_size`。
> - **错误 5**：训练时忘记 `optimizer.zero_grad()`，导致梯度累加，loss 不降反升。
>   - 规避：每个 batch 反向传播前调用 `optimizer.zero_grad()`。
