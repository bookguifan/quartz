# [day13] 学习笔记｜中文文本分类案例（AI 增强版）

## 📌 核心速览

> [!summary] 核心速览
> - **任务目标**：用 BERT 预训练模型实现中文新闻文本分类（10 类），属于迁移学习的"微调"方式。
> - **数据预处理**：用 HuggingFace `load_dataset` 加载 CSV → 自定义 `collate_fn` 完成分词与 padding → `DataLoader` 封装。
> - **模型搭建**：`AiModel` = BERT 特征提取（`AutoModel`）+ Linear 分类层；训练时**冻结 BERT 参数**只训练分类层。
> - **冻结方式**：`for p in bert.parameters(): p.requires_grad = False`，使 BERT 不参与梯度更新。
> - **训练配置**：`AdamW` 优化器（解耦权重衰减）+ `CrossEntropyLoss` 损失函数。
> - **模型预测**：`eval()` 切推理模式 + `torch.no_grad()` 关闭梯度 + `map_location` 处理 GPU/CPU 加载。
> - **关键区别**：`pooler_output`（CLS 句级表示）用于分类，`last_hidden_state`（每个 token 向量）用于序列标注。
> - **AdamW vs Adam**：AdamW 将权重衰减与梯度更新解耦，正则化效果更好，是 BERT 微调标配。

## 1️⃣ 完整知识库

## 数据预处理 🔸 核心

### 定义与本质

中文文本分类的数据预处理目标是把 CSV 格式的"文本+标签"转成 BERT 可消费的数字张量。流程：用 HuggingFace `load_dataset` 加载数据集 → 自定义 `collate_fn` 在批内完成分词与 padding → `DataLoader` 封装为迭代器。核心是 `collate_fn`：它接管 batch 数据，调用 tokenizer 统一编码并补齐到等长。

### 基础用法

#### 加载数据集 load_dataset

数据格式说明：CSV 文件两列，第一列是新闻文本，第二列是类别标签（0~9 共 10 类）。

```python
from datasets import load_dataset
from transformers import AutoTokenizer
from torch.utils.data import DataLoader
import torch
import torch.nn as nn
import torch.optim as optim

# 1. 加载数据集
dataset = load_dataset('csv', data_files='./data/MyTrainData.csv', split='train')
print(dataset[0])
# 输出：{'text': '...', 'label': 3}（示例）
# dataset = dataset.filter(lambda x: x['text'] is not None)  # 过滤空文本
dataset = dataset.filter(lambda x: x['text'] is not None)
```

> [!warning] 本代码依赖外部文件：./data/MyTrainData.csv（中文新闻分类数据集）、./model/bert-base-chinese（预训练模型目录）。

#### 分词器与 collate_fn

`collate_fn` 是 DataLoader 的批处理函数，接收一个 batch 的样本列表，返回编码后的张量。本案例在 `collate_fn` 内调用 `tokenizer.encode_plus` 对 batch 内所有文本统一编码、padding 到等长，并返回 `input_ids`/`attention_mask`/`labels`。

```python
# 2. 加载分词器
tokenizer = AutoTokenizer.from_pretrained('./model/bert-base-chinese')

# 3. 定义collate_fn函数
def collate_fn(batch):
    # batch是DataLoader传过来的一个批次的数据，列表里面是多个样本
    # [{'text': '...', 'label': 0}, {'text': '...', 'label': 1}, ...]
    texts = [item['text'] for item in batch]
    labels = [item['label'] for item in batch]

    # 对文本进行批量编码+padding
    encoding = tokenizer(texts, padding=True, truncation=True, max_length=30, return_tensors='pt')

    # 返回编码后的张量
    input_ids = encoding['input_ids']
    attention_mask = encoding['attention_mask']
    labels = torch.tensor(labels, dtype=torch.long)

    return input_ids, attention_mask, labels
```

#### 构建 DataLoader

```python
# 4. 构建DataLoader
dataloader = DataLoader(dataset, batch_size=32, shuffle=True, collate_fn=collate_fn)

# 验证DataLoader输出
for input_ids, attention_mask, labels in dataloader:
    print(f'input_ids: {input_ids.shape}, attention_mask: {attention_mask.shape}, labels: {labels.shape}')
    # 输出：input_ids: torch.Size([32, 30]), attention_mask: torch.Size([32, 30]), labels: torch.Size([32])
    break
```

### 进阶用法与原理

#### collate_fn 数据流向

```text
CSV 文件
  ↓ load_dataset
Dataset [{'text':..., 'label':...}, ...]
  ↓ DataLoader(batch_size=32, collate_fn)
batch = [{'text':..., 'label':...}, ...] × 32
  ↓ collate_fn 内 tokenizer(texts, padding=True)
input_ids [32, 30] + attention_mask [32, 30] + labels [32]
  ↓ 送入 AiModel
```

> [!note] 💡 AI 扩展（基础）
> **BertTokenizer 的 `batch_encode_plus` 参数说明表**：
>
> | 参数 | 类型 | 作用 | 取值示例 | 效果说明 |
> |------|------|------|---------|---------|
> | `padding` | str / bool | 控制序列补齐策略 | `"max_length"` / `True` / `False` | `"max_length"` 补齐到 `max_length`；`True` 补齐到 batch 内最长序列；`False` 不补齐 |
> | `truncation` | str / bool | 控制超长序列截断 | `True` / `"max_length"` / `False` | `True` 截断到模型最大长度（如 BERT 的 512）；`"max_length"` 截断到 `max_length` |
> | `max_length` | int | 指定最大序列长度 | `30` / `128` / `512` | 仅在 `padding="max_length"` 或 `truncation="max_length"` 时生效 |
> | `return_tensors` | str | 返回数据类型 | `"pt"` / `"np"` / `None` | `"pt"` 返回 PyTorch 张量；`"np"` 返回 NumPy 数组；`None` 返回 Python list |
> | `return_length` | bool | 是否返回序列实际长度 | `True` / `False` | 返回字典中额外包含 `"length"` 字段 |
> | `return_attention_mask` | bool | 是否返回 attention_mask | `True`（默认） / `False` | True 时补零位置为 0、真实 token 为 1 |
> | `return_token_type_ids` | bool | 是否返回 token_type_ids | `True`（默认） / `False` | 句子对任务中区分两句话（0/1） |
>
> - **原理**：`batch_encode_plus` 是 `encode_plus` 的批量版本，接收文本列表，自动处理 padding 和 truncation，返回包含 `input_ids`/`attention_mask`/`token_type_ids` 的字典。
> - **实战**：单文本任务（分类/NER）传 `batch_text_or_text_pairs=texts`；句子对任务（QA/NSP）传 `batch_text_or_text_pairs=[(sent1, sent2), ...]`。
> - **进阶**：`padding="max_length"` + `max_length=30` 会使所有序列等长为 30，浪费算力但形状固定；`padding=True` 按 batch 内最长序列动态补齐，更节省但每个 batch 形状可能不同。

### 避坑与局限

- `load_dataset` 加载的 CSV 列名须为 `text`/`label`（或用 `column_names` 指定）。
- `filter(lambda x: x['text'] is not None)` 过滤空文本，避免 tokenizer 报错。
- `max_length=30` 按数据集最长句子截断，过短会丢失信息，过长浪费算力。

## 模型搭建与训练 🔸 核心

### 定义与本质

模型 `AiModel` 由两部分组成：**BERT 特征提取器**（`AutoModel.from_pretrained`）+ **Linear 分类层**。训练时**冻结 BERT 参数**（`requires_grad=False`），只更新 Linear 层，这是迁移学习"微调"方式的典型做法——用预训练 BERT 提取通用语义特征，只训练任务相关的分类头。

### 基础用法

#### 模型搭建 AiModel

```python
from transformers import AutoModel

# 5. 定义模型
class AiModel(nn.Module):
    def __init__(self):
        super().__init__()
        # 1. BERT特征提取层（预训练模型）
        self.bert = AutoModel.from_pretrained('./model/bert-base-chinese')
        # 2. 分类层：BERT输出768维 → 10类
        self.classifier = nn.Linear(768, 10)

    def forward(self, input_ids, attention_mask):
        # 1. BERT特征提取
        outputs = self.bert(input_ids=input_ids, attention_mask=attention_mask)
        # 2. 取CLS token的pooler_output作为句级表示
        pooled_output = outputs.pooler_output  # [batch, 768]
        # 3. 分类层
        logits = self.classifier(pooled_output)  # [batch, 10]
        return logits
```

> [!note] 💡 AI 扩展（基础）
> **BERT 的 pooler_output 与 last_hidden_state 区别**：
>
> | 输出字段 | 形状 | 含义 | 适用任务 |
> |---------|------|------|---------|
> | `last_hidden_state` | `[batch, seq_len, 768]` | 每个 token 的上下文向量（含 CLS/SEP/PAD） | NER/序列标注/特征抽取 |
> | `pooler_output` | `[batch, 768]` | 取 CLS token（第 0 位）的 `last_hidden_state`，经一层全连接 + Tanh 激活 | 文本分类/句级任务 |
>
> - **原理**：`pooler_output = tanh(W·last_hidden_state[:,0] + b)`，CLS token 在 BERT 预训练时被训练为聚合全句语义的位置。
> - **实战**：本案例用 `pooler_output` 做分类；若做 NER 应取 `last_hidden_state` 的每个 token。
> - **进阶**：部分研究显示直接取 `last_hidden_state[:,0]`（不经 pooler 全连接）效果有时更好，需实验对比。

#### 冻结 BERT 参数

> [!note] 💡 AI 扩展（进阶）
> **冻结预训练参数的两种方式对比**：
>
> | 对比维度 | `requires_grad=False` | `with torch.no_grad()` |
> |---------|----------------------|----------------------|
> | 作用层级 | 参数级（标记参数不参与梯度计算） | 上下文级（该代码块内不构建计算图） |
> | 位置 | 模型定义后、训练前设置 | forward 或推理时包裹 |
> | 是否影响反向传播 | 是（参数无梯度，优化器不更新） | 否（仅不记梯度，参数若有 requires_grad 仍可被外部更新） |
> | 适用场景 | 训练时冻结部分层 | 推理/特征提取（不需梯度） |
> | 内存 | 节省（不存中间梯度） | 节省（不构建图） |
>
> - **原理**：`requires_grad=False` 是"永久"冻结（直到重新设为 True），`torch.no_grad()` 是"临时"关闭梯度（仅作用域内）。
> - **实战**：本案例在训练前用 `requires_grad=False` 冻结 BERT，只训练分类层；推理时用 `torch.no_grad()` 避免浪费显存。两者常配合使用。
> - **进阶**：若想部分微调 BERT（如只训最后几层），可按层名选择性设置 `requires_grad`。

```python
# 6. 实例化模型并冻结BERT参数
model = AiModel()

# 冻结BERT参数，只训练分类层
for param in model.bert.parameters():
    param.requires_grad = False

# 验证：只有classifier层的参数会被更新
trainable_params = sum(p.numel() for p in model.parameters() if p.requires_grad)
total_params = sum(p.numel() for p in model.parameters())
print(f'可训练参数: {trainable_params} / 总参数: {total_params}')
# 输出：可训练参数: 7690 / 总参数: 102267929（示例）
# 仅 Linear(768,10) 的 768*10+10=7690 个参数可训练
```

#### 模型训练

> [!note] 💡 AI 扩展（基础）
> **AdamW 与 Adam 的区别**：
>
> | 对比维度 | Adam | AdamW |
> |---------|------|-------|
> | 权重衰减方式 | L2 正则加入梯度（与动量耦合） | 解耦权重衰减（独立于梯度更新） |
> | 正则效果 | 权重衰减受动量影响，效果不稳定 | 权重衰减稳定，正则化更有效 |
> | 大学习率表现 | 易欠拟合 | 更鲁棒 |
> | 适用场景 | 通用 | Transformer/BERT 微调标配 |
> | 公式 | `w = w - lr*(m_hat/(√v_hat+ε) + λw)` | `w = w - lr*m_hat/(√v_hat+ε) - lr*λw` |
>
> - **原理**：Adam 把权重衰减加进梯度再过动量，导致衰减幅度被动量放大/缩小；AdamW 把权重衰减直接从参数减去，与梯度更新解耦，效果更稳定。
> - **实战**：HuggingFace Trainer 默认用 AdamW；本案例手动用 `optim.AdamW`。
> - **进阶**：AdamW 是 BERT/GPT 等 Transformer 微调的标配，因 Transformer 对权重衰减敏感。

```python
# 7. 训练配置
device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
model = model.to(device)

# 优化器：AdamW（BERT微调标配）
optimizer = optim.AdamW(model.parameters(), lr=2e-5)
# 损失函数：交叉熵
criterion = nn.CrossEntropyLoss()

# 8. 训练循环
epochs = 3
for epoch in range(epochs):
    model.train()
    total_loss = 0
    for input_ids, attention_mask, labels in dataloader:
        input_ids = input_ids.to(device)
        attention_mask = attention_mask.to(device)
        labels = labels.to(device)

        # 前向传播
        logits = model(input_ids, attention_mask)
        # 计算损失
        loss = criterion(logits, labels)
        # 梯度清零
        optimizer.zero_grad()
        # 反向传播
        loss.backward()
        # 参数更新
        optimizer.step()

        total_loss += loss.item()

    print(f'Epoch {epoch+1}/{epochs}, Loss: {total_loss/len(dataloader):.4f}')
    # 输出：Epoch 1/3, Loss: 1.2345（示例，数值逐轮下降）

# 9. 保存模型
torch.save(model.state_dict(), './model/text_classifier.pth')
```

#### 模型预测

预测流程：实例化模型 → 加载训练好的参数 → `eval()` 切推理模式 → `torch.no_grad()` 关闭梯度 → 前向得到 logits → `argmax` 取预测类别。

> [!note] 💡 AI 扩展（基础）
> **GPU/CPU 模型加载的注意事项**：
>
> | 场景 | 保存时设备 | 加载时设备 | 处理方式 |
> |------|-----------|-----------|---------|
> | GPU 训练 → GPU 推理 | cuda | cuda | 直接 `load_state_dict(torch.load(path))` |
> | GPU 训练 → CPU 推理 | cuda | cpu | `torch.load(path, map_location='cpu')` |
> | CPU 训练 → GPU 推理 | cpu | cuda | `torch.load(path, map_location='cuda')` |
> | 跨设备通用 | 任意 | 任意 | `map_location='cpu'` 先加载 CPU 再 `.to(device)` |
>
> - **原理**：`state_dict` 中的张量带有设备信息，直接加载到不同设备会报错" tensors on different devices"。
> - **实战**：推荐先 `map_location='cpu'` 加载到 CPU，再 `.to(device)` 迁移到目标设备，最稳妥。
> - **进阶**：`torch.load` 的 `map_location` 可传函数实现更灵活的设备映射。

```python
# 10. 模型预测
def predict(text):
    # 1. 实例化模型
    model = AiModel()
    # 2. 加载训练好的参数（注意map_location处理设备）
    # GPU训练保存的模型在CPU加载时需 map_location='cpu'
    model.load_state_dict(torch.load('./model/text_classifier.pth', map_location='cpu'))
    model = model.to(device)
    # 3. 切换为推理模式
    model.eval()

    # 4. 文本编码
    encoding = tokenizer(text, padding=True, truncation=True, max_length=30, return_tensors='pt')
    input_ids = encoding['input_ids'].to(device)
    attention_mask = encoding['attention_mask'].to(device)

    # 5. 推理（关闭梯度）
    with torch.no_grad():
        logits = model(input_ids, attention_mask)
        # 6. 取预测类别
        pred = torch.argmax(logits, dim=-1).item()

    return pred

# 测试预测
result = predict('中国队在奥运会上获得了多枚金牌')
print(f'预测类别: {result}')
# 输出：预测类别: 6（示例，6代表体育类）
```

### 进阶用法与原理

#### 模型整体架构与数据流向

```text
输入文本
  ↓ tokenizer 编码
input_ids [batch, 30] + attention_mask [batch, 30]
  ↓ AiModel.forward
  ├── BERT (冻结, requires_grad=False)
  │     └── pooler_output [batch, 768]  (CLS句级表示)
  ├── Linear(768, 10)  (可训练)
  │     └── logits [batch, 10]
  ↓ CrossEntropyLoss
loss (标量)
  ↓ 反向传播 (仅更新Linear层)
更新分类层参数
```

### 避坑与局限

- 冻结 BERT 后只有分类层参数可训练，训练快但可能欠拟合；若效果不佳可解冻最后几层 BERT 做部分微调。
- `CrossEntropyLoss` 内部含 `log_softmax`，故模型 forward 末尾**不应**再加 `log_softmax`/`softmax`，否则重复计算。
- 推理时必须 `model.eval()` + `torch.no_grad()`，前者关 dropout/LayerNorm 训练行为，后者省显存。

## 4️⃣ 避坑指南 & 易错对比

> [!warning] 易混概念对比：pooler_output vs last_hidden_state
>
> | 对比维度 | pooler_output | last_hidden_state |
> |---------|--------------|-------------------|
> | 形状 | `[batch, 768]` | `[batch, seq_len, 768]` |
> | 来源 | CLS token 经全连接+Tanh | 所有 token 的上下文向量 |
> | 粒度 | 句级 | token 级 |
> | 适用任务 | 文本分类 | NER/序列标注 |
> | 是否含特殊 token | 否（仅 CLS 聚合） | 是（含 CLS/SEP/PAD） |

> [!warning] 易混概念对比：冻结参数两种方式
>
> | 对比维度 | requires_grad=False | torch.no_grad() |
> |---------|--------------------|-----------------|
> | 作用层级 | 参数级 | 上下文级 |
> | 持续性 | 持久（直到改回 True） | 临时（仅作用域内） |
> | 适用 | 训练时冻结 | 推理时省显存 |

> [!warning] 易混概念对比：AdamW vs Adam
>
> | 对比维度 | AdamW | Adam |
> |---------|-------|------|
> | 权重衰减 | 解耦（独立减） | 耦合（加进梯度） |
> | 正则稳定性 | 高 | 低 |
> | BERT 微调 | 标配 | 不推荐 |

> [!danger] 常见错误与规避
> 1. **重复 softmax**：`CrossEntropyLoss` 内部已做 `log_softmax`，模型 forward 末尾再加 `softmax`/`log_softmax` 会重复计算导致数值错误。规避：forward 末尾只输出 `logits`（原始 Linear 输出）。
> 2. **冻结后未验证可训练参数**：忘了设置 `requires_grad=False` 或设错对象，导致 BERT 全量参数被训练，显存爆炸。规避：训练前打印 `sum(p.numel() for p in model.parameters() if p.requires_grad)` 验证。
> 3. **推理未 `eval()`**：dropout 在 train 模式下随机置零，推理结果不可复现。规避：推理前必调 `model.eval()`。
> 4. **跨设备加载报错**：GPU 保存的模型在 CPU 加载未用 `map_location`，报"tensors on different devices"。规避：`torch.load(path, map_location='cpu')`。
> 5. **collate_fn 返回类型错误**：`labels` 未转 `torch.tensor` 或 dtype 错误，导致 `CrossEntropyLoss` 报错。规避：`labels = torch.tensor(labels, dtype=torch.long)`。
> 6. **max_length 过小**：截断丢失关键信息，分类效果差。规避：根据数据集句子长度分布合理设置（如覆盖 95% 分位数）。
