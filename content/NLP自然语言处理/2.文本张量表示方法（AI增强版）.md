# [day02] 学习笔记｜文本张量表示方法（AI 增强版）

## 📌 核心速览

> [!summary] 核心速览
> - **文本张量表示**：将文本转为向量（数字）供模型训练，方式有 One-Hot、Word2Vec、WordEmbedding。
> - **One-Hot**：每个词用长度为词表大小 n 的 0/1 向量表示，简单但割裂词间联系、维度爆炸。
> - **One-Hot 实现**：用 Keras 的 `Tokenizer` 拟合文本生成词表索引，再按索引置 1，可用 `joblib` 保存映射器。
> - **Word2Vec 本质**：无监督训练一个模型，把模型参数矩阵当作所有词汇的词向量表示。
> - **CBOW**：给定上下文窗口，用上下文预测中间目标词；**SkipGram** 则用中间词预测上下文。
> - **Fasttext**：Word2Vec 的增强版，支持子词（subword）信息，训练保存加载用 `train_unsupervised`/`save_model`/`load_model`。
> - **WordEmbedding**：将词映射到指定维度的空间，可用 PyTorch 的 `nn.Embedding` 层实现并用 TensorBoard 可视化。
> - **词向量来源**：==训练得到的输入权重矩阵 W 的每一列（行）就是一个词的词向量==。

## 1️⃣ 完整知识库

### One-Hot 词向量表示 🔹 基础

#### 定义与本质

**文本张量表示的意义**：将文本转换为向量（数字）的形式，使得模型能够识别进而实现训练，一般是进行词向量的表示。实现方式有 One-Hot、Word2Vec、WordEmbedding 三种。

**One-Hot 词向量表示定义**：针对每一个词汇，都会用一个向量表示，向量的长度是 n，n 代表去重之后的词汇总量，而且向量中只有 0 和 1 两种数字。俗称：独热编码、01 编码。

> [!note] 💡 AI 扩展（基础）
> **One-Hot 的工程意义与定位**：
> - **背景**：One-Hot 是最直观的"词→向量"映射，它把"词表中有无该词"这一离散信息编码为向量，是理解词嵌入的起点。
> - **实战**：实际工程中几乎不直接用 One-Hot 训练模型（维度爆炸 + 词间正交），但它常作为 Embedding 层的输入形式——One-Hot（或词索引）与权重矩阵相乘等价于"查表"取词向量，这正是 nn.Embedding 的工作原理。
> 因此即便不直接使用，理解 One-Hot 也是理解 Word2Vec/Embedding 的必要基础。

#### 基础用法

**思路分析（生成 One-Hot）**：

```text
1 准备语料 vocabs
2 实例化词汇映射器 Tokenizer，使用映射器拟合现有文本数据（内部生成 index_word、word_index）
   2-1 注意 idx 序号 -1
3 查询单词 idx，赋值 zero_list，生成 onehot
4 使用 joblib 工具保存映射器 joblib.dump()
```

**代码实现（生成 One-Hot）**：

```python
import jieba
# 导入 keras 中的词汇映射器 Tokenizer
from tensorflow.keras.preprocessing.text import Tokenizer
# 导入用于对象保存与加载的 joblib（sklearn 0.23+ 已移除 sklearn.externals.joblib，直接 import joblib）
import joblib

def dm_onehot_gen():
    # 1 准备语料 vocabs
    vocabs = {"周杰伦", "陈奕迅", "王力宏", "李宗盛", "吴亦凡", "鹿晗"}

    # 2 实例化词汇映射器 Tokenizer，使用映射器拟合现有文本数据（内部生成 index_word、word_index）
    # 2-1 注意 idx 序号 -1
    mytokenizer = Tokenizer()
    mytokenizer.fit_on_texts(vocabs)

    # 3 查询单词 idx，赋值 zero_list，生成 onehot
    for vocab in vocabs:
        zero_list = [0] * len(vocabs)
        idx = mytokenizer.word_index[vocab] - 1
        zero_list[idx] = 1
        print(vocab, '的onehot编码是', zero_list)

    # 4 使用 joblib 工具保存映射器 joblib.dump()
    mypath = './mytokenizer'
    joblib.dump(mytokenizer, mypath)
    print('保存mytokenizer End')

    # 注意5-1 字典没有顺序，onehot 编码没有顺序 []-有序 {}-无序 区别
    # 注意5-2 字典有的单词才有 idx，idx 从 1 开始
    # 注意5-3 查询没有注册的词会有异常，eg: 狗蛋
    print(mytokenizer.word_index)
    print(mytokenizer.index_word)
```

> [!warning] 本代码依赖外部环境：需安装 tensorflow（提供 `keras.preprocessing.text.Tokenizer`）与 joblib

**One-Hot 的使用（加载已保存的映射器）**：

```text
思路分析：
1 加载已保存的词汇映射器 Tokenizer joblib.load(mypath)
2 查询单词 idx，赋值 zero_list，生成 onehot（以 token 为 '李宗盛'）
3 token = "狗蛋" 会出现异常
```

```python
def dm_onehot_use():
    vocabs = {"周杰伦", "陈奕迅", "王力宏", "李宗盛", "吴亦凡", "鹿晗"}

    # 1 加载已保存的词汇映射器 Tokenizer joblib.load(mypath)
    mypath = './mytokenizer'
    mytokenizer = joblib.load(mypath)

    # 2 编码 token 为"李宗盛"，查询单词 idx，赋值 zero_list，生成 onehot
    token = "李宗盛"
    zero_list = [0] * len(vocabs)
    idx = mytokenizer.word_index[token] - 1
    zero_list[idx] = 1
    print(token, '的onehot编码是', zero_list)
```

> [!warning] 本代码依赖外部文件：./mytokenizer（由上一步 `joblib.dump` 保存的映射器）

#### 避坑与局限

**One-Hot 编码的缺点**：
- 割裂了词与词之间的联系（主要缺点）；
- 如果 n 过大，会导致占用大量的内存（维度爆炸）。

**实现注意点**：
- 字典（`{}`）无序，因此 One-Hot 编码本身没有固定顺序；列表（`[]`）有序。
- 只有注册（拟合）过的词才有 idx，且 idx 从 1 开始，故赋值时需 `idx - 1`。
- 查询未注册的词（如"狗蛋"）会抛出异常。

### Word2Vec 模型 🔺 难点

#### 定义与本质

**Word2Vec 定义**：Word2Vec 是一种无监督的训练方法，本质是训练一个模型，将模型的参数矩阵当作所有词汇的词向量表示。它有两种训练方式：CBOW 和 SkipGram。

核心思想是：==不直接存储 One-Hot，而是通过一个"预测任务"训练神经网络，训练完成后丢弃预测头，只保留输入权重矩阵作为词向量表==。这样得到的稠密向量能表达词间语义相似度。

#### 基础用法

**（1）CBOW 介绍**

CBOW（Continuous Bag of Words）：给一段文本，选择一定的窗口，然后利用上下文预测中间目标词。

实现过程如下图所示：

![[cbow_intro.png]]

如图，以"我爱黑马程序员啊"为例（词汇总量 5，窗口大小 3，词向量维度 3）：
- **训练样本构造**：窗口大小 3 表示上下文各取 1 个词，第一个样本"我 | 爱 | 黑马"中用上下文"我""黑马"预测中间目标词"爱"。
- **模型结构**：输入层接收上下文词的 One-Hot 向量（5 维），经输入权重矩阵 W（5×3）投影到隐藏层得到 3 维稠密向量，多个上下文词向量求和/平均后，再经输出权重矩阵 W'（3×5）与 softmax 预测中心词的概率分布。
- **词向量来源**：训练完成后，输入权重矩阵 W 的每一列就是一个词的词向量。

**（2）Word2Vec 之 SkipGram 方式**

- **定义**：给你一段文本，选定特定的窗口长度，然后利用中间词来预测上下文。
- **实现过程**：
  1. 选定一个窗口长度：3、5、7 等；
  2. 指定词向量的维度：人为规定。

![[skipgram.png]]

如图，SkipGram 与 CBOW 方向相反：以中心词"爱"为输入，预测其上下文"我"和"黑马"。中心词的 One-Hot 经权重矩阵 W 得到词向量，再经输出矩阵 Z 和 softmax 分别预测每个上下文词。同样，训练后保留 W 作为词向量表。

#### 进阶用法与原理

**获取词向量**：Word2Vec 训练完成后，词向量从输入层到隐藏层的权重矩阵中提取，其本质是"矩阵查找"操作。

![[word2vec_embedding.png]]

如图，参数矩阵 W 的每一列对应一个词的词向量。用词的 One-Hot 向量与 W 相乘，等价于"提取矩阵中对应列"——One-Hot 向量充当选择器，只有该词位置为 1，乘法结果恰好取出该词对应的列向量。这就是"权重矩阵即词向量"的数学本质。

> [!note] 💡 AI 扩展（进阶）
> **CBOW vs SkipGram 如何选择？**
> - **原理**：CBOW 用多个上下文词求平均后预测一个中心词，信号被平滑，适合高频词；SkipGram 用一个中心词分别预测多个上下文词，每个上下文词都构成一个训练样本，对低频词学习更充分。
> - **性能/实战**：CBOW 训练速度快（样本少），适合大规模语料；SkipGram 训练慢但效果通常更好，尤其语料较小或罕见词较多时。原论文实验表明 SkipGram 在语义类比任务上表现更优。
> - **进阶**：当词表很大时，完整 softmax 计算昂贵，实际实现常用**负采样（Negative Sampling）**或**层次 Softmax（Hierarchical Softmax）**加速，二者都是在近似全词表 softmax 的基础上降低计算复杂度。

#### 避坑与局限

- CBOW 与 SkipGram 的预测方向容易混淆：记住"CBOW=上下文→中心（多对一）；SkipGram=中心→上下文（一对多）"。
- 词向量取自输入权重矩阵 W，而非输出矩阵 W'（W' 训练后通常丢弃）。
- Word2Vec 是静态词向量，同一个词在不同语境下向量相同，无法解决一词多义。

### Fasttext 实现 Word2Vec 训练 🔸 核心

#### 定义与本质

Fasttext 是 Facebook 开源的词向量训练工具，是 Word2Vec 的增强版。它不仅支持 CBOW/SkipGram，还引入了**子词（subword）信息**——把词拆成字符 n-gram，从而能对未登录词（OOV）也能生成向量。

#### 基础用法

**基本过程**：

```text
导包：fasttext
获取数据集
训练和保存
加载使用
```

**代码实现**：

```python
# 导入 fasttext
import fasttext

def dm_fasttext_train_save_load():
    # 1 使用 train_unsupervised（无监督训练方法）训练词向量
    mymodel = fasttext.train_unsupervised('./data/fil9')
    print('训练词向量 ok')

    # 2 save_model() 保存已经训练好的词向量
    # 注意，该行代码执行耗时很长
    mymodel.save_model("./data/fil9.bin")
    print('保存词向量 ok')

    # 3 模型加载
    mymodel = fasttext.load_model('./data/fil9.bin')
    print('加载词向量 ok')
```

> [!warning] 本代码依赖外部文件：./data/fil9（英文语料，需自行准备）

```text
Number of labels: 0

```

> [!note] 💡 AI 扩展（基础）
> **Fasttext 相比 Word2Vec 的优势**：
> - **背景**：Word2Vec 以"词"为最小单位，遇到训练时未见的词（OOV）无法给出向量；Fasttext 把每个词拆成字符级 n-gram（如"where"拆为 `<wh`、`whe`、`her`、`ere`、`re>` 等），词向量由其子词向量叠加得到。
> - **实战意义**：因此 Fasttext 能为未登录词生成合理向量，对形态丰富的语言（如英文、俄文）和小语种尤其有效；同时训练速度极快，适合工业级大规模语料。

#### 避坑与局限

- `save_model` 保存的 `.bin` 文件较大，耗时较长，需预留存储与时间。
- Fasttext 默认对空格分词的文本（如英文）效果最好，中文需先分词再加空格。

### WordEmbedding 词向量 🔸 核心

#### 定义与本质

**WordEmbedding 定义**：将词映射到指定维度的空间，是词向量的一种表示方法。它不特指某个算法，而是一种"词嵌入"思想——One-Hot/Word2Vec 都可看作其具体实现，工程上常用 PyTorch 的 `nn.Embedding` 层来完成。

#### 基础用法

**实现过程**：

```python
def dm02_nnembeding_show():
    # 1 对句子分词 word_list
    sentence1 = '传智教育是一家上市公司，旗下有黑马程序员品牌。我是在黑马这里学习人工智能'
    sentence2 = "我爱自然语言处理"
    sentences = [sentence1, sentence2]

    word_list = []
    for s in sentences:
        word_list.append(jieba.lcut(s))

    # 2 对句子 word2id 求 my_token_list，对句子文本数值化 sentence2id
    mytokenizer = Tokenizer()
    mytokenizer.fit_on_texts(word_list)

    # 打印 my_token_list
    my_token_list = mytokenizer.index_word.values()
    print('my_token_list-->', my_token_list)

    # 打印文本数值化以后的句子
    sentence2id = mytokenizer.texts_to_sequences(word_list)
    print('sentence2id--->', sentence2id, len(sentence2id))

    # 3 创建 nn.Embedding 层
    embd = nn.Embedding(num_embeddings=len(my_token_list), embedding_dim=8)

    # 4 创建 SummaryWriter 对象，词向量矩阵 embd.weight.data 和词向量单词列表 my_token_list
    summarywriter = SummaryWriter()
    summarywriter.add_embedding(embd.weight.data, my_token_list)
    summarywriter.close()

    # 5 通过 tensorboard 观察词向量相似性
    # cd 程序的当前目录下执行下面的命令
    # 启动 tensorboard 服务：tensorboard --logdir=runs --host 0.0.0.0
    # 通过浏览器查看词向量可视化效果：http://127.0.0.1:6006

    print('从nn.Embedding层中根据idx拿词向量')
    # 6 从 nn.Embedding 层中根据 idx 拿词向量
    for idx in range(len(mytokenizer.index_word)):
        tmpvec = embd(torch.tensor(idx))
        print('%4s' % (mytokenizer.index_word[idx + 1]), tmpvec.detach().numpy())
```

> [!warning] 本代码依赖外部环境：需导入 `jieba`、`torch.nn as nn`、`torch.utils.tensorboard.SummaryWriter` 及 `Tokenizer`

#### 进阶用法与原理

**可视化**：
- 工具：TensorBoard
- 命令：`tensorboard --logdir=runs --host 0.0.0.0`

> [!info] 🧠 速查卡片
> **TensorBoard 可视化 Embedding 参数说明**
>
> | 参数/API | 说明 | 典型值/示例 |
> |---------|------|-----------|
> | `SummaryWriter(log_dir)` | 指定日志输出目录 | `SummaryWriter('./runs/exp1')` |
> | `add_embedding(mat, metadata)` | 写入词向量矩阵与标签 | `writer.add_embedding(embd.weight.data, word_list)` |
> | `mat` | 词向量矩阵，形状 `(N, D)` | `embd.weight.data`，N=词表大小，D=embedding_dim |
> | `metadata` | 每行对应的词标签列表 | `list(tokenizer.index_word.values())` |
> | `--logdir` | TensorBoard 读取的日志目录 | `tensorboard --logdir=runs` |
> | `--host` | 绑定 IP（远程访问用） | `tensorboard --logdir=runs --host 0.0.0.0` |
> | `--port` | 指定端口号（默认 6006） | `tensorboard --logdir=runs --port 6007` |
> | `writer.close()` | 写入完成后关闭，否则日志不完整 | 必须调用 |

`nn.Embedding` 的本质就是一张"词向量查找表"：`num_embeddings` 是词表大小，`embedding_dim` 是嵌入维度。传入词的索引，返回对应行的向量；这张表的权重会在端到端训练中随任务一起更新，因此可学到任务相关的词向量。

> [!note] 💡 AI 扩展（进阶）
> **nn.Embedding 的反向传播优化原理**：
>
> - **梯度计算方式**：Embedding 层的前向是"查表"（gather 操作），反向传播时只更新被实际查到的行对应的梯度，未参与前向的行梯度为 0。这称为**稀疏梯度更新**——一个 batch 中往往只命中词表的一小部分，只有这些词的嵌入向量被更新。
> - **梯度累积行为**：若一个 batch 中同一词出现多次，该词嵌入向量会累积来自各位置的反向梯度后一次性更新，相当于该词的多个上下文信号叠加。例如句子 "我 爱 学习 爱 运动" 中，"爱"出现了 2 次，它的梯度是 2 个位置梯度的和。
> - **优化挑战**：
>   1. **冷启动问题**：初始随机初始化导致低频词训练不充分，其嵌入向量长期停留在随机值附近。常用缓解方案包括预训练词向量加载 + 微调（warm-start）或在训练初期使用较大的学习率。
>   2. **特征维度的梯度消失/爆炸**：若 embedding_dim 过大而数据量不足，嵌入向量的某些维度可能始终收到接近 0 的梯度，导致"死维度"（dead dimensions）。实践中常用 LayerNorm 或 embedding dropout 缓解。
>   3. **词表截断冲突**：当 `num_embeddings` 远大于实际词频时，大量行从未被更新，浪费显存。工程上应动态裁剪词表或将低频词映射为 `<UNK>` 统一处理。
> - **内存优化 trick**：Embedding 层的反向传播在 PyTorch 中默认使用稀疏梯度模式（`sparse=True`），即只更新命中的行，未命中行的梯度不计算也不存储，可显著降低显存占用。`torch.nn.Embedding` 的 `sparse` 参数默认为 `False`，若词表极大可设为 `True` 以启用稀疏更新。

#### 避坑与局限

- `nn.Embedding` 传入的必须是整数索引张量（`torch.tensor(idx)`），不能传 One-Hot 向量。
- 词表大小 `num_embeddings` 必须覆盖所有可能出现的索引，否则越界报错。
- 若不预训练，Embedding 层初始为随机值，需要足够数据与训练轮数才能学到有意义的向量。

## 3️⃣ 代码库

下面给出 One-Hot 生成与使用的完整可运行代码（Fasttext 与 nn.Embedding 的完整代码见上文对应主题块，此处为 One-Hot 综合案例）：

```python
import jieba
from tensorflow.keras.preprocessing.text import Tokenizer
import joblib  # 注意：sklearn 0.23+ 已移除 sklearn.externals.joblib

# ============ 1 生成 One-Hot 并保存映射器 ============
vocabs = {"周杰伦", "陈奕迅", "王力宏", "李宗盛", "吴亦凡", "鹿晗"}

mytokenizer = Tokenizer()
mytokenizer.fit_on_texts(vocabs)  # 拟合文本，内部生成 word_index / index_word

for vocab in vocabs:
    zero_list = [0] * len(vocabs)
    idx = mytokenizer.word_index[vocab] - 1  # idx 从 1 开始，需 -1
    zero_list[idx] = 1
    print(vocab, '的onehot编码是', zero_list)

joblib.dump(mytokenizer, './mytokenizer')  # 保存映射器
print('保存mytokenizer End')

# ============ 2 加载映射器并编码新词 ============
mytokenizer = joblib.load('./mytokenizer')
token = "李宗盛"
zero_list = [0] * len(vocabs)
idx = mytokenizer.word_index[token] - 1
zero_list[idx] = 1
print(token, '的onehot编码是', zero_list)
# 输出：李宗盛 的onehot编码是 [0, 0, 0, 1, 0, 0]  （具体位置取决于拟合顺序）
```

> [!warning] 本代码依赖外部环境：tensorflow（提供 Tokenizer）与 joblib

## 4️⃣ 避坑指南 & 易错对比

> [!warning] 核心理念
> **词向量选择原则**：小词表演示用 One-Hot；通用语义表示用 Word2Vec/Fasttext 预训练向量；端到端任务用 nn.Embedding 随任务训练。==词向量方法的选择取决于语料规模、是否需要 OOV 处理和是否端到端训练==。

| 对比维度 | One-Hot | Word2Vec | Fasttext | nn.Embedding |
|---------|---------|----------|----------|--------------|
| 训练方式 | 无需训练 | 无监督预测任务 | 无监督 + 子词 | 端到端随任务训练 |
| 向量类型 | 稀疏 0/1 | 稠密静态 | 稠密静态 | 稠密（可动态） |
| 维度 | = 词表大小 n | 50~300 维（人为指定） | 50~300 维 | embedding_dim（人为指定） |
| 词间相似度 | 无（正交） | 有 | 有 | 有 |
| OOV 处理 | 不支持 | 不支持 | ==支持（子词）== | 不支持（需覆盖词表） |
| 典型用途 | 教学演示 | 通用词向量 | 多语种/OOV 场景 | 下游模型嵌入层 |

| 对比维度 | CBOW | SkipGram |
|---------|------|----------|
| 预测方向 | 上下文 → 中心词 | 中心词 → 上下文 |
| 训练样本数 | 少（多对一） | 多（一对多） |
| 训练速度 | 快 | 慢 |
| 高频词效果 | 好 | 一般 |
| 低频词效果 | 一般 | ==更好== |
| 适用语料 | 大规模 | 小规模/罕见词多 |

> [!danger] 常见错误与规避
> - **误用废弃 API**：`from sklearn.externals import joblib` 在 sklearn 0.23+ 报错。规避：直接 `import joblib`。
> - **idx 未减 1**：Tokenizer 的 `word_index` 从 1 开始，直接用作列表下标会越界或错位。规避：统一 `idx - 1`。
> - **查询未注册词**：对未拟合的词查 `word_index` 会抛 `KeyError`。规避：先判断 `if token in mytokenizer.word_index`。
> - **混淆 W 与 W'**：误以为输出层权重也是词向量。规避：词向量只取输入权重矩阵 W。
> - **nn.Embedding 传错输入**：传入 One-Hot 或浮点张量会报错。规避：传入整数索引张量。
