# [day09] 学习笔记｜Seq2Seq英译法案例（AI 增强版）

## 📌 核心速览

> [!summary] 核心速览
> - **任务目标**：给定英文文本，翻译为法文；本质是逐时间步预测法文单词的分类任务。
> - **数据格式**：英文与法文两列用制表符 `\t` 分隔，需经 `normalizeString` 清洗（小写化、标点前加空格、非字母替换为空格）。
> - **数据管线**：构建 `word2index/index2word` 字典 → 继承 `Dataset` 实现 `__len__/__getitem__` → `DataLoader` 封装迭代器。
> - **编码器**：`Embedding + GRU`，将输入英文序列编码为隐层上下文向量 C（`batch_first=True` 直接承接 DataLoader 输出）。
> - **无 Attention 解码器**：仅依赖上下文 C 与上一时间步隐层，逐词解码，长序列易丢失前端信息。
> - **带 Attention 解码器**：引入 Q/K/V 三要素（Q=解码器输入、K=上一隐层、V=编码器输出），动态聚焦源句相关位置。
> - **teacher_forcing 策略**：训练时按概率用真实标签作为下一步输入，加速收敛但需防范暴露偏差。
> - **评估与可视化**：`evaluate_seq2seq` 逐词预测并用 `matshow` 绘制注意力权重热力图。

## 1️⃣ 完整知识库

## 数据预处理 🔸 核心

### 定义与本质

Seq2Seq 翻译任务的数据预处理目标是把"英法句对"这种纯文本转成模型可消费的数字张量。整个流程包含三步：**清洗脏数据 → 构建词表字典 → 封装 Dataset/Dataloader**。

任务实现总流程：

```text
1. 获取数据: 案例中是直接给定的
2. 数据预处理: 脏数据清洗、数据格式转换、数据源Dataset的构造、数据迭代器Dataloader的构造
3. 模型搭建: 编码器和解码器等一系列模型
4. 模型评估（测试）
5. 模型上线---API接口
```

数据格式说明：两列数据，第一列是英文文本，第二列是法文文本，中间用制表符 `\t` 隔开。

```text
i am from brazil .  je viens du bresil .
i am from france .  je viens de france .
i am from russia .  je viens de russie .
```

### 基础用法

#### 样本清洗函数 normalizeString

`normalizeString` 将脏数据清洗，以免影响模型训练。三步处理：小写化 + 去首尾空白；在 `.!?` 前补一个空格（`\1` 引用第一个分组）；将非大小写字母和正常标点的字符替换为空格。

```python
import re

# 文本清洗工具函数
def normalizeString(s):
    """字符串规范化函数, 参数s代表传入的字符串"""
    s = s.lower().strip()
    # 在.!?前加一个空格  这里的\1表示第一个分组   正则中的\num
    s = re.sub(r"([.!?])", r" \1", s)
    # 使用正则表达式将字符串中 不是 大小写字母和正常标点的都替换成空格
    s = re.sub(r"[^a-zA-Z.!?]+", r" ", s)
    return s

# 输出：normalizeString("I'm from Brazil.\tJe viens du Brésil.")
# 输出："i m from brazil ."
```

#### 构建字典 my_getdata

构建字典有两个目的：一是将文本进行数字表示（`word2index`），二是解码时将预测的索引数字映射回真实文本（`index2word`）。字典初始化时预留 `SOS`(0) 和 `EOS`(1) 两个特殊 token，普通词从索引 2 开始累加。

```python
def my_getdata():
    # 1.读取数据
    with open(data_path, 'r' , encoding='utf-8') as fr:
        sentens_str = fr.read()
    sentences = sentens_str.strip().split('\n')

    # 2.构建数据源pair
    my_pairs = [[normalizeString(s) for s in l.split('\t')] for l in sentences]

    # 3.1 初始化两个字典
    english_word2index = {"SOS": 0, "EOS": 1}
    english_word_n = 2
    french_word2index =  {"SOS": 0, "EOS": 1}
    french_word_n = 2
    # 3.2遍历my_pairs
    for pair in my_pairs:
        for word in pair[0].split(' '):
            if word not in english_word2index:
                english_word2index[word] = english_word_n
                english_word_n += 1

        for word in pair[1].split(' '):
            if word not in french_word2index:
                french_word2index[word] = french_word_n
                french_word_n += 1

    english_index2word = {v: k for k, v in english_word2index.items()}
    french_index2word = {v: k for k, v in french_word2index.items()}
    return english_word2index, english_index2word, english_word_n, french_word2index, french_index2word, french_word_n, my_pairs
```

#### 构建数据源 DataSet

使用 PyTorch 框架的规矩：用 `DataSet` 方法构造数据源。必须继承 `torch.utils.data.Dataset`，并实现两个魔法方法：`__len__()` 返回样本总数（可直接 `len(dataset)`）；`__getitem__()` 按索引取样本（可直接 `dataset[index]`）。在 `__getitem__` 内完成文本数字化：英文转 `tensor_x`，法文转 `tensor_y` 并在末尾追加 `EOS_token`。

```python
# 3.构建数据源Dataset
class Seq2SeqDaset(Dataset):
    def __init__(self, my_pairs):
        self.my_pairs = my_pairs
        self.sample_len = len(my_pairs)

    def __len__(self):
        return self.sample_len

    def __getitem__(self, index):
        # 1.index异常值处理[0, self.sample_len-1]
        index = min(max(index, 0), self.sample_len-1)

        # 2. 根据index取出样本数据
        x = self.my_pairs[index][0]
        y = self.my_pairs[index][1]

        # 3.进行文本数据数字化的转换
        x1 = [english_word2index[word] for word in x.split(' ')]
        tensor_x = torch.tensor(x1, dtype=torch.long, device=device)

        y1 = [french_word2index[word] for word in y.split(' ')]
        y1.append(EOS_token)
        tensor_y = torch.tensor(y1, dtype=torch.long, device=device)

        return tensor_x, tensor_y
```

#### 构建数据迭代器 Dataloader

`Dataloader` 对 `Dataset` 再次封装，变成可 `for` 循环的迭代器，自动增维（`batch_size`）并随机打乱取值顺序。本案例 `batch_size=1`、`shuffle=True`。

```python
# 4.构建数据迭代器dataloader
def get_dataloader():
    # 1.实例化dataset
    my_dataset = Seq2SeqDaset(my_pairs)
    # 2.实例化dataloader
    my_dataloader = DataLoader(dataset=my_dataset, batch_size=1, shuffle=True)
    return my_dataloader
```

### 避坑与局限

- `normalizeString` 中两个 `re.sub` 顺序不能颠倒：先给标点前加空格，再把非字母标点替换为空格，否则会把刚加的空格又吞掉。
- `__getitem__` 里 `index` 做了 `min(max(index,0), self.sample_len-1)` 兜底，避免越界。
- 法文 `tensor_y` 必须追加 `EOS_token` 作为解码终止信号，英文 `tensor_x` 不需要 EOS。

## 编码器GRU模型 🔸 核心

### 定义与本质

编码器把变长英文序列压缩成一个固定维度的隐层上下文向量 C。结构上就是 `Embedding + GRU`：Embedding 把单词索引变成稠密向量，GRU 逐步更新隐状态，最终输出所有时间步的 `output` 与最后一步的 `hidden`（即上下文 C）。

> [!note] 💡 AI 扩展（基础）
> **Encoder-Decoder 架构总览**：Encoder 负责把源序列编码为语义表示 C（上下文向量），Decoder 基于 C 自回归地生成目标序列。本案例中 Encoder 用 GRU，Decoder 也用 GRU；带 Attention 后 C 从"一个固定向量"变成"每步动态加权的编码器输出"。
> - **原理**：`C = Encoder(source_seq)`，`y_t = Decoder(y_{<t}, C, hidden)`
> - **实战**：batch_size=1 时，编码器输出形状 `[1, seq_len, hidden_size]`，最后隐层 `[1, 1, hidden_size]`
> - **局限**：无 Attention 时，整个源句被压进一个固定向量，长句前端信息易被覆盖（这正是引入 Attention 的动机）。

> [!note] 💡 AI 扩展（基础）
> **Encoder-Decoder 完整流程图（编码 → 上下文向量 → 解码 → 输出）**：
>
> ```text
> ┌───────────────────────────────────────────────────────────────────┐
> │                     编码阶段（Encoder）                           │
> │                                                                   │
> │  输入英文序列: "I am from China"                                  │
> │  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐                │
> │  │  I  │ │ am  │ │from │ │China│ │ EOS │ │ PAD │ ...             │
> │  └──┬──┘ └──┬──┘ └──┬──┘ └──┬──┘ └──┬──┘ └──┬──┘                │
> │     └───────┴───────┴───────┴───────┴───────┘                     │
> │                    ↓ Embedding                                      │
> │     ┌───────┐ ┌───────┐ ┌───────┐ ┌───────┐ ┌───────┐            │
> │     │ e(I) │ │ e(am)│ │e(from)│ │e(China)│ │ e(EOS)│            │
> │     └──┬───┘ └──┬───┘ └──┬───┘ └──┬───┘ └──┬───┘                │
> │        └───────┴───────┴───────┴───────┘                          │
> │                    ↓ GRU（逐时间步更新隐状态）                     │
> │     h_0 → h_1 → h_2 → h_3 → h_4 → h_5                          │
> │                                                                   │
> │  输出:                                                            │
> │    encoder_output: [1, seq_len, hidden_size] 所有时间步的隐状态    │
> │    encoder_hidden: [1, 1, hidden_size] 最后时间步的隐状态 = C      │
> └────────────────────────┬──────────────────────────────────────────┘
>                          │
>                          │ C = encoder_hidden（上下文向量）
>                          ▼
> ┌───────────────────────────────────────────────────────────────────┐
> │                     解码阶段（Decoder）                            │
> │                                                                   │
> │  初始输入: SOS_token                                              │
> │  初始隐状态: C = encoder_hidden                                   │
> │                                                                   │
> │  时间步 t=1:                                                     │
> │    输入: SOS → Embedding → GRU(input_emb, C) → hidden_1          │
> │    → Linear → Softmax → P(y_1) → 预测出 "je"                      │
> │                                                                   │
> │  时间步 t=2:                                                     │
> │    输入: "je" → Embedding → GRU(input_emb, hidden_1) → hidden_2  │
> │    → Linear → Softmax → P(y_2) → 预测出 "viens"                   │
> │                                                                   │
> │  ... 循环直到预测出 EOS_token 为止 ...                             │
> │                                                                   │
> │  输出: [SOS, "je", "viens", "de", "chine", EOS]                  │
> └───────────────────────────────────────────────────────────────────┘
> ```
>
> - **关键节点**：编码器最终输出 `C`（上下文向量）是解码器的初始隐状态；解码器每步用上一步的预测词（teacher_forcing 时用真实词）作为下一步输入，逐词自回归生成。
> - **带 Attention 变化**：有 Attention 时，解码器每步不只用 C，还通过 Q/K/V 动态聚焦编码器的 `encoder_output` 各位置，C 从"固定"变为"动态加权"。

### 基础用法

> [!warning] 本代码依赖外部文件：eng-fra.txt（英法句对数据集）以及全局变量 data_path / device / EOS_token / SOS_token / MAX_LENGTH。

```python
# 5. 构建GRU编码器模型
class EncoderGRU(nn.Module):
    def __init__(self, vocab_size, hidden_size):
        super().__init__()
        # 1.vocab_size代表英文单词的总个数（去重）
        self.vocab_size = vocab_size
        # 2. hidden_size词嵌入维度/隐藏层输出维度（我们让他相等）
        self.hidden_size = hidden_size
        # 3.定义Embedding层，目的：将每个词汇进行向量表示
        self.embed = nn.Embedding(self.vocab_size, self.hidden_size)
        # 4.定义GRU层第一个self.hidden_size实际上是embedding的输出结果词嵌入维度
        # 4.定义GRU层第二个self.hidden_size实是我们指定的GRU模型的输出维度，只不过这里GRU输入和输出一样
        self.gru = nn.GRU(self.hidden_size, self.hidden_size, batch_first=True)
    def forward(self, input, hidden):
        # 1.input-->[1, 6]需要经过embedding--》[1,6, 256]
        input_x = self.embed(input)
        # 2.将input_x和hidden送入GRU模型
        output, hidden = self.gru(input_x, hidden)
        return output, hidden

    def inithidden(self):
        return torch.zeros(1, 1, self.hidden_size, device=device)
```

> [!warning] 核心理念
> GRU 在实例化时默认 `batch_first=False`，因此需要小心输入数据的形状。因为 `dataloader` 返回的 `x` 形状是 `[batch_size, seq_len, input_size]`，所以这里显式设置 `batch_first=True`，可以直接承接 `x` 的输入，避免手动 `transpose`。

#### 编码器张量形状变化表

| 阶段 | 张量 | 形状 | 说明 |
|------|------|------|------|
| 输入 | `input` | `[1, 6]` | batch=1，6 个英文词索引 |
| 初始隐层 | `h0` | `[1, 1, 256]` | `(num_layers*num_dirs, batch, hidden)` |
| Embedding 后 | `input_x` | `[1, 6, 256]` | 每个词变 256 维向量 |
| GRU 输出 | `output` | `[1, 6, 256]` | 每个时间步的隐层 |
| GRU 末隐层 | `hidden` | `[1, 1, 256]` | 即上下文向量 C |

### 避坑与局限

- `nn.GRU` 默认 `batch_first=False`，此时输入应为 `[seq_len, batch, input_size]`；本案例设 `batch_first=True` 才能直接吃 `[batch, seq_len, input]`。
- `Embedding` 的输入维度 = 英文词表大小（含 SOS/EOS），输出维度 = `hidden_size`，二者相等只是本案例的简化设定，并非必须。

## 解码器（无Attention / 带Attention）🔺 难点

### 定义与本质

解码器在上下文 C 的基础上逐时间步生成法文单词。本案例给出两种解码器：**无 Attention**（仅用 C 与上一隐层）和**带 Attention**（每步动态计算源句各位置权重）。带 Attention 的关键在于引入 Q/K/V 三要素，使解码器在每个时间步都能"回看"编码器输出。

### 基础用法

#### 无 Attention 解码器 DecoderGRU

无 Attention 解码器：输入是一个词（`[1,1]`），经 Embedding → ReLU → GRU → Linear → LogSoftmax，输出该步的法文词概率分布。它每步只依赖上一步的隐层 `hidden`（初始为编码器的 C），无法再访问编码器中间时间步。

```python
#6. 构建没有attention的GRU解码器
class DecoderGRU(nn.Module):
    def __init__(self, vocab_size, hidden_size):
        super().__init__()
        # vocab_size代表解码器中法语单词的词汇总量（去重）
        self.vocab_size = vocab_size
        # hidden_size词嵌入维度
        self.hidden_size = hidden_size
        # Embedding层
        self.embed = nn.Embedding(self.vocab_size, self.hidden_size)

        # GRU层:这里定义GRU模型的输入和输出形状一样
        self.gru = nn.GRU(self.hidden_size,  self.hidden_size, batch_first=True)

        # 定义输出层:判断法语单词属于self.vocab_size里面的哪一个
        self.out = nn.Linear(self.hidden_size, self.vocab_size)

        # 定义softmax层
        self.softmax = nn.LogSoftmax(dim=-1)

    def forward(self, input, hidden):
        # input输入一般是一个字，解码的时候，是一个字符一个字符解码的
        # 1. input-->[1, 1]-->embeding之后--》[1, 1, hidden_size]-->[1, 1, 256]
        input_x = self.embed(input)
        # 2. relu激活函数使用
        input_x = F.relu(input_x)
        # 3. 将数据送入GRU模型:input_x-->[1,1,256],hidden:[1, 1, 256]
        # output-->[1, 1, 256]
        output, hidden = self.gru(input_x, hidden)

        # 4. 将output结果取最后一个词隐藏层输出送入linear层
        # output-->[1, vocab_size]-->[1, 4345]
        output = self.out(output[0])
        return self.softmax(output), hidden

    def inithidden(self):
        return torch.zeros(1, 1, self.hidden_size, device=device)
```

#### 带 Attention 解码器 AttentionDecoderGRU

带 Attention 解码器需要三个参数 Q、K、V。本案例中的对应关系（关键）：

| 注意力要素 | 本案例对应来源 | 形状 | 含义 |
|-----------|---------------|------|------|
| **Q (Query)** | 解码器当前输入 `input` 经 Embedding 后的 `input_x` | `[1, 1, 256]` | "我现在要查什么" |
| **K (Key)** | 上一时间步隐藏层输出 `hidden` | `[1, 1, 256]` | "源端各位置的钥匙" |
| **V (Value)** | 编码器输出 `encoder_output`（补齐到 `MAX_LENGTH`） | `[MAX_LENGTH, 256]` | "源端各位置的内容" |

> [!note] 💡 AI 扩展（进阶）
> **Q/K/V 的本质与计算流程**：注意力机制用 Q 去和所有 K 计算相关度，再用归一化权重对 V 加权求和，得到当前步对源句的"聚焦表示"。
> - **原理**：`score = softmax(Q·K^T)` → `context = score·V`；本案例用两个 `Linear` 层先把 `[query;key]` 拼接送入 `attn` 得到 `[1, MAX_LENGTH]` 权重，再 `bmm` 与 V 相乘得到 `[1,1,256]` 的 context。
> - **实战**：由于训练语料句长不超过 10，用 `MAX_LENGTH` 把编码器输出补齐成定长，方便统一计算注意力；超过部分用 0 填充。
> - **性能**：相比无 Attention，长句翻译 BLEU 显著提升，代价是每步多一次 `[1, MAX_LENGTH]` 的权重计算与 `bmm`。

```python
# 7.带attention的解码器
class AttentionDecoderGRU(nn.Module):
    def __init__(self, vocab_size, hidden_size, dropout_p=0.1, max_length=MAX_LENGTH):
        super().__init__()
        # vocab_size：属于解码器端，代表法语的总的单词个数
        self.vocab_size = vocab_size
        # hidden_size:代表词嵌入的维度
        self.hidden_size = hidden_size
        # 随机失活概率
        self.dropout_p = dropout_p
        # 最大句子长度：因为训练语料里面不管英文还是法文最大句子长度都不超过10，我们这里限定最大长度，目的是方便计算注意力
        self.max_length = max_length

        # 定义Embedding层
        self.embed = nn.Embedding(self.vocab_size, self.hidden_size)
        # 计算注意力的第一个全连接层：得到权重分数
        self.attn = nn.Linear(self.hidden_size*2, self.max_length)
        # 随机失活层
        self.droupout = nn.Dropout(p=self.dropout_p)
        # 计算注意力的第二个全连接层：让注意力按照指定维度输出
        self.attn_combin = nn.Linear(self.hidden_size*2, self.hidden_size)
        # 定义GRU层
        self.gru = nn.GRU(self.hidden_size, self.hidden_size, batch_first=True)
        # 定义输出层
        self.out = nn.Linear(self.hidden_size, self.vocab_size)

        # 定义softmax层
        self.softmax = nn.LogSoftmax(dim=-1)

    def forward(self, input, hidden, encoder_output):
        # input-->query--》解码器输入某个词[1, 1]
        # hidden-->key--》上一时间步隐藏层输出[1, 1, 256]
        # encoder_output--->value-->编码器的输出结果[10, 256]--[max_length, 256]
        # 1. 将input送入embedding-->input_x-->[1,1,256]--query(真正)
        input_x = self.embed(input)
        # 1.1对input_x进行dropout
        input_x = self.droupout(input_x)
        # 2. 计算注意力权重分数self.attn_weight-->[1, 10]-->[1, max_length]
        attn_weight = F.softmax(self.attn(torch.cat((input_x[0], hidden[0]), dim=-1)), dim=-1)
        # 3. 将注意力权重和Value相乘:[1, 1,10]*[1,10,256]-->self.attn_applied-->[1, 1,256]
        attn_applied = torch.bmm(attn_weight.unsqueeze(0), encoder_output.unsqueeze(0))
        # 4.将query和self.attn_applied结果拼接之后，再经过线性的变换self.output1-->[1, 1, 256]
        output1 = self.attn_combin(torch.cat((input_x[0], attn_applied[0]), dim=-1)).unsqueeze(0)
        # 5. 经过relu激活函数
        relu_output = F.relu(output1)
        # 6. 将self.relu_output，以及hidden送入GRU模型中-->gru_output-->[1, 1,256]
        gru_output, hidden = self.gru(relu_output , hidden)

        # 7.将gru的结果送入输出层，得到最后的预测结果output-->[1, 4345]
        output = self.out(gru_output[0])
        return self.softmax(output), hidden, attn_weight

    def inithidden(self):
        return torch.zeros(1, 1, self.hidden_size, device=device)
```

#### 带 Attention 解码器张量形状变化表

| 步骤 | 张量 | 形状 | 说明 |
|------|------|------|------|
| 输入 input | `input` | `[1, 1]` | 当前步词索引 |
| 输入 hidden(K) | `hidden` | `[1, 1, 256]` | 上一步隐层 |
| 输入 encoder_output(V) | `encoder_output` | `[10, 256]` | 补齐到 MAX_LENGTH |
| Embedding | `input_x`(Q) | `[1, 1, 256]` | 查询向量 |
| cat(Q,K) | 拼接 | `[1, 512]` | 送入 attn |
| attn 权重 | `attn_weight` | `[1, 10]` | 对源句 10 位置的相关度 |
| bmm(权重,V) | `attn_applied` | `[1, 1, 256]` | 加权后的上下文 |
| cat(Q,context) | 拼接 | `[1, 512]` | 送入 attn_combin |
| GRU 输出 | `gru_output` | `[1, 1, 256]` | 当前步隐层 |
| Linear+LogSoftmax | `output` | `[1, vocab_size]` | 法文词分布 |

> [!note] 💡 AI 扩展（基础）
> **为何要把 encoder_output 补齐到 MAX_LENGTH**：不同源句长度不同，但 `self.attn = nn.Linear(hidden_size*2, max_length)` 的输出维度固定为 `max_length`，因此 `attn_weight` 恒为 `[1, MAX_LENGTH]`。为了让 `bmm(attn_weight, encoder_output)` 维度对齐，必须把编码器输出也补齐成 `[MAX_LENGTH, hidden_size]`，多余位置用 0 填充（对应权重经 softmax 后被稀释，影响可忽略）。

### 避坑与局限

- `torch.bmm` 要求输入是 3D 张量，因此 `attn_weight`（`[1,10]`）需 `unsqueeze(0)` 成 `[1,1,10]`，`encoder_output`（`[10,256]`）需 `unsqueeze(0)` 成 `[1,10,256]`。
- 补齐 `encoder_output_c` 时，循环范围用 `encoder_output.shape[1]`（真实句长），不能用 `MAX_LENGTH`，否则越界。
- 无 Attention 解码器长句翻译效果差，这是引入 Attention 的根本动机。

## 模型训练与预测 🔸 核心

### 定义与本质

训练流程遵循标准 PyTorch 范式：实例化编码器/解码器 → 实例化优化器（编码器、解码器各一个）与损失函数 → 双层循环（epoch × dataloader）→ 梯度清零/反向/更新 → 保存模型。核心技巧是 **teacher_forcing 策略**：训练时按一定概率用真实标签而非模型预测作为下一步输入。

训练基本过程：


#### 训练内部迭代函数 Train_Iters（含 teacher_forcing）

`Train_Iters` 是单步训练的核心：先把 `x` 送入编码器得到 `encoder_output`（V）与 `encoder_hidden`（K 的初值），并补齐 V；再用 `SOS_token` 作为解码器首步输入（Q 的初值）；然后按 `teacher_forcing_ratio` 概率选择策略。

```python
# 定义内部迭代函数
def Train_Iters(x, y, my_encoder, my_attenDecoder, encoder_optimizer, decoder_optimizer, crossentropy):
    # 1.将x送入编码器得到编码的结果
    h0 = my_encoder.inithidden()
    encoder_output, encoder_hidden = my_encoder(x, h0)
    # 2. 定义解码器的参数
    # 2.1 中间语意张量C：value
    encoder_output_c = torch.zeros(MAX_LENGTH, my_encoder.hidden_size, device=device)
    for i in range(x.shape[1]):
        encoder_output_c[i] = encoder_output[0][i]
    # 2.2 解码器的初始化的hidden, key
    decoder_hidden = encoder_hidden
    # 2.3 解码器的初始化输出：query
    input_y = torch.tensor([[SOS_token]], dtype=torch.long, device=device)

    # 3.定义一个初始化的损失
    my_loss = 0.0
    # 4.选择性的使用teacher_forcing策略
    teacher_forcing = True if random.random() < teacher_forcing_ratio else False
    # 5.开始计算损失
    if teacher_forcing:
        for i in range(y.shape[1]):
            # output_y--》[1, 4345]
            output_y, decoder_hidden, attn_weight =my_attenDecoder(input_y, decoder_hidden, encoder_output_c)
            # 根据预测结果计算损失
            target_y = y[0][i].view(1)
            my_loss = my_loss + crossentropy(output_y, target_y)
            # 将真实的下一个单词当作input_y
            input_y = y[0][i].view(1, -1)
    else:
        for i in range(y.shape[1]):
            # output_y--》[1, 4345]
            output_y, decoder_hidden, attn_weight = my_attenDecoder(input_y, decoder_hidden, encoder_output_c)
            # 根据预测结果计算损失
            target_y = y[0][i].view(1)
            my_loss = my_loss + crossentropy(output_y, target_y)
            topv, topi = output_y.topk(1)
            # 如果output_y预测的最大值对应的索引刚好等EOS，直接终止
            if topi.squeeze().item() == EOS_token:
                break
            # 将预测结果的当作下一个input_y
            input_y = topi.detach()

    # 6. 梯度清零
    encoder_optimizer.zero_grad()
    decoder_optimizer.zero_grad()
    # 7. 反向传播
    my_loss.backward()
    # 8.梯度更新
    encoder_optimizer.step()
    decoder_optimizer.step()

    return my_loss / y.shape[1]
```

#### teacher_forcing 策略详解

| 策略 | 下一步输入来源 | 训练收敛 | 推理一致性 | 适用场景 |
|------|--------------|---------|-----------|---------|
| **teacher_forcing=True** | 真实标签 `y[0][i]` | 快、稳定 | 不一致（推理无标签） | 训练前期、加速收敛 |
| **teacher_forcing=False** | 模型预测 `topi.detach()` | 慢、易发散 | 一致（与推理同） | 训练后期、缓解暴露偏差 |

本案例用 `teacher_forcing_ratio`（如 0.5）控制概率：`teacher_forcing = True if random.random() < teacher_forcing_ratio else False`，每个样本随机选一种，兼顾收敛速度与推理鲁棒性。

> [!note] 💡 AI 扩展（进阶）
> **teacher_forcing 的暴露偏差（exposure bias）问题**：训练时总用真实标签做下一步输入，模型从未见过自己的错误输出；推理时只能用自己上一步的预测，一旦某步出错，错误会沿时间步累积放大。
> - **原理**：训练分布 `P(y_t | y_{<t}^*)` 与推理分布 `P(y_t | \hat y_{<t})` 不一致。
> - **实战**：常用 Scheduled Sampling（计划采样）逐步降低 teacher_forcing_ratio，或用 Beam Search 缓解。
> - **性能**：teacher_forcing_ratio 越高训练越快但暴露偏差越大；0.5 是常见折中。

#### 模型预测 evaluate_seq2seq

预测流程与训练的"非 teacher_forcing"分支一致，但需 `with torch.no_grad()` 关闭梯度，并加载训练好的参数。每步记录注意力权重，最终用 `matshow` 可视化。

预测基本过程：

```text
1.获取数据
2.数据预处理
3.实例化模型: 编码器和解码器
4.加载模型训练好的参数: model.load_state_dict(torch.load("model_path"))
5.with torch.no_grad():
6.将数据送入模型进行预测（注意:张量的形状变换）
```

```python
def evaluate_seq2seq(x, encoder, decoder):
    # x代表当前需要翻译的英文文本
    # encoder代表编码器模型
    # decoder代表解码器模型
    # 1. 将x送入编码器得到编码结果
    h0 = encoder.inithidden()
    encoder_outputs, encoder_hidden = encoder(x, h0)
    # 2. 准备解码的参数
    # 2.1 编码器结果--Value
    encoder_outputs_c = torch.zeros(MAX_LENGTH, encoder.hidden_size, device=device)
    for index in range(x.shape[1]):
        encoder_outputs_c[index] = encoder_outputs[0][index]
    # 2.2 解码器的key
    decode_hidden = encoder_hidden

    # 2.3 解码器的原始输入
    input_y = torch.tensor([[SOS_token]], dtype=torch.long, device=device)

    # 3.准备变量
    # 3.1 存储模型的预测结果
    decoder_words = []
    # 3.2 初始化一个全0的权重矩阵：存储每一步解码出来的注意力权重
    attention_weights = torch.zeros(MAX_LENGTH, MAX_LENGTH, device=device)
    # 4.进行模型的预测
    index = 0

    for i in range(MAX_LENGTH):
        output_y, decode_hidden, atten = decoder(input_y, decode_hidden, encoder_outputs_c)
        # 获取output_y预测的最大概率值对应索引
        topv, topi  = output_y.topk(1)
        # 将真实的注意力权重赋值给attention_weights
        attention_weights[i] = atten
        if topi.item() == EOS_token:
            decoder_words.append("<EOS>")
            break
        else:
            decoder_words.append(french_index2word[topi.item()])
        input_y = topi.detach()
        index = i

    print(f'attention_weights--》{attention_weights}')
    return decoder_words, attention_weights[:index+1]
```

### 避坑与局限

- 损失函数用 `nn.NLLLoss()` 配合 `LogSoftmax`（等价于 CrossEntropy，因为 `LogSoftmax + NLLLoss = CrossEntropy`）；若解码器输出已做 `LogSoftmax`，则不能再用 `CrossEntropyLoss`，否则会重复取 log。
- 预测时 `input_y = topi.detach()` 必须 `detach`，否则会保留计算图导致显存泄漏。
- `matshow` 绘图前需把张量转 CPU 并 `detach`：`attention_weights.cpu().detach().numpy()`。
- GPU 训练保存的模型在 CPU 加载时需 `map_location="cpu"`（本案例预测函数中已用 `.to(device)` 统一设备，加载时注意设备一致）。

## 4️⃣ 避坑指南 & 易错对比

> [!warning] 易混概念对比：无 Attention vs 带 Attention 解码器
>
> | 对比维度 | 无 Attention 解码器 | 带 Attention 解码器 |
> |---------|--------------------|--------------------|
> | 上下文来源 | 固定的 C（编码器末隐层） | 每步动态加权编码器输出 |
> | 能否回看源句中间步 | 否 | 是 |
> | 长句表现 | 差（信息瓶颈） | 好 |
> | 额外参数 | 无 | 2 个 Linear（attn、attn_combin） |
> | 输出 | `(output, hidden)` | `(output, hidden, attn_weight)` |
> | forward 参数 | `(input, hidden)` | `(input, hidden, encoder_output)` |

> [!warning] 易混概念对比：teacher_forcing 两种分支
>
> | 对比维度 | teacher_forcing=True | teacher_forcing=False |
> |---------|---------------------|----------------------|
> | 下一步 input_y | `y[0][i].view(1,-1)` 真实标签 | `topi.detach()` 模型预测 |
> | 是否提前终止 | 否（遍历完整个 y） | 是（预测到 EOS 即 break） |
> | 训练稳定性 | 高 | 低（错误累积） |
> | 与推理一致性 | 不一致 | 一致 |

> [!danger] 常见错误与规避
> 1. **损失函数重复取 log**：解码器输出已 `LogSoftmax`，若再用 `CrossEntropyLoss` 会重复 log 导致数值错误。规避：配 `NLLLoss`，或去掉 `LogSoftmax` 改用 `CrossEntropyLoss`。
> 2. **`bmm` 维度不匹配**：`attn_weight` 是 `[1,10]` 直接 `bmm` 会报错。规避：`unsqueeze(0)` 成 `[1,1,10]`，V 也 `unsqueeze(0)` 成 `[1,10,256]`。
> 3. **预测时未 detach**：`input_y = topi`（未 detach）会保留计算图，多步后显存爆炸。规避：必须 `topi.detach()`。
> 4. **`batch_first` 不一致**：编码器设 `batch_first=True`，但若忘了在解码器 GRU 也设，会导致维度错乱。规避：编/解码器 GRU 的 `batch_first` 保持一致。
> 5. **GPU/CPU 设备不一致**：模型在 GPU、输入在 CPU 会报错。规避：输入张量与模型都 `.to(device)`；CPU 加载 GPU 模型用 `map_location="cpu"`。
