# [day03] 学习笔记｜文本数据分析与特征处理（AI 增强版）

## 📌 核心速览

> [!summary] 核心速览
> - **文本数据分析**：有效帮助理解数据语料、快速检查语料可能存在的问题，是建模前的必要探索。
> - **标签分布分析**：用 `sns.countplot` 查看正负样本比例，指导是否需要数据增强/删减以维持平衡。
> - **句子长度分布**：用 `countplot`+`displot` 查看长度范围，为后续截断补齐的合理长度提供依据。
> - **散点图分析**：用 `sns.stripplot` 按标签分组查看长度散点，定位异常点位置辅助人工审查。
> - **词汇统计与词云**：用 `jieba`+`chain` 统计词汇总数，用 `jieba.posseg`+`WordCloud` 生成形容词词云评估语料质量。
> - **N-gram 特征**：n 个词相邻共现特征（常用 bi-gram/tri-gram），为语料加入局部顺序信息增强模型。
> - **文本长度规范**：用 `pad_sequences` 对文本张量截断补齐到固定长度，满足模型等尺寸输入要求。
> - **回译增强**：翻译成小语种再译回原语言，得到同标签新语料，是效果较好的文本增强方法。

## 1️⃣ 完整知识库

### 文本数据分析总述 🔹 基础

#### 定义与本质

**文本数据分析定义**：文本数据分析能够有效帮助我们理解数据语料，快速检查出语料可能存在的问题。

它是建模前的探索性数据分析（EDA）环节，目的不是训练模型，而是==在动手建模前摸清数据的分布、质量与潜在问题==，从而指导后续的特征工程与数据增强策略。

文本数据分析的常见维度包括：标签数量分布、句子长度分布、正负样本散点图、不同词汇总数统计、形容词词云等，下面逐一展开。

---

### 标签数量分布分析 🔸 核心

#### 定义与本质

**作用**：在深度学习模型评估中，我们一般使用 ACC 作为评估指标。若想将 ACC 的基线定义在 50% 左右，则需要正负样本比例维持在 1:1 左右，否则就要进行必要的数据增强或数据删减。若训练和验证集正负样本都稍有不均衡，可以进行一些数据增强。

> [!note] 💡 AI 扩展（基础）
> **数据不平衡的处理策略**：
> - **背景**：当正负样本比例悬殊时，模型倾向于预测多数类，ACC 虚高但实际泛化差，此时 ACC 不再是可靠指标。
> - **实战**：常用对策有三类——① 过采样少数类（如回译增强）；② 欠采样多数类；③ 调整类别权重 / 换用 F1、AUC 等指标。标签分布分析正是判断"是否需要这些对策"的第一步。

#### 基础用法

```python
def dm_label_sns_countplot():
    # 1 设置显示风格
    plt.style.use('fivethirtyeight')

    # 2 pd.read_csv 读训练集、验证集数据
    train_data = pd.read_csv(filepath_or_buffer='./cn_data/train.tsv', sep='\t')
    dev_data = pd.read_csv(filepath_or_buffer='./cn_data/dev.tsv', sep='\t')

    # 3 sns.countplot() 统计 label 标签的 0、1 分组数量
    sns.countplot(x='label', data=train_data)

    # 4 画图展示
    plt.title('train_label')
    plt.show()

    # 验证集上标签的数量分布
    sns.countplot(x='label', data=dev_data)
    plt.title('dev_label')
    plt.show()
```

> [!warning] 本代码依赖外部文件：./cn_data/train.tsv、./cn_data/dev.tsv（需含 label 列）

#### 避坑与局限

- `sns.countplot` 的 `x` 参数指定要统计的列名，`data` 指定 DataFrame。
- 标签分布只反映数量平衡，不反映样本难度，需结合其他分析综合判断。

### 句子长度分布分析 🔸 核心

#### 定义与本质

**作用**：通过绘制句子长度分布图，可以得知语料中大部分句子长度的分布范围。因为模型的输入要求为固定尺寸的张量，合理的长度范围对之后进行句子截断补齐（规范长度）起到关键的指导作用。

#### 基础用法

```python
def dm_len_sns_countplot_distplot():
    # 1 设置显示风格
    plt.style.use('fivethirtyeight')

    # 2 pd.read_csv 读训练集、验证集数据
    train_data = pd.read_csv(filepath_or_buffer='./cn_data/train.tsv', sep='\t')
    dev_data = pd.read_csv(filepath_or_buffer='./cn_data/dev.tsv', sep='\t')

    # 3 求数据长度列，然后求数据长度的分布
    train_data['sentence_length'] = list(map(lambda x: len(x), train_data['sentence']))

    # 4 绘制数据长度分布图-柱状图
    sns.countplot(x='sentence_length', data=train_data)
    plt.xticks([])  # x 轴上不要提示信息
    plt.show()

    # 5 绘制数据长度分布图-曲线图
    sns.displot(x='sentence_length', data=train_data)
    plt.yticks([])  # y 轴上不要提示信息
    plt.show()

    # 验证集
    dev_data['sentence_length'] = list(map(lambda x: len(x), dev_data['sentence']))
    sns.countplot(x='sentence_length', data=dev_data)
    plt.xticks([])
    plt.show()
    sns.displot(x='sentence_length', data=dev_data)
    plt.yticks([])
    plt.show()
```

> [!warning] 本代码依赖外部文件：./cn_data/train.tsv、./cn_data/dev.tsv（需含 sentence 列）

#### 避坑与局限

- 长度按字符数（`len(x)`）统计，未分词；若按词统计需先 `jieba.lcut` 再取长度，二者分布可能差异较大。
- `plt.xticks([])` 用于隐藏过密的 x 轴刻度，避免长文本重叠。

### 正负样本散点图分析 🔸 核心

#### 定义与本质

**作用**：通过查看正负样本长度散点图，可以有效定位异常点的出现位置，帮助我们更准确进行人工语料审查。

#### 基础用法

```python
def dm03_sns_stripplot():

```

> [!warning] 本代码依赖外部文件：./cn_data/train.tsv、./cn_data/dev.tsv

#### 避坑与局限

- `sns.stripplot` 的 `x` 是分类轴（label），`y` 是数值轴（长度），不要写反。
- 散点图能发现极端长/短句子，但无法判断语义异常，需配合人工阅读。

### 不同词汇总数统计 🔹 基础

#### 基础用法

```python
# 导入 jieba 用于分词
# 导入 chain 方法用于扁平化列表
import jieba
from itertools import chain

# 进行训练集的句子分词，并统计出不同词汇的总数
train_vocab = set(chain(*map(lambda x: jieba.lcut(x), train_data["sentence"])))
print("训练集共包含不同词汇总数为：", len(train_vocab))

# 进行验证集的句子分词，并统计出不同词汇的总数
valid_vocab = set(chain(*map(lambda x: jieba.lcut(x), valid_data["sentence"])))
print("训练集共包含不同词汇总数为：", len(valid_vocab))
```

> [!warning] 本代码依赖外部变量：train_data、valid_data（需先由 pd.read_csv 加载）

#### 避坑与局限

- `chain(*map(...))` 的作用是把"列表的列表"扁平化为一个迭代器，再用 `set()` 去重得到词汇集合。
- 词汇总数反映语料规模，但无法反映词汇质量（如是否含大量噪声词）。

### 形容词词云分析 🔸 核心

#### 定义与本质

**作用**：根据高频形容词词云显示，我们可以对当前语料质量进行简单评估，同时对违反语料标签含义的词汇进行人工审查和修正，来保证绝大多数语料符合训练标准。

#### 基础用法

```python
# 使用 jieba 中的词性标注功能
import jieba.posseg as pseg
from wordcloud import WordCloud

# 每句话产生形容词列表
def get_a_list(text):
    r = []
    # 使用 jieba 的词性标注方法切分文本，找到形容词存入列表中返回
    for g in pseg.lcut(text):
        if g.flag == "a":
            r.append(g.word)
    return r

# 根据词云列表产生词云
def get_word_cloud(keywords_list):
    # 实例化词云生成器对象
    wordcloud = WordCloud(font_path="./SimHei.ttf", max_words=100, background_color='white')
    # 准备数据
    keywords_string = " ".join(keywords_list)
    # 产生词云
    wordcloud.generate(keywords_string)

    # 画图
    plt.figure()
    plt.imshow(wordcloud, interpolation="bilinear")
    plt.axis('off')
    plt.show()


# 思路分析：训练集正样本词云、训练集负样本词云
# 1 获得训练集上正样本 p_train_data
#   eg: 先使用逻辑 == 操作检索符合正样本 train_data[train_data['label'] == 1]
# 2 获取正样本的每个句子的形容词 p_a_train_vocab = chain(*map(a, b))
# 3 调用绘制词云函数
def dm_word_cloud():
    # 1 获得训练集上正样本 p_train_data
    train_data = pd.read_csv(filepath_or_buffer='./cn_data/train.tsv', sep='\t')
    p_train_data = train_data[train_data['label'] == 1]['sentence']

    # 2 获取正样本的每个句子的形容词
    p_a_train_vocab = list(chain(*map(lambda x: get_a_list(x), p_train_data)))

    # 3 调用绘制词云函数
    get_word_cloud(p_a_train_vocab)

    print('*' * 60)

    # 训练集负样本词云
    n_train_data = train_data[train_data['label'] == 0]['sentence']
    n_a_train_vocab = chain(*map(lambda x: get_a_list(x), n_train_data))

    # 3 调用绘制词云函数
    get_word_cloud(n_a_train_vocab)
```

> [!warning] 本代码依赖外部文件：./cn_data/train.tsv、./SimHei.ttf（中文字体，否则中文显示为方块）

> [!note] 💡 AI 扩展（基础）
> **词云的工程意义与配置要点**：
> - **背景**：词云把词频映射为字号大小，直观呈现语料中的高频描述词，是语料质量速览的利器。
> - **实战要点**：中文词云必须指定 `font_path`（如 SimHei.ttf），否则中文显示为乱码方块；通过 `jieba.posseg` 只取形容词（`flag == 'a'`）而非全部词，能聚焦"情感/评价性"词汇，更利于情感分类任务的语料审查。

#### 避坑与局限

- 不指定中文字体会导致词云中文显示为方块。
- 只看形容词可能遗漏名词性关键信息，必要时可对比全词词云。

### 文本特征处理（N-gram + 长度规范 + 回译增强）🔸 核心

#### 定义与本质

**文本特征处理的作用**：文本特征处理包括为语料添加具有普适性的文本特征（如 N-gram 特征），以及对加入特征之后的文本语料进行必要的处理（如长度规范）。这些特征处理工作能够有效地将重要的文本特征加入模型训练中，增强模型评估指标。

**常见的文本特征处理方法**：
- 添加 N-gram 特征
- 文本长度规范

**文本数据增强方法**：回译数据增强。

#### 基础用法

**（1）添加 N-gram 特征**

**定义**：给定一段文本序列，其中 n 个词或字的相邻共现特征即 N-gram 特征。常用的 N-gram 特征是 bi-gram 和 tri-gram 特征，分别对应 n 为 2 和 3。

下图对比了"不添加 N-gram"与"添加 bi-gram"两种方式下的文本特征表示与相似度计算：

![[day03_ngram_method.png]]

如图，左侧（红色）仅用 unigram 词袋表示，文本 1 与文本 2 的欧氏距离为 √3；右侧（绿色）加入 bi-gram 后，特征空间从 5 维扩展到 9 维（新增"我是""你是""是黑马""黑马人"等相邻共现特征），距离变为 √6。这说明==N-gram 把"局部词序"信息编入特征==，能更好区分语义差异（如"我是黑马人"与"你是黑马"）。

代码：

```python
# 一般 n-gram 中的 n 取 2 或者 3，这里取 2 为例
ngram_range = 2

def create_ngram_set(input_list):
    """
    description: 从数值列表中提取所有的 n-gram 特征
    :param input_list: 输入的数值列表，可以看作是词汇映射后的列表，
                       里面每个数字的取值范围为 [1, 25000]
    :return: n-gram 特征组成的集合

    eg:
    >>> create_ngram_set([1, 3, 2, 1, 5, 3])
    {(3, 2), (1, 3), (2, 1), (1, 5), (5, 3)}
    """
    return set(zip(*[input_list[i:] for i in range(ngram_range)]))
```

> [!note] 💡 AI 扩展（进阶）
> **N-gram 的维度爆炸与特征选择**：
> - **原理**：加入 bi-gram 后特征数近似平方增长（n 个词最多产生 n-1 个 bi-gram），tri-gram 更甚，容易引发维度爆炸与稀疏问题。
> - **实战**：工程上通常用 TF-IDF 加权替代简单的 0/1 出现标记，并用卡方检验、互信息等方法做特征选择，只保留对分类最有区分度的 N-gram，控制特征维度。

**（2）文本长度规范及其作用**

**作用**：一般模型的输入需要等尺寸大小的矩阵，因此在进入模型前需要对每条文本数值映射后的长度进行规范。此时将根据句子长度分布分析出覆盖绝大多数文本的合理长度，对超长文本进行截断。

代码：

```python
from tensorflow.keras.preprocessing import sequence

# cutlen 根据数据分析中句子长度分布，覆盖 90% 左右语料的最短长度
# 这里假定 cutlen 为 10
cutlen = 10

def padding(x_train):
    """
    description: 对输入文本张量进行长度规范
    :param x_train: 文本的张量表示，形如: [[1, 32, 32, 61], [2, 54, 21, 7, 19]]
    :return: 进行截断补齐后的文本张量表示
    """
    # 使用 sequence.pad_sequences 即可完成
    return sequence.pad_sequences(x_train, cutlen)
```

> [!warning] 本代码依赖外部环境：tensorflow（提供 `keras.preprocessing.sequence`）

> [!info] 🧠 速查卡片
> **pad_sequences 参数效果对比表**
>
> 假设原始序列 `[1, 2, 3, 4, 5]`，`maxlen=4`：
>
> | 参数组合 | 结果 | 说明 |
> |---------|------|------|
> | `padding='pre', truncating='pre'`（默认） | `[3, 4, 5, 6]`（补0在前面，从前截断） | 注意：默认行为，补0在前面，==超长从前面截断== |
> | `padding='post', truncating='pre'` | `[3, 4, 5, 6]`（补0在后面，从前截断） | 长度不足时尾部补0 |
> | `padding='pre', truncating='post'` | `[0, 2, 3, 4]`（补0在前面，从后截断） | 超长时保留前部，截断尾部 |
> | `padding='post', truncating='post'` | `[0, 2, 3, 4]`（补0在后面，从后截断） | 完全保留前部信息 |
>
> 假设原始序列 `[1, 2, 3]`（长度不足 `maxlen=6`）：
>
> | 参数组合 | 结果 | 说明 |
> |---------|------|------|
> | `padding='pre'`（默认） | `[0, 0, 0, 1, 2, 3]` | 前面补0 |
> | `padding='post'` | `[1, 2, 3, 0, 0, 0]` | 后面补0 |
>
> **实战选择建议**：
> - 文本分类任务中通常选 `padding='post'`，使模型先处理实际内容；
> - RNN 场景有时选 `padding='pre'` 配合 `pack_padded_sequence` 提升效率。

**（3）文本数据增强方法（回译）**

**定义**：回译数据增强目前是文本数据增强方面效果较好的增强方法。一般基于 google、有道等翻译接口，将文本数据翻译成另外一种语言（一般选择小语种），之后再翻译回原语言，即可认为得到与原语料同标签的新语料，新语料加入到原数据集中即可认为是对原数据集的数据增强。

**实现方式**：大模型（或翻译接口）。

#### 避坑与局限

- N-gram 的 n 取值不宜过大（一般 2~3），过大会导致特征维度爆炸且稀疏。
- `pad_sequences` 默认从前面补 0、截断后面，可通过 `padding`/`truncating` 参数调整前后位置。
- 回译要选择与原语言差异大的小语种以获得更多表达变化，但需人工抽检保证语义与标签一致。

### RNN 模型概述 🔸 核心

#### 定义与本质

**RNN（循环神经网络）定义**：当前时间步的输出 = 当前时间步的输入 + 上一时间步的隐层输出。它通过隐藏状态的循环传递，使网络具备"记忆"历史信息的能力。

**作用和应用场景**：很好地处理 NLP 各种任务，如文本翻译、文本分类等。

RNN 的核心机制如下图所示：

![[day03_rnn_structure.png]]

如图，RNN 在时间维度上展开：绿色框 A 代表同一网络在不同时间步（t-1、t、t+1）被重复使用。当前时间步 t 接收两个输入——当前输入 `x_t` 和上一时间步的隐藏状态 `h_{t-1}`，经 `tanh` 激活后得到当前隐藏状态 `h_t`。==`h_t` 同时承担两个职责：向上作为当前输出，向右传递给下一时间步作为"记忆"==。所有时间步共享同一组参数（权重共享），这是 RNN 处理变长序列的基础。

> [!note] 💡 AI 扩展（进阶）
> **RNN 的梯度消失问题为何引出 LSTM**：
> - **原理**：传统 RNN 在反向传播时，梯度需沿时间步连乘，当序列较长时梯度会指数级衰减（梯度消失）或爆炸（梯度爆炸），导致难以学习长距离依赖。
> - **进阶**：为此引入了 LSTM（引入门控与细胞状态）和 GRU（简化门控），通过"门"控制信息的保留与遗忘，缓解长程依赖问题。这几种变体的内部结构与 PyTorch 实现详见 [[4.RNN模型及其变体（AI增强版）]]。

#### 基础用法

**（1）根据输入与输出结构分类**

| 类型 | 特点 | 适用范围 |
|------|------|---------|
| N vs N | 输入和输出是等长的 | 诗歌、对联 |
| N vs 1 | 输入不限定长度，输出是唯一的 | 分类任务 |
| 1 vs N | 输入是唯一的，输出不限定长度 | 图片生成任务 |
| N vs M（seq2seq 架构） | 输入和输出不等长 | 文本翻译任务等 |

**（2）根据 RNN 内部结构分类**

```text
传统 RNN
LSTM
Bi-LSTM
GRU
Bi-GRU
```

这些变体在门控设计与结构上有所不同，详细原理与代码实现见 [[4.RNN模型及其变体（AI增强版）]]，本篇仅作概述。

#### 避坑与局限

- RNN 的"记忆"是局部的，长序列中早期信息易被遗忘（梯度消失），需用 LSTM/GRU 改进。
- 本篇只介绍 RNN 概念与分类，模型结构细节、PyTorch 实现与优缺点对比均归属第 4 篇，避免重复。

## 3️⃣ 代码库

数据分析代码（标签分布、长度分布、散点图、词汇统计、词云）均依赖外部数据集 `./cn_data/train.tsv` 与 `./cn_data/dev.tsv`，已分散在各主题块中并用 `[!warning]` 标注依赖。此处提供不依赖外部文件、可独立运行的 N-gram 特征提取与长度规范综合示例：

```python
from tensorflow.keras.preprocessing import sequence

# ============ 1 N-gram 特征提取 ============
ngram_range = 2

def create_ngram_set(input_list):
    """从数值列表中提取所有的 n-gram 特征"""
    return set(zip(*[input_list[i:] for i in range(ngram_range)]))

# 示例：对数值化后的句子提取 bi-gram
sample = [1, 3, 2, 1, 5, 3]
print("bi-gram 特征:", create_ngram_set(sample))
# 输出：bi-gram 特征: {(1, 3), (3, 2), (2, 1), (1, 5), (5, 3)}

# ============ 2 文本长度规范（截断补齐）============
cutlen = 10

def padding(x_train):
    """对输入文本张量进行长度规范"""
    return sequence.pad_sequences(x_train, cutlen)

# 示例：将不等长张量规范为长度 10
x_train = [[1, 32, 32, 61], [2, 54, 21, 7, 19, 3, 4, 5, 6, 7, 8, 9, 10]]
print("规范后:\n", padding(x_train))
# 输出：规范后:
#  [[ 0  0  0  0  0  0  1 32 32 61]
#   [ 3  4  5  6  7  8  9 10 54 21]]  （前面补 0，超长从前面截断）
```

> [!warning] 本代码 N-gram 部分纯 Python 可运行；padding 部分依赖 tensorflow

## 4️⃣ 避坑指南 & 易错对比

> [!warning] 核心理念
> **数据分析指导特征工程**：句子长度分布决定 `pad_sequences` 的 `cutlen`；标签分布决定是否数据增强；词云决定是否清洗噪声词。==数据分析不是可选步骤，而是特征处理参数的依据==。

| 分析维度 | 工具/API | 输出 | 指导后续 |
|---------|---------|------|---------|
| 标签分布 | `sns.countplot` | 正负样本数量柱状图 | 是否数据增强/删减 |
| 句子长度分布 | `sns.countplot`+`sns.displot` | 长度柱状图+曲线图 | `pad_sequences` 的 cutlen |
| 散点图 | `sns.stripplot` | 按标签分组的长度散点 | 定位异常点人工审查 |
| 词汇统计 | `jieba`+`chain`+`set` | 词汇总数 | 评估语料规模/词表大小 |
| 词云 | `jieba.posseg`+`WordCloud` | 高频形容词词云 | 语料质量评估与噪声清洗 |

| 特征处理方法 | 作用 | 关键参数 |
|------------|------|---------|
| N-gram 特征 | 加入局部词序信息 | `ngram_range`（取 2 或 3） |
| 文本长度规范 | 截断补齐到固定长度 | `cutlen`、`padding`/`truncating` |
| 回译增强 | 生成同标签新语料 | 翻译语种选择、语义抽检 |

> [!danger] 常见错误与规避
> - **词云中文乱码**：未指定 `font_path` 导致中文显示为方块。规避：`WordCloud(font_path="./SimHei.ttf", ...)`。
> - **N-gram n 过大**：n≥4 时特征维度爆炸且极度稀疏。规避：n 取 2~3，并用特征选择降维。
> - **pad_sequences 位置混淆**：默认前补 0、前截断，与预期不符。规避：用 `padding='post'`/`truncating='post'` 显式指定。
> - **回译破坏标签**：翻译往返可能改变情感极性。规避：人工抽检回译样本，剔除语义偏移的语料。
